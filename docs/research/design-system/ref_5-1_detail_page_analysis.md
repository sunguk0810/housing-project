## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장

**Niche에서:**

1. **종합 + 카테고리 등급 그리드**: 대형 종합 배지(좌) + 소형 카테고리 배지(우, 2열) 레이아웃. 우리 Hero 게이지(좌/상단) + 4점수 미니 게이지(2×2 그리드)와 직접 매핑
2. **카테고리 상세의 2컬럼**: 좌측 Key-Value 테이블 + 우측 시각화(바/폴) → 우리 카테고리 상세도 "수치 + 시각화" 조합
3. **"How are grades calculated?" 링크**: 등급 산정 방법론 공개 → 우리 "ⓘ" 3단계 공개의 3단계(전체 방법론)
4. **등급별 색상 차이**: A+~A = 그린(진한), B+~B = 블루그레이(연한) → **색상만으로 즉시 구분**. 다만 우리는 적녹색 금지이므로 블루→오렌지 다이버징 유지
5. **"Hide ∧" 접기/펼치기**: 등급 그리드를 접을 수 있음 → 우리 카테고리 상세도 접기 가능하게

**토스 주식에서:**

6. **레인지 바 (수평)**: 4-6px 높이 + 둥근 끝 + 녹색 포인트(●) → 우리 점수 범위 표시에 적용 가능. 예: "이 단지의 예산 점수는 상위 15-25% 범위"
7. **기간 탭**: 1일/1주/3달/1년/5년/전체 칩 → 우리 실거래가 차트의 "1년/3년/5년/전체" 기간 선택
8. **2×2 Key-Value 그리드**: 라벨(회색)+값(Bold) 2컬럼 → 우리 단지 기본 정보 표시에 직접 적용
9. **탭 내비게이션 스타일**: Bold + 하단 언더라인 — 깔끔하고 공간 효율적

**직방에서:**

10. **3탭 구조 (세대/시세/아파트)**: 정보를 성격별로 탭 분리 → 우리도 "개요/예산/통근/보육/안전" 탭 분리
11. **실거래 테이블**: 날짜 + "신고가" 배지 + 가격 + 평당가·층수 → 우리 실거래 내역에 직접 참고
12. **면책 문구 인라인**: "같은 단지 또는 관련도 높음 주변단지의..." 시세 산정 방법 설명 → 우리 면책 5접점 중 "결과메타" 적용
13. **출처 표기**: "ⓘ 직방에서 수집한 정보입니다." → 우리 "[국토부 실거래 2026.01]" 패턴

**호갱노노에서:**

14. **통근 시간 비교**: "가산 ▾ 30분 / 강남 ▾ 50분" 드롭다운+시간 → 우리 통근 점수 상세에서 "직장1: 30분 / 직장2: 45분" 표시
15. **버스 노선 컬러 배지**: 마을(녹)/간선(파)/지선(녹)/정기(빨) → 대중교통 상세에서 노선별 배지 색상 참고
16. **"지금 24명이 보는 중"**: 실시간 관심도 → 우리는 불필요하지만, 향후 "이 단지를 N명이 비교 중" 기능에 참고
17. **가격 차트 + 거래량 바**: 라인 차트 아래 작은 바 차트로 거래량 표시 → 우리 실거래가 차트에 적용

**Apple Health에서:**

18. **카드형 반복 구조**: "1줄 인사이트 + 큰 수치 + 미니 차트"가 카드로 반복 → 우리 카테고리별 상세를 카드 단위로 구성할 때 직접 적용
19. **오늘 vs 평균 비교**: 오렌지(오늘) vs 회색(평균) 나란히 → 우리 "이 단지 vs 전체 평균" 비교에 적용
20. **카드별 다른 차트**: 같은 카테고리도 카드마다 다른 시각화 → 우리도 예산(바), 통근(타임라인), 보육(거리맵), 안전(바) 카테고리별 최적 시각화 가능

### 💡 신규 인사이트

1. **탭 vs 스크롤 결정**
    - Niche: 탭(10개) + 사이드 내비
    - 토스: 탭(6개)
    - 직방: 탭(3개)
    - 호갱노노: 탭 없이 긴 스크롤
    - → 우리는 **4-5개 탭**(개요/예산/통근/보육/안전)이 적절. 모바일에서 5개까지 한 줄에 들어감

2. **Hero 영역의 정보 밀도**
    - Niche: 이름+등급+순위+CTA (4항목)
    - 토스: 이름+가격+변동 (3항목)
    - 직방: 이름+가격 (2항목)
    - → 우리: 단지명 + Hero 게이지(80-96px) + 등급 라벨 + 주소 + 핵심 스펙(세대수·준공·면적) → **5항목**, 가장 밀도가 높음
    - Hero 게이지가 시각적 앵커이므로 텍스트 정보가 많아도 시선 분산이 적을 것

3. **실거래 테이블의 "신고가" 배지**
    - 직방이 최고가 거래에 빨간 "신고가" pill 배지를 달아줌
    - → 우리도 실거래 내역에서 최고가/최저가에 배지 표시 가능. 단, **빨간색 금지 (안전 섹션이 아니어도 부정적 뉘앙스)** → 코랄(#F97316) 또는 블루(#1565C0) 사용

4. **Apple Health식 "인사이트 카드" 도입**
    - 각 카테고리 상세에 "1줄 인사이트"를 추가하면 숫자만 나열하는 것보다 의미 전달력↑
    - 예: 통근 카테고리 → "이 단지에서 가산디지털단지역까지 30분이에요. 평균(42분)보다 빠릅니다."
    - 예: 예산 카테고리 → "부부 합산 소득 대비 대출 부담이 적은 편이에요."
    - → Agent 구현 시 인사이트 텍스트 생성 로직 필요
