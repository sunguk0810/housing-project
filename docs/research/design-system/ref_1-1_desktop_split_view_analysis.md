# 1-1. 데스크톱 스플릿 뷰 분석 — Zillow / Redfin / Realtor / Homes

> **분석일**: 2026-02-17
> **분석 대상**: 4개 미국 부동산 서비스의 데스크톱 검색 결과 화면
> **목적**: 집콕신혼 `/results` 페이지 구현 시 AI Agent에게 전달할 레퍼런스

## 1. 서비스별 상세 분석

### 1-1. Zillow

**레이아웃**
- 비율: **지도 60% / 리스트 40%** (우리 스펙과 정확히 일치하되, 좌우가 반대 — Zillow는 지도 좌측, 리스트 우측)
- 리스트 패널: **2열 그리드**, 카드 간 간격 약 16-20px
- 지도와 리스트 사이 구분: 시각적 경계선 없음, 자연스러운 영역 분리

**카드 구조**
- **이미지 중심**: 카드 상단에 큰 썸네일 (약 280×180px 추정), 이미지 캐러셀 도트 인디케이터
- 1순위 정보: **가격** ($724,999) — 약 20-22px Bold, 검정
- 2순위: beds | ba | sqft — 약 14px, 파이프(|) 구분, 타입 라벨 ("House for sale") 인라인
- 3순위: 주소 — 약 13-14px, 회색 톤
- 4순위: 중개사 정보 — 약 10-11px ALL CAPS, 매우 연한 회색
- 우상단: 하트(찜) 아이콘, ··· 메뉴
- 좌상단 배지: "Open: Tue 5-6:30pm", "Showcase" 등 상태 라벨 (반투명 검정 배경 + 흰 텍스트)
- 라운딩: 약 8-10px (우리 16px보다 작음)
- 그림자: 거의 없음, hover 시 미세한 elevation 변화

**지도 마커**
- **가격 라벨 핀**: 짙은 빨강/마룬(#8B1A1A 추정) 배경 + 흰 텍스트
- 크기: 약 40-50px 너비, 높이 24-28px
- 모양: 둥근 직사각형(rounded pill), 하단에 뾰족한 꼬리(말풍선형)
- 텍스트: "$725K", "$2.95M" 등 축약 표기
- 특수 마커: ✦ 아이콘이 붙은 "Showcase" 마커 (차별화 강조)
- 클러스터: "400" 같은 숫자 원형 (줌 아웃 시)

**필터/정렬**
- 위치: **상단 고정 바** (검색창 아래)
- 스타일: 둥근 **드롭다운 칩** (For Sale ▾ / Price ▾ / Beds & Baths ▾ / Home Type ▾ / More ▾)
- 강조: "Save search" 버튼만 Blue 채움(#1277E1 추정)
- 정렬: 리스트 상단 "Sort: Homes for You ▾" — 우측 정렬, 드롭다운
- 지도 내: "Schools ▾", "Remove Boundary ×" 오버레이 칩

**색상 톤**
- Primary: **Blue** (#1277E1 추정) — Save search 버튼, 링크
- 배경: 순백 #FFFFFF
- 텍스트: #000000 (가격), #6B7280 (서브 텍스트)
- 마커: Dark Red/Maroon (#8B1A1A 추정)
- Accent: 없음 (매우 절제된 단색 사용)

### 1-2. Redfin

**레이아웃**
- 비율: **지도 100%** (풀스크린 지도 모드) — 리스트 토글("≡ List" 버튼)이 하단 중앙에 존재
- 일반적으로는 스플릿 뷰도 지원하지만, 이 캡처는 지도 전체 화면 모드
- 리스트와 지도 전환: 하단 중앙 "≡ List" 버튼으로 토글

**카드 구조** (이 캡처에서는 리스트 미노출)
- 리스트 모드 전환 시 좌측 패널 등장

**지도 마커** ⭐ 가장 다양한 마커 시스템
- **가격 라벨 핀 (기본)**: 흰 배경 + 검정 텍스트 ("649K", "1.20M", "310K")
    - 크기: 약 50-60px 너비, 20-24px 높이
    - 모양: 둥근 직사각형, 미세한 그림자
- **특수 마커**:
    - 오렌지 배경 ("900K", "1.25M") — 새 매물 또는 가격 변동?
    - "NEW" 라벨 첨부 마커
    - "2 units", "4 units" 클러스터 표시 (회색 라벨)
- **점 마커**: 작은 녹색 원형 (●) — 줌 레벨에 따라 가격 라벨과 혼용
- 우측 사이드 도구: Draw(그리기 검색), Options, Nearby 버튼

**필터/정렬**
- 이 캡처에서는 필터 바 미노출 (풀 맵 모드)
- 일반적으로 상단 필터 칩 바 사용

**색상 톤**
- Primary: **Red** (#A02A2A 추정) — 브랜드 로고, 일부 마커
- 마커 기본: 흰 배경 + 검정 텍스트
- 마커 강조: 오렌지 (#E8731A 추정)
- 점 마커: 녹색 (#4CAF50 추정)
- 전체적으로 지도 위 정보량이 매우 높음 (밀집)

### 1-3. Realtor.com

**레이아웃**
- 비율: **리스트 50% / 지도 50%** (좌측 리스트, 우측 지도)
- 리스트 패널: **2열 그리드**, 카드 간 간격 약 16-20px
- 상단에 "List | Map" 토글 탭 (우상단)

**카드 구조**
- **이미지 중심**: 카드 상단 대형 썸네일 (약 340×200px 추정), 비율 약 16:9
- 좌상단 배지: 중개사 로고(Coldwell Banker 등) + "New construction" / "New" 태그
    - 배지 스타일: 검정 반투명 pill + 흰 텍스트
- 우하단: 하트(찜) 아이콘 — 흰 원형 배경
- 1순위: ● 상태 인디케이터(녹색 점) + "Condo for sale" 타입 라벨
- **가격**: $4,750,000 — 약 22-24px Bold, 검정 (가장 크고 두꺼운 텍스트)
- 2순위: 4 bed · 4 bath · 3,049 sqft — 약 14px, 스페이스 구분
- 3순위: 주소 — 약 13px
- **CTA 버튼**: "Contact builder" / "Email Agent" — 아웃라인 스타일, 우하단
- 라운딩: 약 8-12px
- 그림자: 매우 미세, 카드 구분은 주로 간격으로

**지도 마커**
- **가격 라벨 핀**: 흰 배경 + 검정 텍스트 ("$699K", "$825K", "$1.3M")
    - 크기: 약 50-60px 너비
    - 모양: 둥근 직사각형, 미세한 보더
- "NEW" 배지가 붙은 마커 (마커 위 작은 라벨)
- 하단에 "Nearby", "Schools", "Neighborhoods" 레이어 토글 아이콘
- "Clear Boundary" / "Recenter" / "Satellite" 컨트롤

**필터/정렬**
- 위치: **상단 수평 바** (검색창 아래)
- 스타일: **칩(pill) + 드롭다운** 혼합
    - 아이콘 + 텍스트 칩: "⚙ Filters" / "Price ▾" / "Rooms ▾" / "Home type ▾"
    - 텍스트 전용 칩: "New construction" / "Min $100K" / "Hide pending" 등
    - 선택된 칩: 경량 보더, 비선택은 회색 보더
- 정렬: "Sort by Relevant listings ▾" — 리스트 좌상단
- "How much home can I afford?" CTA 링크 (우측)

**색상 톤**
- Primary: **검정 + 핑크/마젠타** (#D32F77 추정) — 로고, Save search 버튼
- 배경: 순백 #FFFFFF
- 텍스트: 검정 (가격), 회색 (서브)
- 칩: 흰 배경 + 회색 보더
- 전체적으로 깔끔한 흑백 기조

### 1-4. Homes.com

**레이아웃**
- 비율: **리스트 42% / 지도 58%** (좌측 리스트, 우측 지도)
- 리스트 패널: **1열** 수평 카드 (이미지 좌측 + 정보 우측)
- 카드 간 간격: 약 12-16px, 카드 간 미세한 수평 구분선

**카드 구조** ⭐ 가장 정보 밀도 높음
- **수평 레이아웃**: 좌측 이미지(약 240×160px) + 우측 정보
- 1순위: **가격** ($1,188,000) — 약 18-20px Bold + 아이콘 3개 (3D 투어, 비교, 찜)
- 2순위: 3 Beds · 2 Baths · 1,024 Sq Ft — 약 13px, dot(·) 구분
- 3순위: 주소 — 약 13px
- 4순위: **상세 설명문** — 약 12px, 3-4줄 (다른 서비스에는 없는 독특한 요소)
- 5순위: **에이전트 정보** — 프로필 사진(50×50px) + 이름 + 회사명 + 전화번호
  → 중개 연결이 핵심 비즈니스 모델임을 UI에서 드러냄
- 라운딩: 약 6-8px (가장 작은 라운딩)
- 그림자: 거의 없음

**지도 마커**
- **점 마커 (기본)**: 진한 블루/네이비(#1B3A5C 추정) 작은 원형 + 흰 테두리
    - 크기: 약 12-16px (매우 작음)
    - 밀집 지역에서 클러스터 느낌
- 가격 라벨 없음 — 가장 미니멀한 마커 시스템
- 지도 우측: 레이어, 교통, 자전거, 보행자, 지적도 아이콘 버튼

**필터/정렬**
- 위치: **상단 바** (검색창과 같은 행)
- 스타일:
    - "For Sale ▾" 드롭다운 (좌측)
    - 검색 입력창 (중앙, 넓음)
    - 차트 아이콘 + 필터 아이콘 (우측)
- 정렬: "Sort Recommended ▾" — 리스트 상단 우측
- "Save Search" 버튼 — 우상단, 아이콘 + 텍스트

**색상 톤**
- Primary: **Orange/Amber** (#F5A623 추정) — 로고 일부
- 배경: 순백 #FFFFFF
- 텍스트: 검정 (가격), 회색 (서브)
- 정렬 드롭다운: 주황 텍스트 "Recommended"
- 마커: 네이비 블루 (#1B3A5C 추정)
- 전체적으로 정보 밀도가 높고 텍스트 중심

## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장 (레퍼런스에서 가져올 것)

**Zillow에서:**
- **60:40 비율 검증**: Zillow가 정확히 60:40 비율을 사용하고 있어 우리 스펙의 타당성 확인됨
- **정렬 드롭다운 위치**: 리스트 상단 우측에 "Sort:" 배치 — 우리도 동일한 위치 권장
- **결과 수 + 지역명 표시**: "19,313 results" + "New York NY" 조합 → 우리는 "{지역명} 분석 결과 {N}건"
- **카드 이미지 캐러셀의 도트 인디케이터**: 우리는 이미지 없으므로 해당 없으나, 다른 시각 요소(게이지)에 참고

**Redfin에서:**
- **가격 라벨 핀의 크기감**: 흰 배경 + 검정 텍스트가 지도 위에서 가독성 최고 — 우리 Default 마커(흰 배경 + 등급 컬러)와 가장 유사
- **혼합 마커 시스템**: 줌 레벨에 따라 라벨 핀 ↔ 점 마커 전환 — 우리도 클러스터링 시 참고
- **"Draw" 검색 도구**: 우측 사이드 도구 → 향후 "그리기 검색" 기능 레퍼런스
- **"NEW" 배지 마커**: 상태 라벨을 마커에 부착하는 패턴 → 우리는 등급(A+~D) 라벨 부착에 응용 가능

**Realtor에서:**
- **필터 칩 바**: 가장 완성도 높은 칩 바 — 아이콘+텍스트 칩, 선택 상태 시각적 구분 명확
- **"Schools", "Neighborhoods" 지도 레이어 토글**: 하단 아이콘 버튼으로 레이어 전환 → 우리 데이터 레이어 토글(학군/교통/안전) 참고
- **"List | Map" 상단 토글 탭**: 깔끔한 전환 UI

**Homes에서:**
- **1열 수평 카드 레이아웃**: 우리도 1열이므로 카드 내 정보 밀도 참고 — 다만 Homes는 에이전트 중심이라 정보 구조는 다름
- **점 마커의 미니멀함**: 줌 아웃 시 점 마커 + 줌 인 시 라벨 전환을 우리 클러스터링에 참고

### 💡 신규 인사이트 (스펙에 없지만 고려할 것)

1. **지도-리스트 좌우 배치**
    - Zillow만 지도 좌측, 나머지 3개는 지도 우측
    - 우리 스펙에 좌우 배치 명시 없음 → **리스트 좌측 / 지도 우측 권장** (Realtor/Homes 패턴, 리스트 먼저 읽는 좌→우 읽기 방향)

    - 다만 추후 필터 추가 시 Realtor 패턴 참고

3. **지도 컨트롤 UI**

    - → 우리 데이터 레이어 토글(학군/교통/안전)은 **지도 내부 칩 바**(Zillow 스타일) 또는 **하단 아이콘**(Realtor 스타일) 검토
