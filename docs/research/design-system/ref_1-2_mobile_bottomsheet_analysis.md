> **분석일**: 2026-02-17

## 1. 서비스별 상세 분석

### 1-1. Google Maps — Peek 상태 (googlemaps_mobile_bottomsheet-peek.png)

**전체 레이아웃**

| 항목 | 분석 값 |
|------|--------|
| 지도 영역 | 화면의 약 **80-85%** — 거의 전체 화면이 지도 |
| 시트 높이 | 약 **120-140px** (화면 하단 ~15-20%) |
| 시트 노출 콘텐츠 | 드래그 핸들 + "음식점" 타이틀 + × 닫기 + 필터 칩 행 + 첫 번째 카드 잘림 |

**드래그 핸들**

| 항목 | 분석 값 |
|------|--------|
| 크기 | 약 **36-40px × 4px** |
| 색상 | 연한 회색 (#D1D5DB 추정) |
| 위치 | 시트 상단 중앙, 상단 패딩 ~8-12px |
| 라운딩 | 완전 둥근 (radius-full) |

**시트 상단 구조 (Peek에서 보이는 부분)**
1. **타이틀 행**: "음식점" (좌측, ~18px SemiBold) + × 닫기 (우측, ~24px)
2. **필터 칩 행**: 수평 스크롤
    - ⚙ 필터 아이콘 (독립 버튼)
    - "정렬 기준 ▾" / "지금 영업 중" / "최고 평점" / "리뷰 3개 이상" — pill 칩
    - 칩 스타일: 연한 회색 배경(#F3F4F6) + 검정 텍스트, ~32-36px 높이, ~8px 라운딩
3. **첫 번째 카드 상단 잘림**: 가게명만 살짝 보임 → 사용자에게 "올리면 더 있다" 힌트

**지도 영역**
- 마커: 빨간 원형 + 포크나이프 아이콘 + 흰 말풍선에 평점 숫자 ("5.0", "4.7")
- "이 지역 검색" 떠다니는 pill 버튼 (지도 상단 중앙)
- 내 위치 버튼: 우하단 흰 원형
- 검색바: 화면 최상단 고정 (시트와 독립)

**핵심 관찰: Peek 상태의 목적**
→ 사용자가 지도를 최대한 보면서, 필터만 빠르게 조작할 수 있는 상태. 리스트는 "있다"는 힌트만 줌.

### 1-2. Google Maps — Half 상태 (googlemaps_mobile_bottomsheet-half.png)

**전체 레이아웃**

| 항목 | 분석 값 |
|------|--------|
| 지도 영역 | 화면의 약 **40-45%** (상단) |
| 시트 높이 | 약 **55-60%** (화면 중간~하단) |
| 시트 노출 콘텐츠 | 타이틀 + 필터 칩 + 카드 약 2.5개 |

**시트 내부 구조**
1. **타이틀 행**: "음식점" + × (Peek과 동일, 고정)
2. **필터 칩 행**: ⚙ + "정렬 기준 ▾" + "지금 영업 중" + "최고 평점" + "리뷰 3개 이상" (고정, 스크롤되지 않음)
3. **카드 리스트** (스크롤 영역 시작):
    - **카드 1**: 가게명(~16px Bold) → 평점 ★ (리뷰수) · 카테고리 · 거리 → CTA 행(경로/통화/공유/저장)
    - **카드 2**: 동일 구조 + "영업 중" 녹색 라벨 + 영업 시간
    - **카드 3**: 상단 잘림 → 스크롤 힌트
    - 카드 간 구분: 얇은 수평 구분선 (1px, #E5E7EB)

**카드 상세 구조** ⭐

| 행 | 콘텐츠 | 스타일 |
|---|---------|--------|
| 1 | 가게명 "생고기랑찌개랑" | ~16px **SemiBold**, 검정 |
| 2 | 5.0 ★ (1) · 음식점 · 350.0 미터 | ~13px Regular, dot(·) 구분, 회색 |
| 3 | (일부) "영업 중" · 22:30에 영업 종료 · 280.0 미터 | 녹색 "영업 중" / 빨간 "영업 종료" + 회색 |
| 4 | 📍경로 📞통화 ↗공유 🔖저장 | 아이콘+텍스트 pill 버튼 4개, 연한 블루 배경(#E0F2FE) |

**CTA 버튼 행**

| 항목 | 분석 값 |
|------|--------|
| 버튼 수 | 4개 (경로/통화/공유/저장) |
| 스타일 | pill 형태, 연한 블루 배경 + 블루 아이콘 + 검정 텍스트 |
| 높이 | 약 36-40px |
| 간격 | 약 8-12px |
| 가로 배치 | 4개가 한 줄에 나란히, 좌측 정렬 |

**지도 영역 (축소됨)**
- 시트가 올라와도 지도 **여전히 터치 가능** (넌모달) ✅
- 마커가 보이고 패닝/줌 가능

### 1-3. Google Maps — Full/Expanded 상태 (googlemaps_mobile_bottomsheet-full.png)

**전체 레이아웃**

| 항목 | 분석 값 |
|------|--------|
| 지도 영역 | 약 **5-8%** (최상단에 살짝 보임) — 거의 시트가 전체 화면 |
| 시트 높이 | 약 **92-95%** |
| 시트 노출 콘텐츠 | 타이틀 + 필터 + 카드 전체 리스트 (자유 스크롤) |

**핵심 관찰**
- 드래그 핸들 여전히 보임 (상단)
- 타이틀 + 필터 칩 행은 **상단 고정 (sticky)** — 리스트 스크롤해도 유지
- 카드 리스트가 자유롭게 스크롤
- 지도는 거의 안 보이지만 완전히 사라지지는 않음 (상단 틈새에 약간 보임)
- 카드에 **이미지 썸네일 행**이 등장: 일부 카드에 수평 이미지 캐러셀 (3장)
- 리뷰 인용: 💬 "LA갈비 돼지갈비 너무 맛있어요~~" — 아바타 + 인용 텍스트

**Expanded에서의 카드 정보 밀도 증가**
- Half에서 안 보이던 이미지, 리뷰 인용이 추가로 노출
- → Progressive Disclosure: 시트가 커질수록 카드 정보도 풍부해짐

### 1-4. 카카오맵 (kakaomaps_mobile_search.png)

**전체 레이아웃** — Half~60% 상태로 추정

| 항목 | 분석 값 |
|------|--------|
| 지도 영역 | 약 **45-50%** (상단) |
| 시트 높이 | 약 **50-55%** |
| 검색바 | 지도 위 상단 고정 (시트와 독립) — "음식점" + 음성검색 + × |

**드래그 핸들**

| 항목 | 분석 값 |
|------|--------|
| 크기 | 약 **40×4px** (Google과 유사) |
| 색상 | 연한 회색 (#C4C4C4 추정) |
| 위치 | 시트 상단 중앙 |

**시트 내부 구조**
1. **필터 칩 행 (1행)**: 수평 스크롤
    - "지도중심 ▾" / "정확도순 ▾" / "3.5이상" / "영업중" / ⚙ 필터 아이콘
    - 칩 스타일: 흰 배경 + 회색 보더, ~36px 높이, pill 형태
2. **구분선** (1px)
3. **카드 리스트**:

**카드 상세 구조** ⭐ 가장 정보 풍부

| 행 | 콘텐츠 | 스타일 |
|---|---------|--------|
| 1 | "동흥관" + "중국요리" + 🛵배달 📦포장 배지들 | 가게명 ~16px SemiBold + 카테고리 회색 + 서비스 배지(파란 pill) |
| 2 | 2.8 ★★★☆☆ (129) · 영업 전 | ~13px, 평점 **숫자+별+리뷰수**, 영업 상태 |
| 3 | 1.3km · 서울 금천구 시흥동 ▾ | ~12px, 거리 + 주소, 회색 |
| 4 | 태그 행: 백년가게 / 노포 / 친목모임 / 코스요리 / 주차 / 제로페이 | 회색 텍스트 태그, 슬래시 없이 공백 구분 |
| 5 | **이미지 캐러셀**: 4장 수평 스크롤 | 약 120×90px 각, 라운딩 ~8px |
| 6 | **액션 아이콘 행**: 📞전화 / 📍길찾기 / ↗공유 / 🔖저장 / ↗ | 원형 아이콘 버튼 5개, 파란 아이콘(#3B82F6) |

**카카오맵 특유 요소**
- "이 지역 재검색" 떠다니는 pill (지도 상단 중앙, 🔄 아이콘) — Google과 동일 패턴
- **태그 시스템**: 가게 특징을 짧은 키워드 태그로 나열 → 우리 카드의 4점수 태그와 유사한 역할
- **평점 숫자 강조**: "2.8"이 가장 눈에 띄는 숫자 (빨간/주황 컬러) → 우리의 종합 점수와 유사
- 카드 간 구분: 구분선 없이 넉넉한 간격 (~20-24px)

### 1-5. 네이버지도 (navermaps_mobile_search.png)

**전체 레이아웃** — Half~55% 상태로 추정

| 항목 | 분석 값 |
|------|--------|
| 지도 영역 | 약 **45-50%** (상단) |
| 시트 높이 | 약 **50-55%** |
| 검색바 | 지도 위 상단 고정 — "음식점" + 음성검색 + × |

**드래그 핸들**: 보이지 않거나 매우 미세 — 시트 상단 경계가 자연스러움

**시트 내부 구조**
1. **필터 칩 행 (2행!)**: ⭐ 3개 서비스 중 유일하게 2행
    - 1행: ⚙ 필터 + "영업중" + "N 포장주문" + "N 실시간예약" + "% 쿠폰"
    - 2행: "P 주차" + "👥 단체석" + "🌿 신선한" + "🍽 특별한메뉴" + "💻 인터..."
    - 칩 스타일: 연한 회색 배경 + 아이콘+텍스트, ~32px 높이
    - 수평 스크롤 가능
2. **정렬 행**: "현재 지도 중심 ▾" + "관련도순 ▾" — 좌측 정렬, ~13px
3. **카드 리스트**:

**카드 상세 구조**

| 행 | 콘텐츠 | 스타일 |
|---|---------|--------|
| 1 | "인생파곱창" + Ⓝ예약 + 💬톡톡 + "곱창,막창,양" | 가게명 ~16px SemiBold 파란색(링크) + 서비스 배지 + 카테고리 회색 |
| 1-a | 우측 ☆ 찜 아이콘 | 회색 아웃라인 |
| 2 | "청정 대파곱창,대파대창 불맛" + 광고ⓘ | 빨간/오렌지 텍스트 — 광고 캐치프레이즈 |
| 3 | 오늘 휴무 · 리뷰 2,155 · Ⓣ 생방송투데이 | ~12px, dot(·) 구분 |
| 4 | **이미지 캐러셀**: 4장 수평 스크롤 | 약 120×100px 각, 라운딩 ~8px |
| 5 | **리뷰 인용 캐러셀**: 수평 스크롤 텍스트 카드 | "뒤장찌개는 괜찮아요 곱창은..." |

**네이버지도 특유 요소**
- **가게명 블루 링크**: 탭하면 네이버 플레이스로 이동
- **광고 표시**: "광고ⓘ" 명시 + 광고 캐치프레이즈 빨간 텍스트
- **2행 필터 칩**: 필터가 매우 많아서 2행 → 시트 상단 영역을 많이 차지
- **"현재 지도 중심"**: 정렬 기준을 지도 뷰포트와 연동 → 우리도 동일 패턴 필요
- **리뷰 텍스트 캐러셀**: 이미지 아래에 리뷰 텍스트 수평 스크롤
- "현 지도에서 검색" 떠다니는 pill (지도 상단)

**지도 마커**
- 빨간/핑크 원형 아이콘 마커 (카테고리 아이콘 포함)
- 가게명 텍스트 라벨 (마커 아래)
- 선택된 마커: 크기 확대 + 그림자 추정

## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장

**Google Maps에서 (바텀시트의 교과서):**

1. **3단 스냅 비율**: Peek ~15-20% / Half ~55-60% / Expanded ~92-95% → 우리 스펙(25% / 50% / 90%)과 거의 일치. Google이 검증한 비율이므로 우리 스펙 유지 타당
2. **Peek에서 필터 칩 노출**: Peek 상태에서도 필터 칩이 보여서 정렬 변경 가능 → 우리도 Peek에서 "종합점수순/예산순/통근순" 칩 노출 필수
3. **첫 카드 peek 힌트**: Peek에서 첫 카드 상단이 살짝 보여 "올리면 더 있다" 인지 → 우리도 동일 패턴
4. **타이틀 + × 닫기 행**: "음식점" + × 구조 → 우리 "{N}건의 분석 결과" + × (닫으면 지도만)
5. **필터/타이틀 sticky**: Expanded에서 스크롤해도 타이틀+필터가 상단 고정 → 구현 필수
6. **Expanded에서 지도 살짝 보임**: 완전히 사라지지 않고 상단 5%에 지도 틈새 → 사용자가 "아직 지도 위"임을 인지

**카카오맵에서:**

7. **"이 지역 재검색" 떠다니는 pill**: 지도를 이동하면 나타나는 새로고침 버튼 → 우리도 지도 패닝 시 "이 지역 다시 분석" pill 표시 (단, "분석" 워딩 사용 — "검색" ❌)
8. **필터 칩 스타일**: 흰 배경 + 회색 보더 pill → 우리 칩 스타일과 가장 유사 (깔끔, 미니멀)
9. **태그 시스템**: "백년가게 / 노포 / 친목모임" 같은 키워드 태그 → 우리 카드의 4점수 요약을 태그형으로 표시할 때 참고
10. **카드 간 넉넉한 간격**: 구분선 없이 ~20-24px 간격 → 가독성 좋음. 우리 gap-3(12px)보다 넉넉하지만, 우리 카드는 정보 밀도가 높으므로 12px 유지

**네이버지도에서:**

11. **"현재 지도 중심" 정렬**: 리스트를 지도 뷰포트 기준으로 필터링 → 우리도 지도 이동 시 리스트 자동 업데이트 (이미 스펙에 있음)
12. **가게명 링크(블루)**: 탭하면 상세로 이동 → 우리 단지명도 탭하면 `/detail/[id]`로 이동, 블루(brand-500) 처리 고려

### ⚠️ 무시할 것

| 레퍼런스 패턴 | 우리와 다른 이유 | 우리 스펙 |
|-------------|---------------|---------|
| **필터 2행** (네이버) | 네이버는 필터가 10개+, 우리는 정렬 3개 | **1행이면 충분** |
| **CTA 4-5개** (Google 경로/통화/공유/저장, 카카오 5개) | 음식점 = 즉시 액션 필요, 우리는 비교/분석 중심 | **"비교 추가" 1개만** |
| **이미지 캐러셀** (카카오, 네이버) | 매물 사진 미제공 | **이미지 없는 데이터 카드** |
| **광고 표시** (네이버 "광고ⓘ") | 우리는 광고 미적용 | **없음** |
| **음식점 마커 스타일** (빨간 원형 + 포크나이프) | 우리는 점수 라벨 핀 | **등급 컬러 마커** |
| **드래그 핸들 미표시** (네이버) | 핸들 없으면 시트 발견성↓ | **항상 표시** |

### 💡 신규 인사이트

1. **Peek 상태에서 결과 수 표시**
    - Google: Peek에서 "음식점" 타이틀만 → 결과 수 미표시
    - → 우리는 Peek에서 **"{N}건"** 결과 수를 함께 표시하면 사용자가 올리기 전에 결과 규모 인지 가능
    - 제안: 드래그 핸들 아래 "서울시 강남구 분석 결과 47건" + × 닫기

2. **Half → Expanded 전환 트리거**
    - Google: 시트 내부 리스트를 위로 스크롤하면 자동으로 Expanded로 전환
    - → Vaul(shadcn Drawer) 기본 동작과 일치. 별도 구현 불필요할 수 있음
    - 반대로, Expanded에서 리스트 최상단까지 스크롤한 뒤 더 내리면 Half로 복귀

3. **시트 배경의 라운딩**
    - Google: 시트 상단 좌우 코너 ~16px 라운딩 (Peek/Half에서 둥글고, Expanded에서는 덜 둥글어짐)
    - 카카오: 유사하게 ~12-16px
    - → 우리: radius-xl(16px) 적용 — Peek/Half에서만, Expanded에서는 0으로 전환하거나 유지

4. **검색바와 시트의 분리**
    - 3개 모두: 검색바는 **지도 위 상단 고정** (시트와 독립적으로 존재)
    - → 우리도 검색/필터를 시트 안에 넣지 않고, 지도 위 상단에 고정 배치
    - 시트 내에는 정렬 칩만 배치 (검색 조건 변경은 상단에서)

5. **"이 지역 재검색" 패턴의 보편성**
    - 3개 모두 동일 패턴 사용 → 사실상 업계 표준
    - 위치: 지도 상단 중앙, 떠다니는 pill
    - → 우리: "이 지역 다시 분석" (🔄 아이콘 + 텍스트, pill, 지도 상단 중앙)
    - 주의: "검색"이 아닌 "분석"으로 워딩 (컴플라이언스)

## 4. Agent 전달용 요약

### 이 레퍼런스에서 가져올 핵심 3가지

1. **Google Maps 3단 스냅 비율과 Peek 구조** — Peek에서 핸들+타이틀+필터칩+카드힌트까지 보여주는 구조. 필터/타이틀은 모든 상태에서 sticky. 우리 Peek도 동일하게 "결과 N건" + 정렬 칩 + 첫 카드 peek
2. **카카오맵 필터 칩 스타일** — 흰 배경 + 회색 보더 pill이 가장 미니멀하고 우리 디자인 톤과 일치. 네이버의 2행 필터는 과하므로 무시
3. **3개 서비스 공통: 넌모달 + "이 지역 재검색" pill** — 시트가 올라와도 지도 터치 가능, 지도 이동 시 떠다니는 pill 표시는 업계 표준. 반드시 구현

### 반드시 무시할 3가지

1. **CTA 버튼 4-5개** — 음식점용 (경로/통화/공유/저장), 우리는 "비교 추가" 1개
2. **이미지 캐러셀** — 가게 사진, 우리 카드에 이미지 없음
3. **필터 2행** (네이버) — 우리는 정렬 3개 칩이면 1행 충분

### 구현 핵심 체크리스트

```
바텀시트 구현 체크리스트:
□ 3단 스냅: Peek(~25%) / Half(~50%) / Expanded(~90%)
□ 드래그 핸들: 40×4px, #A8A29E, rounded-full, 상단 8px 패딩
□ 넌모달: 시트 위에서 지도 터치/패닝/줌 가능
□ Peek 콘텐츠: "결과 N건" + × + 정렬 칩 + 첫 카드 peek
□ 필터/타이틀 sticky: Expanded에서 스크롤해도 고정
□ 시트 라운딩: radius-xl(16px) 상단만
□ 스냅 전환: spring(stiffness:400, damping:30)
□ "이 지역 다시 분석" pill: 지도 이동 시 표시, 상단 중앙
□ Half→Expanded: 리스트 스크롤 업으로 자동 전환
□ 검색바: 시트 외부, 지도 위 상단 고정
```
