| 필드 라벨 | "연소득" — 블루(#3B82F6 추정) ~13px, 입력 필드 위 |
| 입력 플레이스홀더 | "부부합산 금액" — 회색 ~16px, 밑줄(underline) 스타일 |

| 항목 | 분석 값 |

**숫자 키패드**

| 항목 | 분석 값 |

| 높이 | 화면의 약 35-40% |

**핵심 패턴**: **한 화면에 하나의 질문 + 하나의 입력** (One Question, One Input) → 인지 부하 최소화

### 1-5. 토스 — 로딩 + 결과 (영상 프레임 추출)

**토스 대출 비교 로딩 패턴** ⭐ Progressive Loading

| 단계 | 화면 | 시간 |
|------|------|------|
| 시작 | "80개 중 22개 금융사에 다녀왔어요" + **2%** 원형 게이지 | 0s |
| 진행 중 | "80개 중 44개 금융사에 다녀왔어요" + **15%** + 결과 목록 증가 | ~10s |
| 계속 | **31%** + 은행 목록 계속 추가 | ~20s |
| 완료 | 100% + 전체 결과 리스트 | ~60s |

**로딩 UI 상세**

| 항목 | 분석 값 |
|------|--------|
| 상단 텍스트 | "80개 중 {N}개 금융사에 다녀왔어요" — ~16px, 카운트업 |
| **원형 게이지** | ~80px, 블루, 중앙에 **N%** 퍼센트 표시 |
| 게이지 아래 | 일러스트 아이콘 (집+캐릭터) |
| 탭 | "금리 낮은 순" / "한도 높은 순" — 2탭 전환 |
| **결과 실시간 추가** | 은행이 하나씩 추가됨: 로고 + 은행명 + 상품명 + **금리% + 한도** |
| 결과 카드 구조 | 좌: 은행로고(32px) + 은행명 + 상품명(회색) / 우: 금리(**Bold**) + 한도 |

**핵심 패턴**:
- **진행률(%)을 실시간 표시** → 사용자가 "지금 얼마나 됐는지" 인지
- **결과가 하나씩 나타남** → 기다림이 아니라 "발견"의 경험
- 60초 장시간 로딩이지만 **지루하지 않음** (계속 변화가 있으므로)

## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장

**토스 대출 입력에서:**

1. **원 퀘스천, 원 인풋 패턴**: 한 화면에 질문 하나 + 입력 하나 + 넓은 여백 → 인지 부하 최소화. 우리 5스텝도 각 스텝에서 입력 항목을 최소화해야 함
2. **타이틀 톤 "~알려주세요" / "~확인해주세요"**: 존댓말 요청형 → 우리 위저드 타이틀도 동일 톤 적용 ("부부의 월 소득을 알려주세요", "출퇴근 직장 주소를 입력해주세요")
3. **금액 입력 즉시 "만원" 변환**: "65,613,579원" → "6,561만 3,579" 실시간 표시 → 우리 자산/소득 입력 시 반드시 적용. "35000" 입력 → "3,500만원" 또는 "3억 5,000만" 변환
4. **필드 라벨 블루 + 밑줄 블루**: 활성 필드를 색상으로 명확히 표시 → 우리는 brand-500(#0891B2) 사용
5. **"소득 없어요" / "결혼 안했어요" 예외 버튼**: 입력 불가능한 경우의 이탈 방지 → 우리도 각 스텝에 예외 처리 필요 ("현재 소득이 없어요", "통근하지 않아요" 등)

**토스 송금에서:**

6. **커스텀 키패드 with "00" 키**: 만 단위 금액 입력이 빠름 → 우리 빠른 금액 버튼(+1,000만/+5,000만/+1억) 외에도 "00" 키 포함 커스텀 키패드 고려
7. **큰 금액 표시 (36-40px)**: 금액을 크게 보여줘서 실수 방지 → 우리도 입력된 자산/소득을 크게 표시

**토스 동의 수집에서:**

8. **"꼭 필요한 동의만 추렸어요" 안심 프레이밍**: 동의 거부감 최소화 → 우리도 "분석에 꼭 필요한 정보만 수집해요" 유사 표현
9. **결과 프리뷰 (예시화면)**: 동의 전에 "이런 결과를 받을 수 있다" 보여줌 → 동의 동기 부여. 우리도 동의 화면에 분석 결과 카드 예시(흐린 처리)를 넣으면 효과적
10. **"필수" 라벨 + 셰브론 접기/펼치기**: 간결하면서도 상세 확인 가능 → 우리 동의 UI에 직접 적용

**토스 로딩 영상에서:**

11. **진행률 실시간 표시**: "22개 금융사에 다녀왔어요" + % 게이지 → 사용자가 진행 상황 인지. 우리 4단계("입력분석→통근계산→예산시뮬→단지탐색") 각 단계 완료 시 텍스트 업데이트
12. **결과가 하나씩 나타나는 패턴**: 60초 로딩인데도 지루하지 않음 → 우리는 7초라 이 정도까진 불필요하지만, 단계 전환 시 체크마크(✓) 애니메이션은 적용할 만함

**카카오 로딩에서:**

13. **순차 체크리스트**: "주택시세정보 → 소득정보 → 주택보유수 → 한도와 금리" 하나씩 처리 → 우리 4단계와 가장 유사한 패턴. 각 항목이 처리되면서 텍스트가 등장하는 방식

**카카오 결과에서:**

### 💡 신규 인사이트

4. **Trust 배지 위치 — 스펙 보완 필요**

## 4. Agent 전달용 요약

### 이 레퍼런스에서 가져올 핵심 3가지

1. **토스 "원 퀘스천, 원 인풋" 패턴** — 한 화면에 질문 하나 + 입력 하나 + 넓은 여백. 타이틀은 "~알려주세요"/"~확인해주세요" 톤. 금액 입력 시 "만원" 단위 실시간 변환 표시
2. **토스 동의 수집 구조** — "필수" 블루 라벨 + 셰브론 접기/펼치기 + 결과 프리뷰 조합. "꼭 필요한 동의만 추렸어요" 안심 프레이밍
3. **카카오 순차 체크리스트 로딩** — 4단계 항목이 하나씩 처리되는 패턴이 우리 로딩 스펙(🏘️입력분석→🚇통근계산→📊예산시뮬→🗺️단지탐색)과 가장 유사

### 반드시 무시할 3가지

1. **챗봇 대화형** — 카카오 패턴, 우리는 위저드
2. **60초 장시간 로딩** — 토스 대출 비교, 우리는 7초
3. **시스템 키패드** — 커스텀 키패드 + 빠른 금액 버튼이 우리에게 더 적합

### 구현 핵심 체크리스트

```

□ 금액 입력: 커스텀 키패드("00" 포함) + 빠른 금액(+1,000만/+5,000만/+1억)
□ 단위 변환: 입력 즉시 "3억 2,000만" 자동 표시
□ 필드 라벨: brand-500(#0891B2) + 밑줄 활성 표시
□ CTA: "다음" full-width brand-500, 키패드 위 고정
□ 예외 분기: "결혼 안했어요" / "재택근무예요" 등 예외 버튼
□ 프로그레스: 미니멀 dot/bar 인디케이터 (상단)
□ 동의: "필수" 라벨 + 셰브론 + 결과 프리뷰(흐린 처리)
□ Trust: "분석에 꼭 필요한 정보만 수집해요" + 🔒 배지
□ 로딩: 4단계 순차 (이모지+텍스트), ~7초, fade 전환
□ 로딩 완료: /results로 라우팅
```
