### 1-1. 토스 — 신용점수 메인 (toss_mobile_credit-score.png)

**원형 게이지** ⭐ 우리 Hero 게이지의 핵심 레퍼런스

| 항목 | 분석 값 |
|------|--------|
| 전체 크기 | 약 **140-160px** 지름 (화면 너비 390px 기준), 2개 나란히 배치 |
| 배치 | **좌우 2개** — KCB(좌), NICE(우), 약간 겹침 (오버랩 연출) |
| 스트로크 두께 | 약 **6-8px** (Hero급에서 충분한 시각적 존재감) |
| 트랙 색상 | 매우 연한 회색 (#F0F0F0 추정) — 거의 배경에 녹아듦 |
| 필 색상 | **블루 그라데이션** — 밝은 하늘색(#60BFFF) → 진한 블루(#3B82F6) |
| 게이지 각도 | 약 270-300° (원의 약 80-85% 채움) — 완전한 원은 아님, 하단에 갭 |
| 게이지 끝 처리 | **round linecap** (둥근 끝) |
| 배경 | 순백~크림 배경, 게이지 자체에 그림자 없음 |

**중앙 텍스트**

| 항목 | 분석 값 |
|------|--------|
| 상단 라벨 | "KCB ›" / "NICE ›" — 약 12-13px, 회색, 화살표(›)로 탭 유도 |
| **점수 숫자** | **"989점" / "945점"** — 약 **32-36px Bold**, 검정 |
| "점" 단위 | 숫자와 같은 줄, 숫자보다 약간 작은 크기 (~24px) |
| 폰트 | tabular-nums 적용 추정 (숫자 고정폭) |

**게이지 아래 요소**

| 항목 | 분석 값 |
|------|--------|
| CTA 버튼 | "버튼 눌러서 점수 올리기" — pill 형태, 약 14px, 회색 텍스트, 배경 없는 아웃라인 |
| 요약 카드 | 둥근 모서리(~16px) 회색 배경 카드 안에 3줄 요약 |
| 요약 항목 | ● 불릿 + 좌측 라벨 + **우측 수치(Bold)**, 구분선으로 분리 |
| 예시 | "● 오늘의 예상 대출  4.32% | 1억원 ↑" |
| 긍정 원인 | "● 긍정 원인 3개" → 우측에 "3개" pill 배지 |
| 카드한도 | "● 카드한도 1% 사용 중" → 우측에 "1%" pill 배지 |
| pill 배지 | 약 28×24px, 연한 회색 배경, 검정 텍스트, ~12px |

**전체 레이아웃 특징**
- 배경: 매우 밝은 그레이 (#F7F7F7 추정) — 순백이 아님
- 카드형 섹션: 여러 개의 둥근 흰 카드가 수직으로 나열
- 하단 CTA: "다음" 파란 버튼 (full-width, rounded, 하단 고정 아님 — 스크롤 내 배치)
- 네비게이션 관련: 아이콘+라벨 리스트 (내 사업자 신용점수, 신용성향 분석 등 → 셰브론(›))

### 1-2. 토스 — 요인 분해 (toss_mobile_credit-factor.png)

**페이지 구조**

| 항목 | 분석 값 |
|------|--------|
| 타이틀 | **"989점인 이유예요"** — ~24-26px **Bold**, 검정 |
| 부제 | "점수에 영향을 준 원인을 3개 찾았어요." — ~15px Regular, 회색 |
| 섹션 라벨 | "📊 긍정 영향 3개" — 블루 컬러(#3B82F6 추정) + 아이콘, ~14px SemiBold |

**요인 리스트** ⭐ Progressive Disclosure 패턴

| 항목 | 분석 값 |
|------|--------|
| 컨테이너 | 둥근 카드(~16px radius), 연한 블루-그레이 배경 (#F0F4FF 추정) |
| 항목 구조 | 좌측 아이콘(~32px, 컬러 일러스트) + 타이틀(~15px SemiBold) + 부연(~13px 회색) |
| 항목 간 간격 | 약 20-24px, 구분선 없음 |
| 아이콘 스타일 | 둥근 사각형 배경 + 컬러 일러스트 아이콘 (💰 동전더미, 💳 카드 등) |
| 긍정 요인 | 3개: "우량대출을 가지고 있어요" / "신용카드 일시불을 꾸준히 사용했어요" / "최근 카드 사용이 무리 없이 적당했어요" |

**톤 앤 매너** ⭐ 우리 서비스와 직접 연결
- **~해요체** 사용 — "가지고 있어요", "사용했어요", "적당했어요"
- **긍정 프레이밍** — 부정적 요인도 긍정적으로 표현
- **부연 설명**: 본문 아래 1줄 회색 → 우리의 ⓘ 1줄 요약과 동일 패턴

**하단 CTA**
- "다음" 버튼 — **full-width**, 블루(#3B82F6) 채움, 흰 텍스트, ~48px 높이, ~12px radius
- **하단 고정 아님** — 콘텐츠 아래 자연 배치 (safe-area 패딩 있음)

|------|--------|
| 소형 배지 | 좌측에 "Walk Score 88" 쉴드 소형(~40px) + 우측 "New York is Very Walkable" 타이틀 |
| 부연 설명 | "Most errands can be accomplished on foot." ~14px 회색 |

- 카테고리별 3개 나란히 표시 → 우리 카드 내 4점수 2×2 그리드와 유사한 개념

## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장

**토스 신용점수에서:**

1. **게이지 스트로크 비율**: 150px 게이지에 6-8px 스트로크 → 비율 약 4-5%. 우리 Hero(80-96px)에 6px = 약 6-7% → 토스보다 약간 굵지만 작은 크기에서 시인성을 위해 적절
2. **게이지 각도 270-300°**: 하단에 갭을 둬서 "얼마나 찼는지" 직관적 → 우리도 270° (3/4 원) 채택 확인
3. **round linecap**: 게이지 끝이 둥글어서 부드러운 느낌 → 우리 SVG 구현 시 `stroke-linecap="round"` 필수
4. **트랙 색상의 은은함**: 트랙이 거의 배경에 녹아드는 수준 (#F0F0F0) → 우리 `brand-100(#A5F3FC)`은 이보다 좀 더 존재감 있는데, 틸 계열이라 브랜드 아이덴티티 유지 목적이므로 OK
5. **요인 분해 구조**: 아이콘(32px) + 타이틀(SemiBold) + 부연(회색 Regular) → 우리 4점수 상세 공개에 직접 적용 가능한 패턴

**토스 요인 분해 화면에서:**

6. **"~인 이유예요" 타이틀 패턴**: "989점인 이유예요" → 우리 "78점인 이유예요" 또는 "A등급인 이유" — 사용자가 "왜 이 점수인지" 바로 이해
7. **긍정 프레이밍 톤**: "우량대출을 가지고 있어요" → 우리도 "통근 거리가 30분 이내예요", "보육 시설이 충분해요" 식의 긍정 서술
8. **연한 컬러 카드 컨테이너**: 요인 목록을 연한 배경 카드로 감쌈 → 우리 score-*-subtle 토큰 활용 (예: #E3F2FD)

**Niche에서:**

9. **인라인 등급 배지**: "(A+) Overall Niche Grade" 패턴 — 텍스트 안에 작은 원형 배지 삽입 → 우리 카드 내 4점수 행에서 등급 아이콘 인라인 배치에 직접 참고
10. **등급 필터 원형 버튼 UI**: A/B/C/D 원형 토글 → 향후 필터 기능 추가 시 레퍼런스
11. **ⓘ 아이콘 위치**: 카테고리명 바로 옆 → 우리 ⓘ progressive disclosure 트리거 위치

**Walk Score에서:**

12. **카테고리별 나란히 표시**: Walk/Transit/Bike 3개를 수평 배치 → 우리 4점수(예산·통근·보육·안전) 배치에 참고. 다만 우리는 **2×2 그리드** (수평 4개는 모바일에서 비좁음)
13. **점수+1줄 해석 조합**: "88" + "New York is Very Walkable" → 숫자만으로는 의미 전달 부족, 1줄 해석 필수 → 우리 등급 라벨(A+~D)이 이 역할

### 💡 신규 인사이트

1. **게이지 필 색상 — 그라데이션 vs 단색**
    - 토스: 블루 그라데이션 (밝은→진한) → 고급스러운 느낌
    - 우리 스펙: 등급 컬러 단색 (예: A+ = #1565C0 단색)
    - → 제안: **단색 유지하되, 게이지 끝부분에 미세한 밝기 변화**를 줄 수 있음 (SVG linearGradient로 같은 hue의 밝은→진한). 다만 5등급 컬러가 이미 다양하므로 단색이 더 깔끔할 수 있음 → **Agent 판단에 맡기되 기본은 단색**

2. **게이지 아래 등급 라벨의 시각적 무게**
    - 토스: 등급 라벨 없음 (숫자가 충분히 의미 전달)
    - Niche: 등급 배지가 핵심 시각 요소 (원형 채움)
    - Walk Score: 숫자가 핵심, 텍스트 해석이 보조
    - → 우리: 숫자 + 등급 라벨 **둘 다** 표시 (WCAG 1.4.1) → **등급 라벨을 어떤 크기/위치로?**
    - 제안: 게이지 **아래** "A+" 텍스트, ~12px SemiBold, 등급 컬러 → 숫자보다 작지만 색상으로 시선 유도

3. **요인 분해의 긍정/부정 분류**
    - 토스: "긍정 영향 3개" 섹션으로 분류 (부정은 이 캡처에 없지만 별도 섹션 추정)
    - → 우리 4점수에도 적용 가능: "강점 2개 · 보완점 2개" 식으로 긍정/부정 그룹핑
    - 다만 **안전 카테고리에서 "부정"이라는 표현은 컴플라이언스 이슈** → "충분" / "보통" / "확인 필요" 3단계 표현 권장

4. **카운트업 애니메이션 duration**
    - 토스: 체감상 약 800-1200ms (빠르게 올라가다 마지막에 감속)
    - 우리 스펙: 800ms easeOutExpo → 토스보다 약간 짧지만, Hero 표시 시 적절
    - → 카드(64px) 미니(36px)에서는 카운트업 **생략 또는 300ms로 축소** 권장 (작은 게이지에서 긴 애니메이션은 산만)

5. **SVG 구현 시 참고: 토스 게이지의 갭**
    - 토스는 하단에 약 60-90° 갭 (6시~7시, 5시 방향)
    - 우리 270° 아크 = 하단 90° 갭 → 토스와 거의 동일
    - SVG 구현: `stroke-dasharray` + `stroke-dashoffset` + `transform: rotate(-135deg)` (12시 방향 시작이 아닌 7시 30분 방향 시작)

## 4. Agent 전달용 요약

### 이 레퍼런스에서 가져올 핵심 3가지

1. **토스 게이지의 스트로크 비율과 round linecap** — 6-8px 스트로크 + round 끝 처리 + 270° 아크가 만드는 부드러운 느낌. SVG `stroke-linecap="round"`, `stroke-dasharray`로 구현
2. **토스 요인 분해의 아이콘+타이틀+부연 구조** — 32px 아이콘 + SemiBold 타이틀 + Regular 부연 1줄, 연한 배경 카드로 감싼 패턴 → 우리 4점수 상세 공개에 직접 적용
3. **Niche 인라인 등급 배지** — 텍스트 행 안에 작은 원형 배지(24-28px)를 삽입하는 패턴 → 우리 카드 내 4점수 표시(미니 게이지 36-48px)에 참고

### 반드시 무시할 3가지

1. **색상 체계** — 토스 블루 단일, Niche 그린 단일 모두 무시 → 우리 5등급 블루-오렌지 다이버징 (#1565C0 → #D84315) 사용
2. **적녹색** — Niche의 그린 등급 배지 색상 절대 따라가지 않기 (WCAG 접근성)
3. **게이지 형태** — Walk Score 쉴드 모양 무시 → 표준 원형 유지

### SVG 구현 가이드 (Agent 참고)

```
게이지 SVG 구현 핵심:
- viewBox: "0 0 {size} {size}" (size = 64/80/96)
- 원 중심: (size/2, size/2)
- 반지름: (size - strokeWidth) / 2
- stroke-linecap: "round"
- 아크 270°: stroke-dasharray = circumference * 0.75
- 시작점: transform rotate(-135deg) from center
- 카운트업: stroke-dashoffset를 CSS transition(800ms easeOutExpo)으로 애니메이션
- 필 색상: score-grade별 CSS custom property 동적 바인딩
```
