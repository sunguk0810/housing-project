## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장

1. **"한눈에 보기" 섹션 최상단 배치**: Apple이 핵심 스펙을 큰 숫자로 먼저 보여주는 패턴 → 우리도 종합등급 + 가격 + 통근시간을 **큰 수치(~28px Bold)**로 3컬럼 나란히 → 차이가 즉시 보임

2. **값을 크게, 라벨을 작게**: "34.5cm" (크게) + "Retina 디스플레이" (작게 회색) → 우리도 "A+ (78점)" (크게) + "종합 등급" (작게 회색). 숫자 차이가 먼저 눈에 들어오는 구조

3. **섹션 타이틀 + 여백으로 구분**: 구분선 없이 섹션 타이틀(~20px Bold) + 넉넉한 여백(48-64px)으로 영역 구분 → 깔끔함. 우리도 "예산 상세" / "통근 상세" 섹션을 동일 방식으로

4. **3컬럼 상단 정렬**: 텍스트 길이가 달라도 같은 행에서 시작 → CSS `align-items: start` 또는 Tailwind `items-start`. 비교 가독성의 핵심

5. **제품 이미지 → 우리는 게이지**: Apple은 제품 사진으로 시각적 차별화, 우리는 **원형 게이지(등급 컬러)**로 차별화. 게이지 색상(블루~오렌지)이 3개 나란히 있으면 등급 차이가 직관적

6. **CTA 버튼 위치**: 제품 헤더 바로 아래에 "구입하기" → 우리는 **"상세 보기"** (각 단지 아래) + **"이 단지 선택하기"** (하단 고정 or 헤더)

7. **단일 페이지 긴 스크롤**: Apple은 탭 없이 스크롤 → 우리도 비교 페이지는 **탭 없이 스크롤**이 적합 (탭 전환하면 비교가 끊김)

### ⚠️ 무시할 것

| 레퍼런스 패턴 | 우리와 다른 이유 | 우리 스펙 |
|-------------|---------------|---------|
| **제품 사진** | 우리는 단지 이미지 없음 | 원형 게이지 + 등급 라벨 |
| **차이 비강조** (Apple은 다른 값도 동일 크기) | Apple은 자사 제품끼리 비교라 차이 강조 안 함, 우리는 단지 간 차이를 부각해야 함 | **차이나는 셀에 subtle 배경색** |
| **sticky 헤더 없음** | Apple은 3개만 비교 + 이미지로 인지, 우리는 단지명이 텍스트라 스크롤 시 혼동 가능 | **단지명 sticky 헤더 권장** |
| **"구입하기" CTA** | 우리는 구매 아님 | "상세 보기" / "보드에 저장" |

### 💡 신규 인사이트

1. **모바일 비교 처리**
    - Apple 데스크톱: 3컬럼 나란히
    - Apple 모바일: 2개씩 비교 or 좌우 스와이프 (별도 캡처 필요했으나 추정)
    - → 우리 모바일: **2컬럼** (2개 비교가 기본, 3개는 좌우 스크롤)
    - 또는: **카드 스택형** (위아래로 쌓고 같은 항목끼리 묶어 비교)
    - 제안: **모바일 2컬럼 + 3번째는 좌우 스크롤**이 가장 자연스러움

2. **차이 하이라이트 추가**
    - Apple은 안 하지만, 우리는 해야 함 (단지 간 차이가 핵심 가치)
    - 제안: 값이 다른 행에서 **가장 좋은 값에 brand-500 컬러 또는 subtle 배경** 적용
    - 예: 통근 28분(최소) → brand-500 텍스트 or score-excellent-subtle 배경

3. **레이더 차트 추가**
    - Apple에는 없지만, 우리 design_brief에 레이더 차트 비교가 있음
    - 비교 페이지 최상단에 **레이더 차트 오버레이** (2-3개 단지 겹쳐 표시) → 전체적인 프로필 차이를 한눈에
    - 레이더 축: 예산 / 통근 / 보육 / 안전 (4축)
    - Recharts RadarChart로 구현

4. **비교 대상 추가/제거 UX**
    - Apple: 고정 3개 (변경하려면 새 페이지)
    - 우리: **동적 추가/제거** 필요 → 빈 컬럼에 "+ 단지 추가" 점선 카드
    - 최소 2개, 최대 3개 (모바일 고려)

5. **sticky 단지 헤더**
    - Apple은 sticky 없음 (제품 이미지가 강력해서 기억됨)
    - 우리는 단지명이 텍스트라 스크롤하면 "어느 컬럼이 어느 단지인지" 혼동
    - → **단지명 + 미니 게이지(36px) sticky 헤더** 필수

## 4. Agent 전달용 요약

### 이 레퍼런스에서 가져올 핵심 3가지

1. **"한눈에 보기" 큰 수치 비교 패턴** — 핵심 스펙을 ~28px Bold로 3컬럼 나란히 배치, 라벨은 작게 아래. 값 차이가 즉시 눈에 들어오는 구조. 우리 "종합등급 + 가격 + 통근시간"에 직접 적용
2. **섹션 타이틀 + 여백 구분** — 구분선 없이 섹션 타이틀(~20px Bold) + 넉넉한 여백으로 영역 분리. 깔끔하고 스캔하기 좋음
3. **3컬럼 상단 정렬 + 단일 스크롤** — 탭 없이 긴 스크롤로 모든 스펙을 비교. 컬럼 내용 길이가 달라도 `items-start`로 같은 행 시작

### 반드시 무시할 3가지

1. **차이 비강조** — Apple은 자사 제품 비교라 안 하지만, 우리는 차이 값 하이라이트 필수
2. **sticky 헤더 없음** — 우리는 단지명+미니 게이지 sticky 헤더 필요
3. **제품 사진** — 우리는 이미지 없음, 원형 게이지로 대체

### Apple에 없지만 우리에게 필요한 것

1. **레이더 차트 오버레이** — 2-3개 단지의 4축(예산/통근/보육/안전) 프로필을 겹쳐 표시
2. **차이 하이라이트** — 가장 좋은 값에 subtle 배경색 또는 brand-500 텍스트
3. **동적 추가/제거** — "+ 단지 추가" 점선 카드, 최대 3개
4. **sticky 단지 헤더** — 단지명 + 미니 게이지(36px) 스크롤 시 고정
5. **모바일 2컬럼** — 3번째 단지는 좌우 스크롤
