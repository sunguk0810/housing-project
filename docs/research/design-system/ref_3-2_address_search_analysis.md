> **목적**: 집콕신혼 `/onboarding` Step 2(통근지 입력) 주소 검색 구현 시 AI Agent에게 전달할 레퍼런스

## 1. 서비스별 상세 분석

### 1-1. 토스 — 주소 검색 (배송지 입력)

**플로우 (6단계)**

| 단계 | 화면 | 설명 |
|------|------|------|
| 1 | 전체 폼 화면 | "기타 주소" 타이틀 + 주소/상세주소/주소별명/받는분/휴대폰/배송요청 필드 목록 |
| 2 | 주소 필드 포커스 | 필드 탭 → 별도 검색 화면으로 전환 (전체 화면 점유) |
| 3 | 빈 검색 상태 | "주소" 라벨(블루) + 입력 필드 "건물, 지번 또는 도로명 검색" + 키보드 |
| 4 | 입력 중 (결과 없음) | "가산디지털" 입력 → "검색 결과가 없어요." 빈 상태 메시지 |
| 5 | 자동완성 결과 | "가산디지털2로" 입력 → 결과 리스트 표시 |
| 6 | 선택 후 상세 | 주소 선택 → "자세한 주소를 알려주세요" + 상세주소 입력 |

**검색 화면 상세** (단계 3-5) ⭐

| 항목 | 분석 값 |
|------|--------|
| 네비게이션 | ← 뒤로 (좌상단) |
| 필드 라벨 | **"주소"** — 블루(~13px), 입력 필드 위 |
| 플레이스홀더 | "건물, 지번 또는 도로명 검색" — 회색, ~14px |
| 입력 스타일 | 밑줄(underline) 스타일, 블루 활성 라인 |
| 클리어 버튼 | ⊘ 회색 원형, 입력 필드 우측 끝 |
| CTA | "확인" — full-width 블루 버튼, 키보드 위 고정 |
| 키보드 | 시스템 한글 키보드 → 숫자 입력 시 숫자 키보드로 전환 |
| 빈 상태 | "검색 결과가 없어요." — 중앙 정렬, ~14px 회색 |

**자동완성 결과 리스트** (단계 5) ⭐⭐

| 항목 | 분석 값 |
|------|--------|
| 결과 구조 | **행 반복**: 우편번호 + 도로명주소 + 지번주소 |
| 행 1줄 | "우편번호" — ~12px 회색 + "08506" |
| 행 2줄 | **도로명주소** — ~14px **SemiBold** 검정, "서울특별시 금천구 가산디지털2로 128 (가산동)" |
| 행 3줄 | "지번주소" — ~12px 회색, "서울특별시 금천구 가산동 547-3" |
| 행 간 구분 | 없음 (여백으로 구분, ~16px) |
| 결과 수 | 한 화면에 약 3개 표시 |
| 스크롤 | 리스트 스크롤 가능 |
| **검색어 하이라이트** | 없음 (입력 텍스트와 일치하는 부분 강조 안 함) |

**상세주소 입력 화면** (단계 6)

| 항목 | 분석 값 |
|------|--------|
| 타이틀 | **"자세한 주소를 알려주세요"** — ~22px Bold, ~세요체 |
| 선택된 주소 | "서울특별시 금천구 가산디지털2로 179 (가산동)" — ~14px, 읽기 전용 |
| 상세주소 필드 | "상세주소 (예시: 토스빌딩 7층 / A동 201호)" — 플레이스홀더, 밑줄 스타일 |
| CTA | "확인" — full-width 블루, 키보드 위 |

### 1-2. 카카오맵 — 장소/주소 검색

**플로우 (7단계)**

| 단계 | 화면 | 설명 |
|------|------|------|
| 1 | 메인 지도 | 상단 검색바 "📍 금천구 독산동" + 카테고리 칩 |
| 2 | 검색바 탭 | 전체 화면 검색 전환, "장소·주소·버스 검색" |
| 3 | 빈 검색 상태 | 빠른 카테고리 칩 + 최근 검색 + 히스토리 |

| 5 | 더 구체적 입력 | "금천 롯데캐슬" → 자동제안(상단) + 장소결과(하단) |

**검색 화면 상세** (단계 2-3) ⭐

|------|--------|

## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장

**토스에서:**

1. **전체 화면 검색 전환**: 주소 필드 탭 → 별도 전체 화면 검색으로 전환. 키보드가 올라와도 결과 리스트 공간 확보. 우리 Step 2도 동일 패턴
2. **밑줄(underline) 입력 + 블루 라벨**: 토스의 일관된 입력 스타일. 우리 온보딩의 다른 입력 필드와 통일성 유지
3. **"건물, 지번 또는 도로명 검색" 플레이스홀더**: 검색 가능한 항목을 명시 → 우리도 "건물명, 도로명 또는 지번으로 검색"
4. **도로명 + 지번 2줄 표시**: 도로명(메인, Bold) + 지번(보조, 회색) → 사용자가 익숙한 주소 형태로 확인
5. **"자세한 주소를 알려주세요" ~세요체 타이틀**: 다만 우리는 상세주소 불필요 (직장 건물 단위라 동/호 불필요)
6. **"검색 결과가 없어요" 빈 상태**: 토스 톤의 빈 상태 메시지 → 우리도 동일 톤

**카카오맵에서:**

7. **키워드 자동제안 + 장소 결과 2계층**: 상단에 검색어 자동완성, 하단에 실제 장소 → 우리도 "가산디지털" 입력 시 상단에 "가산디지털단지역", "가산디지털1로" 제안 + 하단에 건물/회사 결과
8. **장소명 + 카테고리 + 거리 + 주소 행**: 장소 정보를 한 행에 압축 → 우리 결과 행: "**롯데이노베이트** 서울 금천구 가산디지털2로 179" (건물명 Bold + 주소 회색)
9. **🏢 회사/학교 바로가기**: 카카오맵이 "집/회사" 즐겨찾기를 제공 → 우리 Step 2는 직장 입력이므로 "내 직장으로 저장" 옵션 고려 (PHASE 2)
10. **최근 검색 기능**: "최근" 탭에 이전 검색 기록 → 우리 Step 2에서 "직장 1" 입력 후 "직장 2" 입력할 때 유사 주소 추천 가능

### ⚠️ 무시할 것

| 레퍼런스 패턴 | 우리와 다른 이유 | 우리 스펙 |
|-------------|---------------|---------|
| **상세주소 입력** (토스) | 우리는 직장 건물 단위, 동/호 불필요 | 건물명 선택으로 완료 |
| **지도 핀 표시** (카카오맵) | 검색 결과에서 지도 불필요 (온보딩이라 속도 우선) | 텍스트 리스트만 |
| **"출발"/"도착" CTA** (카카오맵) | 길찾기 서비스, 우리는 입력 용도 | 선택 즉시 반영 |
| **카테고리 칩** (카카오맵 AI맛집/음식점) | 장소 검색 전용 | 불필요 |
| **최근/장소/주소/대중교통/길찾기 5탭** (카카오맵) | 과도한 탭 분리 | 단순 리스트 |
| **우편번호 표시** (토스) | 직장 주소에 우편번호 불필요 | 미표시 |

### 💡 신규 인사이트

1. **검색 API 선택**
    - 토스: 다음(카카오) 주소 API 사용 추정 (도로명/지번 구조가 동일)
    - 카카오맵: 자체 장소 검색 API (Kakao Local API)
    - → 우리: **Kakao Local API (키워드 검색)** 사용 권장. 건물명+주소 동시 검색 가능. Daum Postcode는 주소만 반환하므로 건물명 검색이 약함
    - 단, Daum Postcode 서비스가 팝업형이라 모바일에서 더 간편할 수도 있음 → Agent가 판단

2. **검색어 하이라이트 추가**
    - 토스, 카카오맵 모두 입력 텍스트와 일치하는 부분을 하이라이트하지 않음
    - → 우리는 **입력 텍스트 매칭 부분을 Bold 처리**하면 더 직관적 (Google 검색 패턴)
    - 예: "가산디" 입력 → "**가산디**지털단지역" / "**가산디**지털2로 179"

3. **"현재 위치 기반 검색" 옵션**
    - 카카오맵은 거리(930m)를 표시 → 현재 위치 기반
    - → 우리 Step 2에서 "📍 현재 위치로 설정" 버튼 추가하면 → 사용자가 지금 직장에 있을 때 바로 설정 가능
    - 다만, 위치 권한 필요 → 선택 기능

4. **선택 즉시 반영 vs 확인 버튼**
    - 토스: 선택 → 상세주소 화면 → "확인" (2단계)
    - 카카오맵: 선택 → 지도+카드 (추가 단계)
    - → 우리: **선택 즉시 폼에 반영** (추가 단계 없음). 직장 주소는 건물 단위라 상세주소 불필요. 탭 한 번이면 입력 완료

## 4. Agent 전달용 요약

### 이 레퍼런스에서 가져올 핵심 3가지

1. **토스 전체 화면 검색 전환 + 밑줄 입력**: 필드 탭 → 전체 화면 검색, 밑줄(underline) + brand-500 라벨, "건물, 지번 또는 도로명 검색" 플레이스홀더. 빈 상태 "검색 결과가 없어요"
2. **카카오맵 2계층 결과 (자동제안 + 장소)**: 상단에 키워드 자동완성, 하단에 장소 결과(건물명 Bold + 주소 회색 + 거리). 우리 결과도 건물명 우선 표시
3. **선택 즉시 반영**: 토스처럼 상세주소 단계를 거치지 않고, 건물 선택 즉시 폼에 반영하여 입력 단계 최소화

### 반드시 무시할 3가지

1. **상세주소 입력** — 직장 건물 단위, 동/호 불필요
2. **지도 핀 표시** — 온보딩 속도 우선, 텍스트 리스트만
3. **카테고리 칩/5탭** — 장소 검색 전용 기능, 주소 입력에 불필요

### 참고 첨부 파일

- `toss_address_search.mp4` — 참고: 전체 화면 전환, 밑줄 입력, 자동완성(우편번호+도로명+지번), 상세주소 입력, ~세요체
- `kakao_address_search.mp4` — 참고: 2계층 결과(자동제안+장소), 건물명+카테고리+거리+주소 행, 최근 검색, 장소 선택 카드
