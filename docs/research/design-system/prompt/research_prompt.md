# 집콕신혼 — 디자인 레퍼런스 리서치 프롬프트 모음

> **목적**: AI Agent에게 디자인 시스템 구현 시 참고할 레퍼런스와 이미지를 수집하도록 지시하는 프롬프트
> **작성일**: 2026-02-17
> **사용 방법**: 각 섹션의 프롬프트를 AI Agent(Claude, ChatGPT, Perplexity 등)에 직접 입력하여 레퍼런스를 수집

---

## 사용 가이드

각 프롬프트는 **[검색 키워드]** + **[구체적 요청]** + **[프로젝트 컨텍스트]** 구조로 작성되었습니다.
수집한 결과물은 `docs/design-system/references/` 하위에 카테고리별로 저장하는 것을 권장합니다.

## 1. 지도 기반 부동산 검색 UI

### 1-1. 데스크톱 스플릿 뷰 (지도 60% + 리스트 40%)

```
다음 서비스들의 데스크톱 검색 결과 화면 스크린샷과 UI 패턴을 분석해줘:

- Zillow (zillow.com) 검색 결과 페이지
- Redfin (redfin.com) 검색 결과 페이지
- Realtor.com 검색 결과 페이지
- Homes.com 검색 결과 페이지

각 서비스에 대해 다음을 분석해줘:
1. 지도와 리스트의 비율 (px 또는 % 기준)
2. 리스트 패널의 카드 레이아웃 (1열 vs 2열, 카드 높이)
3. 지도 위 마커 디자인 (가격 표시 vs 점 마커)
4. 정렬/필터 바의 위치와 스타일
5. 지도-리스트 간 인터랙션 (hover 싱크, 스크롤 연동)

우리 프로젝트 컨텍스트: 60% 지도 / 40% 리스트 스플릿 레이아웃, 
마커에 종합 점수(0-100) 라벨 핀 사용, 1024px 브레이크포인트에서 전환
```

### 1-2. 모바일 바텀시트 + 지도 패턴

```
모바일에서 지도 위에 바텀시트(Bottom Sheet)를 사용하는 앱/웹 UI 레퍼런스를 찾아줘:

검색 대상:
- Google Maps 모바일 앱의 검색 결과 바텀시트
- Uber/Lyft 앱의 3단 바텀시트 (peek/half/expanded)
- Airbnb 모바일 웹의 지도+리스트 전환
- 직방/다방 앱의 지도 검색 화면
- 토스 (Toss) 앱의 바텀시트 인터랙션 패턴

각 레퍼런스에서 확인할 사항:
1. Peek 상태의 높이와 표시 콘텐츠 (핸들 + 요약 정보)
2. Half 상태에서 카드가 몇 개 보이는지
3. Expanded 상태의 헤더 처리 (검색바, 필터 고정 여부)
4. 드래그 핸들의 크기와 스타일
5. 바텀시트가 올라와 있을 때 지도 인터랙션 가능 여부 (넌모달)
6. 스냅 포인트 간 전환 애니메이션

우리 스펙: Peek ~25%(100-160px) / Half ~50%(350-400px) / Expanded ~90%,
드래그 핸들 40×4px, 넌모달 필수, spring(stiffness:400, damping:30) 트랜지션
```

### 1-3. 지도 마커 디자인 시스템

```
부동산/위치 기반 서비스에서 사용하는 커스텀 지도 마커 디자인을 리서치해줘:

참고 서비스:
- Zillow: 가격 라벨 핀 마커 (흰 배경 + 텍스트)
- Redfin: 가격 + 색상 코딩 마커
- Niche.com: 등급(A+~D) 색상 마커
- Walk Score: 점수 기반 색상 마커
- Trulia: 이소크론(등시간선) 시각화 오버레이

분석 포인트:
1. Default / Hover / Selected / Visited 상태별 스타일 변화
2. 마커 크기 (px), 터치 영역 확보 방법
3. 클러스터링 시 표현 (숫자 원형, 확대 시 분리 애니메이션)
4. 마커 클릭 시 미니 프리뷰 카드의 크기, 위치, 애니메이션
5. 마커에 점수/등급을 표시하는 구체적 방법

우리 스펙: 흰 배경+등급 컬러 텍스트(Default) → 등급 컬러 배경+흰 텍스트(Selected),
5등급 블루-오렌지 다이버징 스케일, 종합 점수 라벨 핀 방식
```

## 2. 점수/등급 시각화 시스템

### 2-1. 원형 게이지 (Circular Gauge) + 등급 배지

```
0-100 숫자 점수를 원형 게이지와 문자 등급으로 시각화하는 UI 레퍼런스를 찾아줘:

참고 서비스:
- 토스(Toss) 신용점수 화면: 원형 게이지 + 카운트업 애니메이션 + 요인 분해
- Niche.com: A+~D 레터 등급 + 카테고리별 서브 등급
- Walk Score: 0-100 원형 점수 + 카테고리 분류
- Credit Karma: 신용 점수 대시보드
- Apple Health / Samsung Health: 링 게이지 UI

각 레퍼런스에서 확인할 사항:
1. 원형 게이지 크기 (px), 스트로크 두께, 트랙/필 색상 처리
2. 중앙 숫자의 폰트 크기와 weight
3. 카운트업 애니메이션 (0에서 목표값까지) 이징과 duration
4. 게이지 아래 또는 옆의 등급 라벨 표현 방식
5. 요인 분해(기여도) 시각화 방법 (바 차트, 파이, 리스트)
6. "ⓘ 더 보기" 같은 계층적 정보 공개(progressive disclosure) 패턴

우리 스펙: 카드 64px / Hero 80-96px / 미니 36-48px,
5등급 블루-오렌지 다이버징(#1565C0 → #D84315), 적녹색 사용 금지,
텍스트 라벨 A+/A/B/C/D 병행 필수 (WCAG 1.4.1)
```

### 2-2. 수평 바 차트 (Score Bar) + 비교 레이더

```
카테고리별 점수를 수평 바와 레이더 차트로 표현하는 UI 패턴을 리서치해줘:

참고할 패턴:
- Niche.com 도시/학교 상세 페이지: 카테고리별 등급 바
- Glassdoor 회사 리뷰: 카테고리별 별점 수평 바
- Trulia Neighborhood 점수: 카테고리 시각화
- FIFA/eFootball 선수 카드: 레이더 차트 (6축)
- 토스 소비 분석: 카테고리별 수평 바 비교

분석 포인트:
1. 수평 바의 높이(px), 코너 라운딩, 배경/필 색상 대비
2. 라벨-바-숫자의 배치 구조 (좌측 라벨 | 바 | 우측 숫자)
3. 바 확장 애니메이션의 방향과 타이밍
4. 2-3개 항목 비교 시 레이더 차트 겹침 처리 (투명도, 스트로크)
5. 레이더 차트의 축 레이블 배치와 가독성 확보

우리 스펙: 바 높이 8px(compact 6px), rounded-full,
레이더 4축(통근·보육·안전·가성비), Recharts,
비교 색상 #0891B2 / #F97316 / #8B5CF6, 채움 투명도 10%
```

## 3. 온보딩 위저드 (Step-by-Step 입력)

### 3-1. 모바일 퍼널형 온보딩 플로우

```
금융/부동산 앱의 단계별 입력 온보딩 UI를 리서치해줘:

참고 서비스:
- 토스(Toss) 송금/대출 신청 플로우: 원 퀘스천 퍼 스크린, 키패드 즉시 노출
- 카카오뱅크 대출 한도 조회: 소득/자산 입력 → 즉시 결과
- Robinhood 계좌 개설: 단계별 프로그레스
- Zillow Home Value: 주소 입력 → 예상 가치 즉시 표시
- 뱅크샐러드 재무 분석: 자산/부채 입력 위저드

각 레퍼런스에서 확인할 사항:
1. 프로그레스 인디케이터 디자인 (바, 스텝 숫자, 점)
2. 금액 입력 UI: 직접 입력 vs 빠른 선택 버튼(+1000만, +5000만 등) 병행
3. 카드형 선택지 디자인 (이모지+라벨 카드, 라디오 대체)
4. 주소 자동완성 입력 필드 UI
5. 하단 고정 CTA 버튼 위치와 safe-area 처리
6. "왜 이 정보가 필요한지" 인라인 설명(ⓘ 툴팁) 배치
7. 금융 정보 입력 시 신뢰 확보 UI (🔒 배지, 비저장 안내)

우리 스펙: 5스텝(부부정보→통근→소득자산→부채예산→분석중), 
총 ~2분 30초 목표, 390px 기준 디자인, 
빠른 입력 버튼(+1,000만/+5,000만/+1억) + 직접 입력 병행
```

### 3-2. 분석 중 로딩 애니메이션

```
데이터 분석/계산 중 로딩 상태를 보여주는 UI 애니메이션 레퍼런스를 찾아줘:

참고 패턴:
- 토스 신용점수 조회: 단계별 텍스트 전환 + 프로그레스
- Stripe Radar 분석: 순차 체크마크 + 상태 텍스트
- Lottie 애니메이션 라이브러리에서 "analysis", "loading", "processing" 검색
- Linear 앱의 이슈 생성 시 로딩 애니메이션
- Vercel 배포 중 단계별 로그 표시

분석 포인트:
1. 순차 단계 표시 (아이콘 + 텍스트 페이드 전환)
2. 각 단계의 duration과 전환 이징
3. 완료된 단계의 체크마크/색상 변화
4. 전체 프로그레스 바와 개별 단계의 관계
5. 결과 화면으로의 전환 애니메이션 (페이드, 슬라이드, 확대)

우리 스펙: 4단계 순차 전환 (🏘️입력분석 1200ms → 🚇통근계산 1400ms 
→ 📊예산시뮬 1600ms → 🗺️단지탐색 1800ms),
sequential fade, 총 ~7초
```

## 4. 결과 카드 & 리스트 UI

### 4-1. 부동산 매물 카드 디자인

```
부동산/숙소 리스트 카드의 정보 배치와 시각적 계층을 분석해줘:

참고 서비스:
- 직방 앱: 매물 카드 (가격 강조, 면적, 층수)
- 호갱노노: 아파트 리스트 카드 (실거래가 중심)
- Airbnb: 숙소 카드 (이미지 + 가격 + 별점)
- Zillow: Property Card (가격 + Zestimate + 침실/욕실)
- 네이버 부동산: 매물 카드 레이아웃

각 카드에서 분석할 사항:
1. 정보의 시각적 계층 (1순위: 어떤 정보가 가장 크고 굵은가)
2. 가격 표시 방법 (폰트 크기, weight, 통화 기호 처리)
3. 서브 정보의 구분 방법 (dot 구분, 슬래시, 줄바꿈)
4. 카드 간 간격, 패딩, 라운딩, 그림자
5. 이미지 유무에 따른 레이아웃 차이
6. "비교 추가" 같은 액션 버튼의 위치와 스타일
7. 순위/등급 배지의 크기, 색상, 위치

우리 카드 구조:
- 1순위: 순위 배지(좌상단) + 단지명(16px Bold) + 원형 게이지(우상단, 64px)
- 2순위: 주소(12px) → 세대수·준공·면적(11px dot) → 실거래가(13px Bold)
- 3순위: 4점수 2×2 그리드 (예산·통근·보육·안전)
- 4순위: 통근시간 + "비교 추가" 버튼
- rounded-2xl, p-4, shadow-sm
```

## 5. 커플 협업 보드 UI

### 5-1. 공유 찜 & 투표 UI 패턴

```
두 명 이상이 협업하여 항목을 저장/비교/투표하는 UI 패턴을 찾아줘:

참고 서비스:
- Redfin Shared Search / Collections
- Pinterest 공유 보드 (핀 + 코멘트)
- Notion 공유 데이터베이스 (다중 뷰, 코멘트)
- Trello/Linear 칸반보드 (투표, 이모지 반응)
- 웨딩 플래닝 앱 (Zola, The Knot): 커플 공유 위시리스트

분석 포인트:
1. "둘 다 찜 vs 한 명만 찜" 매칭 상태 표시 방법
2. 블라인드 투표 UI (투표 전 상대 선택 미노출)
3. 인라인 코멘트/이모지 반응의 카드 내 배치
4. 초대/공유 플로우 (링크 공유, 앱 내 초대)
5. 실시간 동기화 표시 (상대가 보고 있는 항목 하이라이트)

우리 스펙: /board/[id] 공유 보드, 
"둘 다 찜/한 명만 찜" 매칭 표시, 블라인드 투표, 
인라인 코멘트+이모지 반응, 카카오 로그인 기반 파트너 초대
```

## 6. 컬러 시스템 & 디자인 토큰 레퍼런스

### 6-1. 틸 블루 + 코랄 오렌지 팔레트 사례

```
Teal/Cyan 계열 Primary + Orange/Coral 계열 Accent 조합을 사용하는 
프로덕션 서비스의 UI 스크린샷과 컬러 적용 사례를 찾아줘:

검색 키워드:
- "teal and coral color scheme UI"
- "cyan orange complementary palette web app"
- "warm teal blue accent coral design system"

참고할 수 있는 서비스:
- Canva (틸 계열 Primary 사용)
- Notion (웜 뉴트럴 + 컬러 액센트)
- Headspace (틸 + 오렌지 조합 사용 앱)
- Calm 앱 (틸/블루 기반)

분석 포인트:
1. Primary(틸)가 전체 UI에서 차지하는 면적 비율
2. Accent(코랄/오렌지)를 CTA 버튼에만 제한적으로 쓰는지 vs 넓게 쓰는지
3. 웜 뉴트럴(Stone 계열) 배경과의 조화
4. 다크모드에서 Primary 색상 밝기 조정 방법
5. 5등급 다이버징 스케일의 실제 적용 사례

우리 팔레트: Primary #0891B2 (Warm Teal), Accent #F97316 (Coral),
Neutral Stone 계열(#FAFAF9~#1C1917),
Score 5등급 #1565C0→#2196F3→#B0BEC5→#EF6C00→#D84315
```

### 6-2. 접근성 준수 컬러 시각화

```
색맹/색약 사용자를 고려한 데이터 시각화 컬러 스케일 레퍼런스를 찾아줘:

검색 키워드:
- "colorblind safe diverging color scale"
- "WCAG compliant score visualization"
- "blue orange diverging palette accessibility"

참고 자료:
- ColorBrewer2.org의 다이버징 스케일
- Viridis/Cividis 컬러맵
- IBM Design Language 접근성 가이드
- Material Design 3 Dynamic Color 접근성 지침

분석 포인트:
1. 적녹 색맹에 안전한 다이버징 스케일 (블루-오렌지 계열)

## 10. 모션 & 마이크로인터랙션

### 10-1. 점수 카운트업 & 프로그레스 바 애니메이션

```
숫자 카운트업과 프로그레스 바 확장 애니메이션의 고품질 구현 레퍼런스를 찾아줘:

- "count up animation easeOutExpo CSS"
- "progress bar animation examples codepen"
- "score reveal animation UI"
- "number counter animation react framer-motion"

- 토스 신용점수 카운트업 (0→786, easeOutExpo)
- Stripe Dashboard 수치 카운터
- GitHub Contribution 그래프 로딩
- Apple Fitness 링 채우기 애니메이션

분석할 구현 세부사항:
1. easeOutExpo 커브의 체감 속도와 멈춤 느낌
2. 카운트업 시작 트리거 (viewport 진입, 페이지 로드)
