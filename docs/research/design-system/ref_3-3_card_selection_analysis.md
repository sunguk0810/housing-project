# 3-3. 카드형 선택지 UI 분석 — 토스 / 카카오톡

> **분석일**: 2026-02-17
> **분석 대상**: 토스 운세 카드 선택(1장) + 카카오톡 투표(1장, 참고용)
> **목적**: 집콕신혼 `/onboarding` Step 1(주거형태) 및 Step 4(선호 조건) 카드형 선택지 구현 시 AI Agent에게 전달할 레퍼런스

## 1. 상세 분석

### 1-1. 토스 — 이모지 + 라벨 카드 선택 (toss_card_selection.jpeg) ⭐⭐⭐

**페이지 구조 — 원 퀘스천, 카드 선택 패턴**

| 항목 | 분석 값 |
|------|--------|
| 네비게이션 | ← 뒤로 (좌상단) + "설정" (우상단 텍스트) |
| 상단 여백 | 매우 넓음 (~150-180px) — 화면 중앙에 콘텐츠 배치 |
| **부제** | "오늘 이루고 싶은 것을 생각하며" — ~14px Regular, 회색 |
| **타이틀** | **"궁금한 운세를 눌러주세요"** — ~22px **Bold**, 검정, ~세요체 |
| 타이틀 정렬 | **중앙 정렬** |

**카드 그리드** ⭐⭐⭐

| 항목 | 분석 값 |
|------|--------|
| 카드 수 | **3개** 수평 나란히 |
| 카드 크기 | 약 **100-110px × 120-130px** (정사각형에 가까움) |
| 카드 간격 | 약 **12-16px** |
| 카드 배경 | 연한 회색 (#F3F4F6 추정, neutral-100 수준) |
| 카드 라운딩 | 약 **16px** (radius-xl) |
| 카드 보더 | 없음 (배경색으로만 구분) |
| 카드 그림자 | 없음 |
| 전체 정렬 | **중앙 정렬** (좌우 여백 균등) |

**카드 내부 구조**

| 항목 | 분석 값 |
|------|--------|
| **이모지** | ~48-56px, 카드 중앙 상단 |
| **라벨** | ~14px Regular, 회색-검정, 이모지 아래 중앙 |
| 이모지-라벨 간격 | ~8-12px |
| 카드 내 패딩 | ~16-20px |
| 수직 정렬 | 이모지+라벨이 카드 내 수직 중앙 |

**3개 카드 내용**

| # | 이모지 | 라벨 |
|---|-------|------|
| 1 | 💎 (다이아몬드) | 재물운 |
| 2 | 🍀 (네잎 클로버) | 성공운 |
| 3 | ❤️ (하트) | 애정운 |

**선택 상태**: (이 캡처에서는 미선택 상태만 보임)
- 미선택: 연한 회색 배경, 보더 없음
- 선택 시 추정: 배경색 변화 or 보더 추가 or 스케일 효과

**핵심 관찰**:
- **이모지가 주인공**: 48-56px로 매우 크게, 텍스트보다 먼저 눈에 들어옴
- **라벨은 최소한**: 2-3글자 짧은 한국어
- **넓은 여백**: 카드 3개와 타이틀 외에 화면이 거의 비어있음 → 선택에만 집중
- **CTA 버튼 없음**: 카드 탭 자체가 다음 단계로 넘어가는 액션 (별도 "다음" 버튼 없음)

### 1-2. 카카오톡 — 투표 (kakaotalk_survey_selection.jpeg)

**투표 UI** (참고용 — 우리와 다른 패턴이지만 선택지 표현 방식 참고)

| 항목 | 분석 값 |
|------|--------|
| 질문 | "어디어디?" — ~18px Bold |
| 선택지 | 텍스트 목록형: "가디" / "구디" / "씨티렉스" / "제3의 선택지" |
| 결과 표시 | 좌: 선택지 텍스트 / 우: "1명" 투표 수 |
| **프로그레스 바** | 선택지 아래 수평 바 — 노란색(선택됨) / 회색(미선택) |
| 바 높이 | ~4px |
| 바 라운딩 | full |

→ 카카오톡 투표는 **텍스트 리스트형**이라 우리 카드형과 다르지만, **프로그레스 바로 비율 표시**하는 패턴은 Step 4 우선순위 슬라이더에 참고 가능

## 2. 집콕신혼 적용 매핑

### Step 1: 주거형태 선택

토스 카드 패턴을 직접 적용:

```
"어떤 형태의 집을 찾고 계세요?"

[🏢]        [🏠]        [🏗]
아파트       빌라/주택     오피스텔
```

| 항목 | 토스 레퍼런스 | 우리 적용 |
|------|-----------|---------|
| 카드 수 | 3개 | **3개** (아파트/빌라/오피스텔) |
| 이모지 크기 | 48-56px | **48px** |
| 카드 크기 | ~110×130px | **100-120px × 120-130px** |
| 카드 배경 | neutral-100 (#F3F4F6) | **neutral-100 (#F5F5F4)** |
| 카드 라운딩 | 16px | **radius-xl (16px)** |
| 라벨 | ~14px Regular | **text-body-sm (14px) Medium** |
| 선택 상태 | (추정) 배경 변화 | **brand-100 배경 + brand-500 보더 2px** |
| 타이틀 | "궁금한 운세를 눌러주세요" | **"어떤 형태의 집을 찾고 계세요?"** |
| CTA | 없음 (탭=전환) | **"다음" 버튼 (복수 선택 가능하므로)** |

### Step 4: 우선순위 선택 (가능한 적용)

카드형으로 우선순위 항목을 표시할 수도 있음:

```
"가장 중요한 조건을 골라주세요" (복수 선택)

[💰]        [🚇]        [👶]        [🛡]
예산         통근         보육         안전
```

| 항목 | 적용 |
|------|------|
| 카드 수 | **4개** (2×2 그리드 또는 4개 수평) |
| 복수 선택 | ✅ (체크 표시 추가) |
| 선택 순서 | 선택한 순서대로 1,2,3,4 넘버링 표시 |

## 3. 집콕신혼 적용 인사이트

### ✅ 채택 권장

1. **이모지 중심 카드 구조**: 48px 이모지 + 짧은 라벨(2-3글자). 이모지가 시각적 앵커 역할, 텍스트를 읽기 전에 직관적 인지
2. **연한 배경 + 라운딩 16px**: 보더 없이 배경색(neutral-100)으로 카드 영역 구분. 깔끔하고 가벼운 느낌
3. **중앙 정렬 + 넓은 여백**: 화면 중앙에 질문+카드만 배치, 나머지는 빈 공간 → "원 퀘스천" 패턴과 일관
4. **~세요체 타이틀 ("눌러주세요")**: 온보딩 전체 톤과 일치
5. **카드 탭 = 즉시 액션**: 토스는 카드 탭 자체가 다음 단계 전환. 우리는 복수 선택이 필요할 수 있으므로 "다음" 버튼 별도 배치하되, 단일 선택 스텝은 탭 즉시 전환 고려

### ⚠️ 무시할 것

| 패턴 | 이유 |
|------|------|
| **카카오톡 텍스트 리스트형** | 우리는 카드형 (이모지+라벨) |
| **카카오톡 투표 수/바 표시** | 개인 선택이라 투표 결과 불필요 |
| **토스의 CTA 없음 패턴** | 우리 복수 선택 스텝에서는 "다음" 필요 |

### 💡 신규 인사이트

1. **선택 상태 시각 피드백**
    - 토스 캡처에서 선택 상태가 안 보이므로 우리가 정의해야 함
    - 제안: 미선택 = neutral-100 배경 / **선택 = brand-50(#CFFAFE) 배경 + brand-500(#0891B2) 보더 2px + 체크마크**
    - 체크마크: 카드 우상단에 ✓ 원형 배지 (brand-500 채움 + 흰 체크)
    - 선택 시 미세한 scale(1.02) + shadow-sm 트랜지션 추가

2. **복수 선택 시 넘버링**
    - Step 4 우선순위에서 복수 선택+순서가 중요하면
    - 선택한 순서대로 카드 좌상단에 ①②③④ 넘버 배지 표시
    - 배지: brand-500 원형(24px) + 흰 숫자

3. **2×2 그리드 vs 수평 나란히**
    - 토스: 3개 수평 → 카드 너비 ~110px
    - 우리 주거형태(3개): 수평 나란히 OK
    - 우리 우선순위(4개): **2×2 그리드**가 모바일에서 더 안정적 (4개 수평이면 너무 좁음)

## 4. Agent 전달용 요약

### 이 레퍼런스에서 가져올 핵심 3가지

1. **토스 이모지 카드 구조** — 48px 이모지 + 짧은 라벨, neutral-100 배경, radius-xl(16px), 보더 없음. 중앙 정렬 + 넓은 여백
2. **카드 탭 = 액션 패턴** — 단일 선택 시 카드 탭이 즉시 다음 단계로 전환 (CTA 버튼 불필요). 복수 선택 시에만 별도 "다음" 버튼
3. **선택 상태 정의** (토스에 없으므로 우리가 설계) — 미선택: neutral-100 / 선택: brand-50 배경 + brand-500 보더 2px + ✓ 배지
