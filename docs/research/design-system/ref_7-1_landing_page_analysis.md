### 1-2. 직방 — zigbang.com (zigbang_landing.png)

**Hero 섹션** ⭐⭐

| 항목 | 분석 값 |
|------|--------|
| 배경 | **풀 블리드 사진** (야경 아파트 + 따뜻한 조명) — 어두운 오버레이 |
| **헤드라인** | **"부동산은 직방으로"** — ~36px Bold, **흰색**, 중앙 정렬 |
| 부제 | "원룸에서 빌라, 투룸+부터 오피스텔, 아파트 그리고 상가, 사무실까지" — ~14px, 흰색, 중앙 |
| CTA | ❌ 별도 버튼 없음 — **카테고리 카드가 CTA 역할** |
| 네비게이션 | 좌: 직방 로고(오렌지) / 우: 로그인 및 회원가입 + 중개사무소 가입 |

**카테고리 진입 카드** ⭐⭐⭐

| 항목 | 분석 값 |
|------|--------|
| 위치 | Hero 하단, 흰색 카드 오버레이 (Hero와 겹침) |
| 카드 수 | **5개** 수평: 아파트 / 빌라, 투룸+ / 원룸 / 오피스텔 / 상가, 사무실 |
| 카드 구조 | **아이콘(~40px)** + **라벨(~16px Bold)** |
| 카드 스타일 | 흰 배경, 그림자, ~16px 라운딩 |
| 동작 | 탭 → 해당 카테고리 검색으로 바로 진입 |

**하단 섹션**
- 프로모 영상 + 중개사무소 배너
- "빠르게 전달하는 부동산 뉴스" 섹션

**직방 랜딩의 핵심 패턴:**
- Hero: **임팩트 사진 + 짧은 헤드라인(7글자)** + 카테고리 진입 카드
- **CTA가 카테고리 카드**: 버튼이 아니라 "어떤 집을 찾을 건지" 선택으로 바로 진입
- → 우리도 랜딩에서 바로 "아파트/빌라/오피스텔" 선택 → 온보딩으로 자연스럽게 연결 가능

## 3. 집콕신혼 랜딩 페이지 플랜

### 섹션 구성 (위→아래)

```
[Nav]  로고 + "로그인" 
──────────────────────────────────────
[S1: Hero]
  헤드라인 + 부제 + CTA + Hero 일러스트/미리보기
──────────────────────────────────────
[S2: 문제 공감]
  "신혼집 찾기, 이런 고민 있으시죠?"
──────────────────────────────────────
[S3: 3스텝 설명]
  ① 입력 → ② 분석 → ③ 탐색
──────────────────────────────────────
[S4: 4가지 분석 카테고리]
  💰예산 + 🚇통근 + 👶보육 + 🛡안전
──────────────────────────────────────
[S5: 결과 미리보기]
  실제 분석 결과 스크린샷 (흐린 처리)
──────────────────────────────────────
[S6: 신뢰 요소]
  데이터 출처 + 면책 + 사용자 수
──────────────────────────────────────
[S7: 최종 CTA]
  "지금 무료로 시작하기"
──────────────────────────────────────
[Footer]
```

## 6. Agent 전달용 요약

### 이 레퍼런스에서 가져올 핵심 3가지

1. **토스 Hero 구조** — 2줄 헤드라인(32-40px Bold) + 앱 스크린샷 + 단일 CTA. 깔끔하고 빠르게 핵심 전달. 기능 설명은 좌우 교차 레이아웃
2. **직방 카테고리 진입 카드** — Hero에서 바로 "어떤 집을 찾을지" 선택. 우리는 CTA 클릭 → 온보딩이지만, 4카테고리(예산/통근/보육/안전) 카드로 "이런 것들을 분석해요"를 보여주는 데 참고
3. **뱅크샐러드 문제→해결 프레이밍 + 소셜 프루프** — "비교하기 힘드시죠?" → "쉽고 정확하게 비교합니다" 구조. 3개 가치 아이콘. 파트너 로고로 신뢰 확보

### 반드시 무시할 3가지

1. **호갱노노의 "랜딩 없이 바로 서비스"** — 우리는 신규 사용자 대상이라 설명 필요
2. **뱅크샐러드 프로모 배너 슬라이더** — 우리는 프로모 없음, Hero는 고정
3. **직방 풀 블리드 사진 배경** — 우리는 사진 에셋 없음, 일러스트 or 그래디언트
