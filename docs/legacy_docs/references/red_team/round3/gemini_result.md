# Round 3: Trust Breakdown Stress Test (Product Red-Team Analysis)

> **작성자**: Product Red-Team Analyst
> **일자**: 2026-02-13
> **목적**: Round 1(전환), Round 2(입력) 판정을 넘어, 사용자가 **결과를 확인한 후에도 신뢰 부족으로 이탈(Trust Breakdown)**하는 최악의 시나리오를 검증하고 Step3 진입 가능 여부를 재판정함.

---

### 1) Executive Summary (5줄 이내)
입력 마찰을 해결하고 결과 페이지에 도달하더라도, 추천 결과가 **"광고성 블로그"**나 **"근거 없는 조작(Black Box)"**으로 인식되면 전환율은 0에 수렴한다. 본 라운드 분석 결과, 신뢰 붕괴 시 `결과→문의 전환율`은 [가정] 5~10% 수준으로 급락하며, 이는 전체 컨시어지 CTR을 [가정] 2~5%까지 끌어내린다. 특히 점수 산출 근거의 부재와 단점(Bad News) 은폐가 핵심 이탈 요인이다. 최종 판정은 **Conditional Pass**이나, 신뢰 UI(점수 분해/단점 명시) 구현 없이는 Step3 진입이 불가하다(Pending).

---

### 2) Trust Funnel Breakdown (표)

> **전제**: 사용자가 최소 입력을 마치고 결과 리포트 페이지에 도달한 시점부터의 퍼널 분석임.

| 단계 | 기준값(문서근거) | 스트레스값 [가정] | 신뢰 붕괴 원인 | source |
| :--- | :--- | :--- | :--- | :--- |
| **① 결과 노출 (Exposure)** | 결과 조회율 90~95% | **80~85%** | 리포트가 로딩되는 동안 "제휴/광고 로딩 중" 같은 인상을 주거나, 첫 화면이 일반 블로그 랭킹과 유사해 보임 |, |
| **② 근거 확인 (Evidence Check)** | 문서 내 명시 없음 | **체류시간 < 10초** | "85점"이라는 점수만 있고 **"왜?"**가 없다. 데이터 출처(국토부/KB)나 갱신일이 없어 "옛날 데이터/뇌피셜"로 간주됨 |, |
| **③ 신뢰 판단 (Judgment)** | 문서 내 명시 없음 | **이탈률 60~80%** | 추천 단지의 **장점만 나열**되어 있음. 사용자는 "광고비 받은 단지"라고 판단하고 네이버 부동산으로 이탈(체리피킹) | |
| **④ 문의 클릭 (Action)** | 결과→문의 전환율 20~25% (역산) | **5~12%** | "문의하기" 버튼이 **중개사 연결/영업 전화 유도**로 보임. "내 정보가 팔린다"는 공포가 호기심을 압도함 |, |

### 3) Stress Scenarios (표)

| 시나리오 | 공격 가정 | 예상 영향(4지표) [가정] | 조기 징후 | 즉시 대응 | source |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **A. 블랙박스 알고리즘 불신 (The Black Box Doubt)** | 사용자는 "직주근접 S등급" 같은 단순 태그를 믿지 않는다. "내 직장까지 도어-투-도어 몇 분인지", "왜 S인지" 숫자가 없으면 **알고리즘을 조작된 상술**로 치부한다. | - 결과→문의 CTR: **8~15%**<br>- 컨시어지 CTR: **3~6%**<br>- 랜딩 CTR: **1~2%**<br>- 입력 완료율: 영향 없음 | - 결과 페이지 스크롤 깊이가 얕음<br>- "점수 기준이 뭐냐"는 불만 접수 | - 점수를 **레이더 차트**나 **세부 지표(통근시간/학군수)**로 분해<br>- **데이터 출처(KB/국토부)**와 갱신일자 명시 |, |
| **B. 광고성 편향 인식 (The Ad Bias)** | 추천된 Top 3 단지의 단점(언덕, 소음, 노후도)이 전혀 언급되지 않으면, 사용자는 이를 **"돈 받고 쓴 광고"**로 확신하고 즉시 이탈한다. | - 결과→문의 CTR: **5~10%**<br>- 컨시어지 CTR: **2~4%**<br>- 랜딩 CTR: **0.5~1.5%**<br>- 입력 완료율: 영향 없음 | - 리포트 체류시간이 극단적으로 짧음(5초 미만)<br>- CTA 클릭 없이 이탈 | - 모든 추천 단지에 **"이런 점은 아쉬워요(Bad News)"** 1개 이상 필수 포함 (신뢰의 역설 활용) |, |
| **C. 개인정보 인질극 오해 (The Hostage Fear)** | 결과를 일부만 보여주고(블러 처리), "상세 내용을 보려면 문의하세요"라고 하면 사용자는 이를 **"낚시(Bait & Switch)"**로 인식하여 분노한다. | - 결과→문의 CTR: **3~8%**<br>- 컨시어지 CTR: **1.5~3.5%**<br>- 랜딩 CTR: **0.5~1%**<br>- 입력 완료율: **10~20%**(입소문 악화) | - 랜딩 페이지 이탈률 급증<br>- "사기", "낚시" 키워드 피드백 증가 | - **Core 정보(단지명/점수)는 즉시 공개**<br>- 문의는 "정보 해금"이 아니라 **"매물 매칭/동행 예약"** 등 추가 가치로 포지셔닝 |, |

---

### 4) Decision

- **Verdict**: `Conditional Pass`
- **fatal_found**: `false`
- **step3_gate**: `pending`

#### 판정 근거 3개
1. **신뢰 붕괴 시 KPI 달성 불가**: `결과→문의 전환율`이 스트레스 시나리오에서 [가정] 5~12%까지 떨어지며, 이는 Round 1/2에서 합의된 CTR 하한선(3~5%)을 붕괴시킬 위험이 있다.
2. **구현 가능한 방어책 존재**: 점수 분해, 단점 명시, 데이터 출처 표기는 UI/UX 차원에서 해결 가능하므로 Fatal Issue는 아니다.
3. **법적 리스크와의 연계**: "광고 오인"을 피하고 "정보 분석"으로 포지셔닝해야 한다는 Legal Adjudication 결과와 일치한다. 신뢰 UI는 법적 안전장치 역할도 겸한다.

---

### 5) Must-Do Top5

1. **(P0)** **점수 분해 및 근거 시각화**: 단순 총점 대신, 통근시간(분)/대출한도(원)/보육시설(개) 등 정량적 근거를 레이더 차트나 표로 반드시 제시한다.
2. **(P0)** **단점(Bad News) 필수 표기**: Top 10 리포트의 각 단지마다 "아쉬운 점(예: 언덕, 주차난, 층간소음 등)"을 1개 이상 명시하여 객관성을 확보한다.
3. **(P0)** **데이터 출처 및 갱신일 스탬프**: 모든 데이터 블록 하단에 `[Source: 국토부 실거래가 25.02.13 기준]` 형태의 스탬프를 찍어 "뇌피셜" 의심을 차단한다.
4. **(P1)** **문의 목적 재정의 (Anti-Sales)**: 문의 버튼의 문구를 "상담 신청" 대신 "상세 분석 리포트 받기" 또는 "매물 잠복 의뢰(비대면)"로 변경하여 영업 공포를 제거한다.
5. **(P1)** **법적 면책 및 포지셔닝 명시**: 하단 푸터 및 리포트 서두에 "본 서비스는 직접 중개하지 않으며, 데이터 기반 분석 정보를 제공합니다"를 명시하여 신뢰와 컴플라이언스를 동시에 잡는다.
