# Round 3: Trust Breakdown Stress Test (Red-Team Analysis)

> **작성자**: Product Red-Team Analyst  
> **일자**: 2026-02-13  
> **목적**: 추천 결과의 신뢰 붕괴가 전환 퍼널에 미치는 영향을 공격적으로 분해하여, Step2 가정치의 과대추정 여부를 검증

---

## 1) Executive Summary

Round1은 전환 하락의 3축(입력 마찰, 신뢰 붕괴, 저의도 유입)을 식별했고, Round2는 입력 마찰을 퍼널 단계별로 분해했다. [source: step2_summary.md] 본 Round3는 **사용자가 입력을 완료하고 결과를 받은 이후**의 신뢰 퍼널에 집중한다. 핵심 발견: Round2까지의 스트레스 테스트는 "입력 완료 → 문의 클릭" 구간을 단일 전환으로 처리했으나, 실제로 이 구간은 **결과 노출 → 근거 확인 → 신뢰 판단 → 문의 클릭**의 4단계로 분해되며, 각 단계에서 독립적 이탈이 발생한다. [가정] 특히 "점수 산식 불투명 + 통근 체감 괴리 + 광고/제휴 오인"이 결합되면, 결과→문의 전환율은 Round2 스트레스 하한(8~15%)보다 더 낮은 **3~10%**까지 하락할 수 있다. [가정] 결론: **Conditional Pass** — 신뢰 붕괴는 Fatal이 아니나, 신뢰 UI 최소 세트(점수 분해/출처/갱신일/단점/Why-not) 없이 Step3에 진입하면 입력 마찰 방어가 무력화된다.

## 2) Trust Funnel Breakdown

Round2의 퍼널 4단계(랜딩→입력→완료→문의)에서 **④ 결과 확인 → 문의 클릭** 구간을 신뢰 관점으로 재분해한다.

| 단계 | 기준값 (문서 근거) | 스트레스값 [가정] | 신뢰 붕괴 원인 | source |
|:---|:---|:---|:---|:---|
| **④-a. 결과 노출 → 결과 스캔** | 암묵적 90~95% (결과를 받으면 대부분 훑어봄) | **75~88%** | 결과 로딩 후 첫 화면에 "유명 대단지"만 나열되면 **"이거 뻔한데?"** 판단으로 스크롤 없이 이탈. Top10 중 상위 3개가 이미 알고 있는 단지면 나머지를 확인하지 않음 [가정] | round1_adjudication.md §4, step2_market_discovery.md §10 |
| **④-b. 결과 스캔 → 근거 확인** | 암묵적 60~70% (점수/순위를 본 사용자 중 근거까지 확인하는 비율) | **30~50%** | 점수가 숫자만 표시되고 **산식/가중치/출처가 없으면** 근거를 확인할 수단 자체가 없음. "왜 이 단지가 1위인지" 설명이 없으면 점수를 무시하고 체리피킹(단지명만 확보→네이버 이동) [가정] | round1_adjudication.md §4, step2_market_discovery.md §3 |
| **④-c. 근거 확인 → 신뢰 판단** | 암묵적 50~65% (근거를 본 사용자 중 "이걸 믿을 수 있다"고 판단하는 비율) | **35~55%** | 통근 시간이 체감과 다르면 즉시 불신. "60분"이라 했는데 환승/도보 포함하면 80분이면 **전체 점수 체계에 대한 신뢰가 무너짐**. 치안/학군 점수의 출처·기준일이 없으면 "어디서 가져온 건지" 의심. 추천 근거에 특정 중개사/매물 링크가 포함되면 **"광고 아닌가?"** 오인 [가정] | step2_market_discovery.md §10, round1_adjudication.md §5 |
| **④-d. 신뢰 판단 → 문의 클릭** | Round2 채택: 결과→문의 전환율 8~15% [가정] | **3~10%** | 신뢰를 형성했더라도 **"문의=전화/영업"** 인식이 잔존하면 클릭 억제. 또한 신뢰가 높아도 **"이 정도면 혼자 찾아볼 수 있다"**는 자기충족감(Self-sufficiency)이 문의를 불필요하게 만듦. 체리피킹 이탈의 최종 관문 [가정] | round1_adjudication.md §4, claude_result.md §2 |

### 신뢰 퍼널 종합 추정 (결과 확인 이후만)

- **기준 시나리오**: 결과 확인 100명 → 스캔 92 → 근거 확인 60 → 신뢰 판단 36 → 문의 14.4 = **result_to_inquiry_ctr ~14.4%** [가정]
- **스트레스 시나리오**: 결과 확인 100명 → 스캔 78 → 근거 확인 31 → 신뢰 판단 12 → 문의 3.6 = **result_to_inquiry_ctr ~3.6%** [가정]

Round2에서 **입력 퍼널 스트레스를 통과한 사용자조차** 신뢰 퍼널에서 추가 이탈하므로, End-to-End 컨시어지 CTR은 **두 퍼널의 곱**으로 하락한다. [가정]

## 4) Decision

- **Verdict**: `Conditional Pass`
- **fatal_found**: `false`
- **step3_gate**: `pending`

### 판정 근거

1. **신뢰 퍼널이 입력 퍼널과 독립적으로 전환을 깎는다.** Round2는 입력 완료까지의 마찰을 분석했으나, 결과 확인 이후의 4단계 신뢰 퍼널에서 추가로 **60~90%가 이탈**할 수 있다. [가정] 두 퍼널의 곱셈 효과로 End-to-End CTR은 Round2 스트레스 하한보다 더 낮아질 수 있으며, 이는 신뢰 UI 없이는 입력 마찰 방어만으로 KPI를 달성할 수 없음을 의미한다. [source: step2_summary.md §3, claude_result.md §2]

2. **Fatal은 아니다.** 신뢰 붕괴는 **정보 설계(Information Design)**로 해결 가능한 영역이다. 점수 분해, 출처/갱신일, 단점 표시, Why-Not 비교는 모두 데이터가 있으면 구현 가능하며, 사업 모델 자체의 결함이 아니다. 다만, Round1 Must-Do에서 이미 권고된 "신뢰 패키지"가 Step3까지 미구현 상태로 남으면 Fatal 후보로 격상해야 한다. [source: round1_adjudication.md §5, step2_market_discovery.md §4]

3. **Round3 고유 발견: "통근 체감 괴리"가 서비스의 핵심 가치를 직격한다.** 이 서비스의 차별 포인트는 "2직장 통근 균형"이며, 통근 시간의 신뢰가 무너지면 **서비스 존재 이유 자체가 흔들린다**. [가정] 점수 블랙박스나 광고 오인은 UI로 완화 가능하지만, 통근 데이터의 정합성은 **데이터 소스 + 계산 로직**의 문제이므로 Step3에서 반드시 검증 우선순위를 높여야 한다. [source: step2_market_discovery.md §1, §8]

## 5) Must-Do Top 5

1. **(P0) 점수 분해 UI 표준화**: 총점 아래에 항목별(통근/재무/치안/보육/생활) 점수 막대 + 각 항목의 근거 1문장(출처, 기준일 포함)을 필수 표시. 숫자만 있는 점수는 금지. 시나리오A(점수 블랙박스)의 직접 대응이며, Round1 Must-Do "점수 분해" 미이행 시 신뢰 퍼널 ④-b에서 대량 이탈 발생. [source: round1_adjudication.md §5, step2_market_discovery.md §10]

2. **(P0) 통근 시간 표기를 "범위+조건" 방식으로 전환**: 단일 값(예: "42분") 대신 **"35~50분 (출근 시간대, 환승 1회, 문앞-문앞)"** 형태로 표기. 첫 버전에서는 API 값에 +15분 보수 보정을 적용. 시나리오B(통근 체감 괴리)가 서비스 핵심 가치를 직격하므로 P0. [source: step2_market_discovery.md §10]

3. **(P0) 단점(Bad News) + Why-Not 분석 필수 포함**: 추천 단지마다 약점 최소 1개를 표시하고, 인접 비추천 단지에 대한 "왜 추천하지 않는지" 비교를 추가. 시나리오C(광고 오인)의 구조적 해법이며, 단점 없는 추천은 "광고"로 인식됨. [source: round1_adjudication.md §5, step2_experiment_plan.md §3.1]

4. **(P1) 아웃링크 복수 플랫폼 분산 + 비광고 고지**: 매물 확인 링크를 네이버/직방/호갱노노 등 복수 플랫폼으로 분산하고, 결과 하단에 "이 추천은 광고/제휴가 아닙니다" 고정 문구 배치. 특정 플랫폼 단일 연결은 제휴 오인을 유발. [source: step2_market_discovery.md §10, step2_experiment_plan.md §3.1]

5. **(P1) 신뢰 지표 계측 세팅**: 결과 페이지 스크롤 깊이, 점수 분해 클릭률, 아웃링크 vs 문의 클릭 비율, "왜 이 단지?" VOC 빈도를 트래킹. Round2의 퍼널 이벤트 계측에 신뢰 퍼널 4단계(④-a~d)를 추가하여, 어디서 신뢰가 무너지는지 실측 가능하게 함. [source: step2_summary.md §5, round1_adjudication.md §5]
