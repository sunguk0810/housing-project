# Round 2: Input Friction Stress Test (Red-Team Analysis)

> **작성자**: Product Red-Team Analyst  
> **일자**: 2026-02-13  
> **목적**: 입력 퍼널 각 단계의 마찰 비용을 공격적으로 분해하여, Step2 가정치의 과대추정 여부를 검증

---

## 1) Executive Summary

Round1은 전환 하락의 구조적 원인을 3축(입력 마찰, 신뢰 붕괴, 저의도 유입)으로 식별했으나, 입력 퍼널 내부를 단계별로 분해하지 않았다. [source: round1_adjudication.md] 본 라운드는 퍼널을 4단계로 쪼개어 **각 전환점에서의 마찰 비용을 개별 추정**한다. 핵심 발견: 민감정보 입력 수용률 65~70%는 "조건부 수용 의향"이지 "실제 완료"가 아니며, 의향→행동 간 감쇠율을 반영하면 최소 입력 완료율은 **15~25%**까지 하락할 수 있다. [가정] 특히 **2단계(직장2/예산 범위) 진입 시점**이 퍼널의 구조적 병목이며, 이 지점의 이탈률이 전체 CTR을 지배한다. [가정] 결론: **Conditional Pass** — 입력 마찰 자체는 Fatal이 아니나, 현재 플로우로는 기준 KPI 달성이 극히 어렵고, 점진 입력 + 무입력 Teaser 적용 후 재측정이 필수다.

### 퍼널 종합 추정 (End-to-End)

- **기준 시나리오**: 랜딩 100명 → 입력 시작 75 → 최소 완료 41 → 결과 확인 37 → 문의 7.4 = **컨시어지 CTR ~7.4%** [가정, 기준값 기반 역산]
- **스트레스 시나리오**: 랜딩 100명 → 입력 시작 45 → 최소 완료 18 → 결과 확인 14 → 문의 1.7 = **컨시어지 CTR ~1.7%** [가정, 스트레스 하한 기반 역산]

이 역산은 Round1 채택 스트레스 기준(컨시어지 CTR 5~8%)의 **하한 이하까지 떨어질 수 있음**을 보여준다. [가정]

## 4) Decision

- **Verdict**: `Conditional Pass`
- **fatal_found**: `false`
- **step3_gate**: `pending`

### 판정 근거

1. **입력 퍼널 역산 결과, 기준 KPI 달성이 현재 플로우에서는 구조적으로 어렵다.** 4단계 퍼널을 곱하면 스트레스 시나리오에서 컨시어지 CTR이 1.7~5%까지 하락하며, 이는 Round1 채택 스트레스 기준(5~8%)의 하한 이하다. [가정] 그러나 이는 **현재 설계 기준**이며, No-Input Teaser + 점진 입력 적용 시 ①→② 전환율 개선으로 회복 가능한 영역이다. [가정] [source: step2_experiment_plan.md §2, round1_adjudication.md §3]

2. **Fatal은 아니다.** 입력 마찰은 UX 설계로 완화 가능한 문제이며, 사업 모델 자체의 결함이 아니다. 다만, "입력 없이는 가치를 보여줄 수 없는 구조"가 고착되면 Fatal 후보로 격상해야 한다. [가정] [source: round1_gemini.md §3, step2_market_discovery.md §4]

3. **Round2 고유 발견: "의향-행동 갭"이 Round1에서 과소평가되었다.** AI Proxy 기반 민감정보 수용률 65~70%는 실제 행동 완료율로 직역할 수 없으며, 실제 완료율은 의향의 55~75% 수준(=36~53%)으로 감쇠될 가능성이 높다. [가정] 이 갭을 No-Input Teaser로 우회하는 것이 Step3 진입의 필수 전제다. [source: step2_market_discovery.md §3]

## 5) Must-Do Top 5

1. **(P0) No-Input Teaser 경로 확정**: 입력 없이 지역만 선택해도 샘플 결과(블러/제한)를 보여주는 플로우를 기본 경로로 설계. 입력은 "정밀 결과 해제" 트리거로 전환. Round2 최우선 리스크(의향-행동 갭)의 유일한 구조적 해법. [source: round1_gemini.md 시나리오A 대응, round1_adjudication.md §5]

2. **(P0) 2단계 입력 필드 최소화 + Optional 전환**: 직장2를 선택사항으로 전환. 보유현금/월상환은 3-버튼 프리셋(체감 라벨)으로 축소. 정확값 입력 필드를 완전 제거. 이것이 ②단계 절벽(시나리오A)의 직접 대응. [source: step2_market_discovery.md §3, round1_adjudication.md §5]

3. **(P0) 퍼널 4단계 이벤트 계측 세팅**: 랜딩 도착 / 입력 시작 / 최소 입력 완료 / 결과 확인 / 문의 클릭의 5개 이벤트를 분리 계측. 특히 1→2단계, 2단계 내 필드별 이탈 지점을 트래킹. Round1 Must-Do(이벤트 분모 고정)를 입력 마찰 관점으로 확장. [source: round1_adjudication.md §5, step2_experiment_plan.md §2]

4. **(P1) 입력 폼 안심 패키지 표준화**: 모든 금액/직장 필드에 "저장 안 함 / 연락 안 함 / 대략 범위 OK" 3문장 고정 노출. 입력 단계마다 진행률 바 + "이 정보가 필요한 이유" 1문장. 시나리오 B·C의 심리적 저항을 직접 완화. [source: round1_gpt.md 시나리오2 대응, round1_gemini.md Must-Do #3]

5. **(P1) No-Input vs Minimal-Input vs Full-Input 3-arm 비교 설계**: Step3에서 실측 실험을 돌릴 때, 세 플로우의 CTR 차이를 측정할 수 있도록 사전에 프로토타입 3종과 측정 기준을 확정. Round1에서 Gemini가 제안한 A/B 테스트를 3-arm으로 확장하여 입력 마찰의 정확한 비용을 실측. [source: round1_gemini.md Must-Do #1, step2_experiment_plan.md §3 R2]
