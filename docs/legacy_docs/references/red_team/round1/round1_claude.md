# Round 1 — Conversion Stress Test (전환성 반박)

> **역할:** Product Red-Team Analyst
> **대상:** 신혼부부 맞춤형 주거 추천 서비스 Step2 기준 가정
> **목적:** 현재 기준 KPI가 과대추정되었을 가능성을 최대한 공격적으로 반박
> **원칙:** 근거 없는 단정 금지, 불확실성은 [가정]으로 표기

---

## 섹션 1: Executive Summary

Step2에서 채택된 기준 KPI(컨시어지 CTR 18~21%, 랜딩 CTR 6~9%, 입력 완료율 50~60%)는 **세 가지 구조적 과대추정 편향**을 내포하고 있다. 첫째, AI 가상 페르소나의 "의향"과 실제 "행동" 사이 의향-행동 괴리(Intent-Action Gap)가 반영되지 않았다. 둘째, 서비스 인지→랜딩→입력→결과→문의의 5단계 퍼널에서 각 단계별 이탈을 과소추정하는 **복합 이탈 효과**가 누락되었다. 셋째, 동시 존재하는 "무료 대안"(네이버 부동산+호갱노노)과의 비교 판단에 걸리는 **전환 비용 저항**이 충분히 가격화되지 않았다. 스트레스 시나리오 적용 시 컨시어지 CTR은 **8~14%**[가정], 랜딩 CTR은 **3~6%**[가정], 입력 완료율은 **28~42%**[가정]까지 하락할 수 있다. **판정: Conditional Pass** — 치명결함은 아니나 현재 수치를 액면 그대로 Step3 Gate 기준으로 사용하는 것은 위험하다.

---

## 섹션 2: Stress Scenarios

### 반박 근거: 왜 현재 기준이 과대추정인가

현재 Step2 기준 가정의 산출 구조를 역추적하면 다음 3개의 과대추정 편향이 확인된다:

**편향 1 — AI 가상 페르소나의 의향-행동 괴리 (Intent-Action Gap)**

가상 페르소나 20명의 "문의 행동 가능성" 평균은 약 62%[가정]이다. 그러나 행동경제학 문헌과 SaaS 업계 벤치마크를 참고하면, 설문/인터뷰 단계에서 "사용하겠다"고 답한 비율과 실제 행동 전환율 사이에는 통상 **3~5배의 갭**이 존재한다[가정]. 특히 금융 민감정보 입력이 수반되는 서비스의 경우 이 갭은 더 벌어진다[가정]. Step2 Adjudication에서 이미 "행동-의향 괴리"를 한계로 인정했지만, 실제 KPI 수치 보정에는 체계적으로 반영하지 않았다.

**편향 2 — 복합 이탈 효과(Compound Drop-off)**

Step2 KPI는 "랜딩 CTR"과 "컨시어지 CTR"을 독립적으로 추정하지만, 실제 퍼널은 **광고 노출 → 클릭 → 랜딩 도착 → 입력 시작 → 입력 완료 → 결과 확인 → 컨시어지 문의**의 7단계이다. 각 단계별 이탈률을 보수적으로 추정하면:

| 단계 | 전환율 추정 [가정] | 누적 전환율 [가정] |
|---|---|---|
| 광고 노출 → 클릭 | 1.5~3.0% (카카오/네이버 부동산 키워드 경쟁 반영) | 1.5~3.0% |
| 클릭 → 랜딩 도착 | 85~90% (이탈/로딩) | 1.3~2.7% |
| 랜딩 도착 → 입력 시작 | 40~55% (가치 제안 이해 후 행동 개시) | 0.5~1.5% |
| 입력 시작 → 입력 완료 | 50~65% (민감정보 이탈 반영) | 0.3~1.0% |
| 입력 완료 → 결과 확인 | 90~95% | 0.25~0.9% |
| 결과 확인 → 컨시어지 문의 | 18~24% (Step2 기준) | **0.05~0.22%** |

최종 End-to-End 전환율은 **0.05~0.22%**[가정]에 불과할 수 있다. Step2에서는 이 복합 효과를 다루지 않았다.

**편향 3 — 전환 비용 저항(Switching Cost Resistance)**

사용자의 현재 대체행동(네이버 부동산+카카오맵+엑셀)은 **무료이고 이미 습관화**되어 있다. 새로운 서비스에 민감정보를 입력하는 행위는 "편의성 향상"이라는 보상 대비 "프라이버시 비용"이라는 지출이 수반된다[가정]. 가상 리서치에서 "민감정보 수용도 5.8/10"이라는 결과가 나왔지만, 이는 "가상의 서비스가 있다면"이라는 전제 하의 답변이며, 실제 브라우저 앞에서 소득과 현금을 입력하는 마찰(friction)은 2~3배 높게 작용한다[가정].

### 시나리오 A: "입력 절벽" — 민감정보 이탈 폭증

| 항목 | 내용 |
|---|---|
| **공격 가정** | 점진 입력 UX에도 불구하고, Step1(월상한+직장 프리셋)에서 Step2(현금/소득 정밀화)로 넘어가는 전환율이 Step2 가정(50~60%)의 절반 수준에 그친다[가정]. 이유: ① 결과를 이미 봤으므로 추가 입력 동기 소멸 ("이 정도면 됐지"), ② "정밀화하면 내 예산이 부족하다는 현실 직면"에 대한 심리적 회피, ③ 경쟁 서비스(호갱노노) 대비 추가 입력에 대한 피로감 |
| **예상 영향** | 입력 완료율: **28~35%**[가정] (기준 50~60% 대비 -40~-50%). 정밀 점수를 받는 사용자가 줄어 컨시어지 문의 근거(상세 리포트)의 품질도 하락 → 컨시어지 CTR 연쇄 하락 **12~16%**[가정] (기준 18~24% 대비 -30%). 랜딩 CTR은 Step1만으로도 결과가 나오므로 직접 영향 제한적이나, 결과의 정밀도 하락이 "이 서비스 믿을 수 있나" 신뢰 저하를 유발하면 **4~7%**[가정]로 하락 가능 |
| **조기 징후** | ① Step1→Step2 전환율이 30% 미만[가정] ② Step2 입력 화면 체류 시간 < 15초(입력 시도 없이 즉시 이탈)[가정] ③ Step1 결과 페이지에서 "공유하기" 비율 > "정밀화하기" 비율[가정] |
| **즉시 대응** | ① Step1 결과를 "미리보기(Blurred/Partial)"로 제한 — 정밀화 입력 시에만 Full 결과 해금 (단, 사용자 반발 리스크 존재) ② 범위 선택형을 "슬라이더"로 변경 — 구체 금액이 아닌 "대략 이 정도" 제스처로 부담 감소 ③ 입력 완료 시 "맞춤 리포트 무료 제공" 인센티브 추가 |

### 시나리오 B: "점수 불신" — 추천 결과 신뢰 실패

| 항목 | 내용 |
|---|---|
| **공격 가정** | 사용자가 결과를 받아본 뒤 **"내가 아는 현실과 다르다"**고 느낀다[가정]. 이유: ① 공공데이터 기준일이 3~6개월 전이라 현재 시세/정책과 괴리, ② 통근 시간이 ODsay API 기반이나, 사용자 본인이 체감하는 시간과 10~15분 차이 발생(도보+대기 체감시간 과소반영)[가정], ③ 추천 상위 단지가 "이미 물건이 없는" 인기 단지이거나, 반대로 "비선호 지역"이 올라와 있는 경우 → "이 서비스 별로다" 즉시 이탈 |
| **예상 영향** | 결과 확인 후 컨시어지 문의로의 전환이 급락 → 컨시어지 CTR: **8~13%**[가정] (기준 18~24% 대비 -45%). 랜딩 CTR은 첫 인상에서 결정되므로 직접 영향은 적으나, 입소문/재방문율에 영향 → 장기적으로 **3~5%**[가정]로 수렴. 입력 완료율 자체는 유지(35~50%)[가정]되나 "의미 있는 행동(문의/공유)"으로 이어지지 않음 |
| **조기 징후** | ① 결과 페이지 스크롤 깊이 < 30%(상위 3개만 보고 이탈)[가정] ② 같은 사용자의 재검색 비율 < 10%(한 번 보고 안 돌아옴)[가정] ③ 점수 breakdown 클릭률 < 5%(근거에 관심 없음 = 신뢰 포기)[가정] ④ 커뮤니티/SNS에서 "신혼집 추천 서비스 써봤는데 별로" 류의 부정 후기 조기 감지 |
| **즉시 대응** | ① 데이터 기준일을 결과 상단에 대형 표시 + "이 데이터는 YYYY.MM 기준입니다" 경고 ② ODsay 통근 시간에 "+10~15분 버퍼"를 사용자 옵션으로 제공("출퇴근 여유 포함" 토글) ③ "이 결과가 맞지 않나요?" 피드백 버튼 → 사용자 보정 데이터 수집 ④ 결과 단지별 "현재 매물 유무" 외부 링크(네이버 부동산 아웃링크) 즉시 노출 |

### 시나리오 C: "카테고리 무관심" — 타겟 시장 자체의 좁은 폭

| 항목 | 내용 |
|---|---|
| **공격 가정** | "신혼부부 주거 추천"이라는 카테고리 자체가 광고 도달 시 **즉각적 관심을 끌지 못한다**[가정]. 이유: ① 신혼부부의 주거 탐색 활성 기간은 결혼 전후 3~6개월로 **극히 한정적**이며[가정], 이 기간 외에는 광고 노출 대비 클릭이 극도로 낮음, ② "AI가 집 추천"이라는 프레이밍이 사용자에게 **"부동산 광고/스팸"으로 오인**될 가능성[가정], ③ 핵심 타겟(결혼 3~6개월 전 맞벌이)의 모수 자체가 연 ~22만 쌍(2024 혼인 건수 기준)[가정] × 서울·수도권 비율 50% × 맞벌이 비율 65% ≈ **7.2만 쌍/년**으로 한정 → 월간 활성 탐색자 약 **1.2~1.8만 명**[가정]에 불과 |
| **예상 영향** | 랜딩 CTR: **3~5%**[가정] (타겟 정밀도 하락, "이게 뭐지?" 이탈 증가). 입력 완료율: **30~42%**[가정] (도달한 소수만 관심층이므로 동기는 높으나, 절대 수 부족). 컨시어지 CTR: **10~15%**[가정] (결과를 본 소수 중에서도 "무료 서비스인데 문의까지?"라는 추가 장벽). 핵심 문제: **전환율이 아무리 높아도 모수(TAM)가 작아** 월간 컨시어지 문의 건수가 한 자릿수에 머물 위험[가정] |
| **조기 징후** | ① 광고 CPC가 부동산 키워드 경쟁으로 3,000원+ 급등[가정] ② 월간 랜딩 방문자 < 500명(광고 예산 50만원 기준)[가정] ③ 자연 유입(SEO/입소문)이 4주 후에도 전체 트래픽의 10% 미만[가정] ④ SNS 광고 CTR < 0.5%[가정] |
| **즉시 대응** | ① 카테고리 확장: "신혼" → "맞벌이 부부 전체"로 타겟 재정의 (TAM 3~5배 확대)[가정] ② CTA 리프레이밍: "AI 주거 추천" → "출퇴근 30분 줄이는 동네 찾기" — 구체적 편익 중심 ③ 채널 전환: 유료 검색 광고보다 결혼 커뮤니티(더웨딩, 결혼준비 카페) 타겟 콘텐츠 마케팅 우선 ④ 포트폴리오 목적이므로 절대 수가 작아도 **전환율 데이터 자체를 확보**하는 것에 초점 재설정 |

## 섹션 3: Decision

### 3.1 최종 판정

| 항목 | 판정 |
|---|---|
| **Verdict** | **Conditional Pass** |
| **Fatal(치명결함) 여부** | **No** — 서비스 자체의 존재 가치(Pain은 실재)는 유효하나, 전환 가정치가 2~3배 과대추정되었을 수 있다[가정]. |
| **Step3 진입 가능 여부** | **Pending(조건부)** — 아래 3개 조건 충족 시 진입 가능 |

### 3.2 Step3 진입 조건

| # | 조건 | 임계치 | 검증 방법 |
|---|---|---|---|
| 1 | **실제 프로토타입 입력 완료율** | ≥ 40% (Step2 기준 50%에서 하향) | 프로토타입 테스트 5쌍에서 측정 |
| 2 | **랜딩 A/B 실측 CTR** (500 클릭 이상) | ≥ 4% (Step2 기준 6%에서 하향) | 유료 광고 4주 집행 후 측정 |
| 3 | **컨시어지 문의 의향** (인터뷰 기반) | 인터뷰 10쌍 중 3쌍 이상이 "결과를 보면 문의하겠다"고 응답 | 심층 인터뷰 시 프로토타입 결과 보여준 후 측정 |

### 3.3 판정 근거

현재 Step2 기준 가정이 과대추정 편향을 내포하고 있지만, 이것이 **치명결함(Fatal)**은 아닌 이유:

1. **Pain의 실재성은 의심하지 않는다** — 3개 AI 리서치 모두 합의했고, 논리적으로도 "2직장 통근 최적화의 수작업 고통"은 구조적으로 해소되지 않는 문제다.
2. **과대추정의 원인이 수정 가능하다** — 의향-행동 괴리는 UX 최적화와 실측으로 보정할 수 있고, 복합 이탈 효과는 퍼널 설계로 줄일 수 있다.
3. **포트폴리오 + 자가 사용 목적이므로** — 상업적 손익분기 달성이 필수 조건이 아니다. 전환율이 낮더라도 "기술 구현 + 실제 사용 가능성" 자체가 목표를 충족한다.
4. **다만**, 현재 수치를 "이미 검증된 것"으로 간주하고 개발에 몰입하면, 나중에 "왜 아무도 안 쓰지?"라는 함정에 빠질 위험이 있다. 스트레스 테스트된 보수 수치(컨시어지 8~14%, 랜딩 3~6%)를 **계획 기준(Planning Baseline)**으로 채택할 것을 권고한다.
