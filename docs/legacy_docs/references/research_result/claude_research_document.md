# 신혼부부 주거 추천 서비스 — 시장성·수요·MVP 타당성 종합 리서치

> **작성일**: 2026-02-13  
> **목적**: 신혼부부 대상 주거 추천 서비스의 시장성/수요/MVP 타당성 검증  
> **대상 지역**: 서울 + 경기 + 인천 (수도권)  
> **주거유형**: 아파트 (전세/매매)

## A. Executive Summary

**수도권 신혼부부 약 51.5만 쌍을 위한 "예산+통근+보육+치안 통합 추천" 서비스는 현존 8개 주요 플랫폼 어디에서도 제공하지 않는 명확한 경쟁 공백이다.** 2024년 혼인 건수가 22.2만 건으로 역대 최대 증가율(+14.8%)을 기록하며 시장이 회복세에 진입했고, 맞벌이 비중 59.7%로 "양쪽 직장 기준 통근 최적화" 수요가 구조적으로 증가하고 있다. 핵심 데이터(실거래가·통근·보육·학군·치안)가 모두 공공 API로 무료 확보 가능하며, 8~12주 MVP 개발이 기술적으로 현실적이다. 가장 큰 리스크는 매물 데이터 실시간성(크롤링 법적 제약)과 ODsay API 대용량 호출 비용이며, 수요 검증 3~4주 후 Go/No-Go를 판단해야 한다. 핵심 성공지표인 **매물 문의 클릭률 5% 이상**이 확인되면 MVP 개발을 진행할 근거가 충분하다.

### B-2. 5대 미충족 니즈가 만드는 경쟁 공백

기존 플랫폼은 크게 "매물 검색형"(직방·다방·네이버)과 "투자 분석형"(아실·리치고·부동산 지인)으로 양분되어 있으며, **신혼부부의 의사결정 초기 단계("어디에 살아야 하지?")를 지원하는 서비스는 전무**하다.

| # | 미충족 니즈 | 현황 |
|---|-----------|------|
| 1 | **"내 예산으로 어디에 살 수 있나?" 역산 기능** | KB부동산이 대출한도 조회를 제공하지만 개별 매물 단위에 그침. 보유현금+연봉+기존대출을 종합해 접근 가능 지역/단지를 자동 추천하는 서비스 부재 |
| 2 | **치안 데이터 완전 공백** | 8개 서비스 중 지역별 범죄율·안전지수를 제공하는 곳이 단 하나도 없음 |
| 3 | **보육시설 접근성이 추천 조건으로 미활용** | 호갱노노가 어린이집 정보를 표시하지만 정보 조회 수준이며 추천 알고리즘에 미반영 |
| 4 | **통근+예산+보육+치안 통합 가중치 추천 알고리즘 부재** | 개별 요소는 분산되어 있으나 하나의 점수로 통합하는 서비스 없음 |
| 5 | **맞벌이 부부 "양쪽 직장" 동시 최적화 부재** | 기존 통근 검색 기능은 단일 직장 기준 |

> **참고**: 직방이 2025년 5월 **"찾아줘 신혼집"** 서비스를 출시했으나, 조건 기반 매물 추천+단계별 상담 서비스로 예산 역산이나 통합 스코어링과는 다른 접근

### C-3. 랜딩페이지/설문 실험 2안

#### 실험 A — 랜딩페이지 사전등록형

| 항목 | 내용 |
|------|------|
| **헤드라인** | "신혼집, 두 사람 직장에서 60분 이내 + 예산 맞는 아파트만 골라드립니다" |
| **서브헤드** | 보유현금·연봉·직장주소만 넣으면 Top 10 단지 + 예상 월주거비 |
| **CTA** | "출시 알림 받기" 이메일 수집 |
| **트래픽 소스** | 인스타그램 광고(30대 기혼/예비부부 타겟) + 네이버 키워드 광고("신혼부부 전세", "신혼집 추천") |
| **예산** | 50~100만 원, 2~3주 운영 |
| **측정 지표** | 페이지 방문 → 이메일 등록 전환율, 평균 체류시간, 스크롤 깊이(Hotjar) |

#### 실험 B — 설문 + 모의 결과 제공형

| 항목 | 내용 |
|------|------|
| **도구** | Tally.so (5분 분량 설문) |
| **설문 내용** | 혼인상태, 예산 구간, 직장 위치, 기능 선호도 5점 척도, NPS 0–10 |
| **모의 결과** | 제출 후 입력값 기반 미리 큐레이션한 3~5개 샘플 단지 추천 |
| **핵심 측정** | 결과 화면 "매물 문의" 모의 버튼 **클릭률** |
| **후속 CTA** | "실제 서비스 출시 시 알림 받기" 이메일 수집 |
| **목표 표본** | 설문 300건 이상, 랜딩 1,000 방문 이상 |

#### 실험 일정

```
Week 1: 리쿠르팅 + 인터뷰 시작
Week 2: 인터뷰 완료 + 랜딩 구축
Week 2–3: 설문·광고 집행
Week 3–4: 데이터 분석 → Go/No-Go 결정
총 3~4주
```

### C-4. Go/No-Go 판단 기준

| 지표 | 🔴 No-Go | 🟡 조건부 Go (피벗 검토) | 🟢 확신 Go |
|------|----------|------------------------|-----------|
| 랜딩페이지 이메일 전환율 | < 3% | 3–8% | > 8% |
| 설문 완료율 | < 30% | 30–60% | > 60% |
| NPS 평균 | < 6 | 6–8 | > 8 |
| "사용하겠다" 응답 비율 | < 40% | 40–70% | > 70% |
| **모의 매물 문의 클릭률** | **< 5%** | **5–15%** | **> 15%** |
| 인터뷰 10건 중 "꼭 사용" 비율 | < 3명 | 3–6명 | > 6명 |
| 유료 결제 의향 비율 | < 15% | 15–30% | > 30% |

> **참고**: 부동산 서비스는 고관여 상품으로 일반 B2C 대비 전환율이 낮음. Unbounce 2024 보고서 기준 부동산 업종 랜딩페이지 중간 전환율 **2.6~4.0%**

### D-2. 매물 데이터의 핵심 난제

실거래가(과거 거래)는 공공 API로 완전 무료 확보 가능하나, **현재 시점의 활성 매물(호수 단위)**은 공공 데이터로 제공되지 않는다. 네이버 부동산·직방·다방의 매물은 공인중개사가 직접 등록하는 데이터로, 크롤링 시 **저작권법·컴퓨터프로그램보호법 위반 리스크**가 있다.

**현실적 대안 3가지:**

| 전략 | 설명 | 적용 시점 |
|------|------|----------|
| **실거래가 기반 시세 추정 + 외부 링크** | 공공 데이터로 시세 추정 후 네이버 부동산 등으로 매물 연결 | MVP 즉시 |
| **제휴/파트너십** | 직방·다방·네이버 부동산과 데이터 제휴 협상 | 트래픽 확보 후 |
| **공인중개사 직접 등록** | 자체 매물 등록 시스템 구축 | 중장기 |

### D-3. 치안 데이터 설계안

공개 범죄 통계의 세밀도가 경찰서 단위(구 수준)에 한정되므로, **다층 보정 방식**을 설계한다.

```
치안 점수 산출 공식:

Safety = 0.6 × 범죄밀도등급_역수 
       + 0.2 × CCTV_밀도 
       + 0.1 × 치안시설_근접도 
       + 0.1 × 가로등_밀도
```

| 레이어 | 데이터 소스 | 역할 |
|--------|-----------|------|
| 1차 (기반) | 생활안전지도 WMS — 5대 범죄 밀도 10등급 | 기본 치안 수준 |
| 2차 (보정) | 반경 500m 내 CCTV 수 (공동주택 기본정보 + SafeMap 치안시설 API) | 범죄 억제력 보정 |
| 2차 (보정) | 경찰서·지구대·파출소 거리 | 대응 속도 보정 |
| 2차 (보정) | 가로등 밀집도 (서울열린데이터) | 야간 안전 보정 |

> [가정] 가중치(0.6/0.2/0.1/0.1)는 전문가 검증 및 사용자 피드백으로 조정 필요

## E-4. 점수화 모델 초안

### 3가지 가중치 프로필

| 요소 | 🏦 예산 중심형 | 🚇 통근 중심형 | ⚖️ 균형형 |
|------|-------------|-------------|---------|
| 예산 (Budget) | **45%** | 20% | 30% |
| 통근 (Commute) | 15% | **45%** | 25% |
| 학군 (School) | 10% | 10% | 15% |
| 보육 (Childcare) | 20% | 10% | 15% |
| 치안 (Safety) | 10% | 15% | 15% |

### 각 요소 정규화 방법 (0–100점)

| 요소 | 계산식 | 설명 |
|------|--------|------|
| **예산** | `Budget_norm = (최대월주거비 - 해당단지 월고정비) / 최대월주거비 × 100` | 월고정비 = 대출원리금 + 관리비 + (월세) |
| **통근** | `Commute_norm = max(0, (60 - max(통근1, 통근2)) / 60 × 100)` | 60분 이하만 후보, 더 긴 통근 기준 |
| **학군** | `School_norm = (5 - 평균등급) / 4 × 100` | 학교알리미 1–5등급 역수 변환 |
| **보육** | `Childcare_norm = min(어린이집수, 10) / 10 × 100` | 도보 10분(800m) 이내, PostGIS ST_DWithin |
| **치안** | `Safety_norm = (10 - 범죄밀도등급) / 9 × 100 + 보정값` | 생활안전지도 1–10등급 + 보조지표 보정 |

### 최종 점수 계산식

```
Final_Score = W_budget × Budget_norm 
            + W_commute × Commute_norm 
            + W_school × School_norm 
            + W_childcare × Childcare_norm 
            + W_safety × Safety_norm
```

### 균형형 프로필 구체 예시 — A단지

| 요소 | 정규화 점수 | 가중치 | 기여 점수 |
|------|-----------|--------|----------|
| 예산 | 70 | 0.30 | 21.0 |
| 통근 | 55 | 0.25 | 13.75 |
| 학군 | 80 | 0.15 | 12.0 |
| 보육 | 60 | 0.15 | 9.0 |
| 치안 | 75 | 0.15 | 11.25 |
| **합계** | | | **67.0점** |

## 결론: 경쟁 공백은 실재하며, 핵심은 속도다

이 리서치의 가장 중요한 발견은 **"보유현금+연봉 기반 접근 가능 지역 + 통근/보육/치안 통합 추천"이라는 기능 조합이 현존 8개 주요 플랫폼 어디에도 존재하지 않는다**는 점이다. 맞벌이 비중 59.7%, 수도권 신혼부부 51.5만 쌍이라는 수치는 이 공백이 단순한 기능 누락이 아니라 실질적 시장 기회임을 시사한다.

그러나 직방이 "찾아줘 신혼집"으로 이 영역에 진입하기 시작했으며, 대형 플랫폼이 본격 진출하면 트래픽 경쟁에서 불리해진다. **핵심은 3~4주 내 수요 검증을 완료하고, 8~12주 내 MVP를 출시하는 속도**다.

기술적으로는 모든 핵심 데이터가 공공 API로 무료 확보 가능하며, Next.js + PostgreSQL/PostGIS + ODsay API 조합으로 소규모 팀의 빠른 개발이 가능하다. 매물 데이터 실시간성은 가장 큰 기술적 리스크이지만, "실거래가 기반 시세 추정 + 외부 매물 링크" 전략으로 MVP 단계에서 우회할 수 있다.

최종적으로, 이 서비스의 성패는 "추천 결과의 정확성"보다 **"신혼부부가 '여기서 살아도 되겠다'는 확신을 얼마나 빠르게 주는가"**에 달려 있다. 매물 문의 클릭률 5%를 넘기는 것이 그 확신의 첫 번째 증거가 될 것이다.
