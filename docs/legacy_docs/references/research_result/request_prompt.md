역할:
너는 한국 PropTech 도메인 전문 PM 리서처다. 목표는 신혼부부 대상 주거 추천 서비스의 시장성/수요/MVP 타당성을 검증하는 것이다.

제품 컨셉:
- 사용자: 신혼부부
- 지역: 서울 + 경기 + 인천
- 주거유형: 아파트
- 거래유형: 전세 / 매매 (탭 분리)
- 입력값: 보유현금, 합산연봉, 기존대출, 직장1/직장2 주소, 최대 월주거비, (선택) 자차 통근
- 추천기준: 예산, 통근(더 긴 쪽 기준), 학군, 보육(도보 10분), 치안
- 통근 기준: 대중교통 Door-to-Door 60분 이내
- 결과: 단지 Top 10 + 매물(호수) 정보 + 월 고정비 추정 + 추천 이유 문장
- 대단지 라벨: 1000+, 3000+, 5000+
- 핵심 성공지표: 매물 문의 클릭률

요청 작업:
1) 경쟁/대체 서비스 분석
- 직방, 호갱노노, KB부동산, 리치고 포함 최소 8개 비교
- 각 서비스의 강점/약점/미충족 니즈 정리
- “보유현금+연봉 기반 접근 가능 지역 + 통근/보육/치안 통합 추천” 공백 여부 판단

2) 수요 검증 설계
- 신혼부부 대상 인터뷰 가이드 12문항
- 랜딩/설문 실험안 2개
- 수요 검증 임계치(Go/No-Go 기준) 제안

3) 데이터 소스 타당성
- 매물, 실거래가, 통근시간, 학군/보육, 치안 데이터의 출처 후보
- 각 데이터별 커버리지/업데이트 주기/제약/리스크 정리
- 치안은 경찰서 관할 기반 + 보조지표 보정 방식으로 설계안 제시

4) MVP 범위 제안
- P0/P1/P2 기능 분리
- 8~12주 개발 기준 현실적 MVP 스코프
- 제외 권고(Non-goals)와 근거

5) 점수화 모델 초안
- 기본 가중치안 2~3개 제시
- 사용자 가중치 범위 조정 UX 제안
- 최종 점수 계산식 예시 제시

출력 형식:
A. Executive Summary (10줄 이내)
B. 경쟁서비스 비교표 (표)
C. 수요검증 플랜 (인터뷰/설문/랜딩)
D. 데이터 소스 타당성 표
E. MVP 기능 우선순위표 (P0/P1/P2)
F. 리스크 Top 5 + 대응
G. Step 2 종료 시 의사결정 체크리스트

품질 규칙:
- 모든 핵심 주장에 출처 URL + 발행/확인 날짜 포함
- 최신 자료 우선, 한국 시장 기준
- 확실하지 않은 내용은 [가정]으로 명시
- 실행 가능한 수준으로 구체적으로 작성
