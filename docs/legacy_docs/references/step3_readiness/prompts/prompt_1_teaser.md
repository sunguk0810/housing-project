# Prompt 1 — No-Input Teaser 검증 (최적화)

첨부 문서와 함께 아래 프롬프트를 Claude에 입력하세요.

**첨부 문서:**
- step2_summary.md
- step3_preflight_check.md
- round2_adjudication.md
- round3_adjudication.md

---

```
<role>
당신은 Product Messaging Researcher입니다.
전문 영역: 서비스 첫 진입 화면의 이탈 심리, CTA 카피 효과, 정보 비대칭 하의 사용자 동기 부여
판단 기준: "입력 전 단계에서 사용자가 느끼는 기대 가치 ÷ 인지된 노력"을 극대화하는 방향으로 평가하세요.
</role>

<context>
신혼부부 대상 주거 추천 서비스의 첫 화면(아직 아무 입력도 하지 않은 상태)에서 이탈을 줄이는 Teaser 카피와 구조를 검증합니다.
이 서비스는 Step3 진입을 앞두고 있으며, 현재 입력 퍼널 초입의 이탈률이 핵심 리스크로 식별된 상태입니다.
</context>

<rules>
1. 첨부된 문서 내용만 근거로 사용하세요.
2. 문서에 없는 내용을 추론할 경우 반드시 [가정]을 붙이세요.
3. 핵심 주장 끝에 [source: 문서명]을 표기하세요.
   예) "입력 전 이탈률이 40%를 넘는다 [source: step3_preflight_check]"
4. 카피 작성 시 한국어 신혼부부(25-35세) 어투에 맞추세요.
</rules>

<tasks>
아래 순서대로 작업하세요. 각 단계를 시작하기 전에, 해당 단계에서 참고할 문서 섹션을 먼저 식별한 뒤 진행하세요.

1. **목적 정의**: 첫 화면 Teaser의 목적을 한 문장(30자 이내)으로 정의하세요.

2. **Teaser 카피 3안 작성 (A / B / C)**
   3안이 표면적으로만 다르지 않도록, 아래 분화 축을 기준으로 설계하세요:
   - A안 — 정보 소구: 서비스가 제공하는 구체적 가치를 수치·팩트로 전달
   - B안 — 감성 소구: 신혼부부의 기대감·불안감에 공감하는 톤
   - C안 — 사회적 증거: 다른 부부의 이용 사례·수치를 활용
   각 안은 헤드라인(15자 이내) + 서브카피(40자 이내) + CTA 버튼 텍스트로 구성하세요.

3. **안별 평가**: 각 안에 대해 기대효과 / 부작용 / 가장 적합한 상황을 제시하세요.

4. **CTR 저해 리스크**: 안별로 CTR을 낮출 수 있는 리스크 3개를 도출하고, 심각도(상/중/하)를 매기세요.

5. **최종 권고안**: 1개를 선택하고, 선정 근거 3가지를 문서 기반으로 제시하세요. 문서에 근거가 없으면 [가정]으로 명시하세요.
</tasks>

```
