### 1) Executive Summary

서울/수도권 맞벌이 신혼부부 대상 통합 주거추천 서비스는 Step1~Step2(AI Proxy + Red-Team 3라운드)를 모두 `Conditional Pass`로 통과했으며, 치명결함(fatal)은 발견되지 않았다. 그러나 입력 마찰(최소 입력 완료율 15~25%)과 신뢰 붕괴(result_to_inquiry_ctr 3~10%)가 KPI 달성의 구조적 병목으로 확인되었고, 법무 게이트(공인중개사법 자문·위치정보 신고·표시광고 스키마)가 미완료 상태다. [source: step2_summary.md, round2_adjudication.md, round3_adjudication.md, step0_legal_compliance.md]
따라서 Step3 진입은 **조건부(pending)**로 판정하며, 입력/신뢰 전환 방어 구조 고정과 법무 선결과제 일정 확정이 Gate 해제의 전제조건이다.

### 2) Step1~Step2 핵심 요약

- 타깃은 서울/경기/인천 맞벌이 신혼부부이며, 재무+2직장 통근+보육+치안 통합 랭킹이 기존 앱에 없는 핵심 공백이다. [source: step1_define_discovery.md]
- North Star Metric은 추천 결과 노출 대비 `매물 문의 클릭률`이며, 1차 KPI는 컨시어지 문의 CTR ≥ 20%다. [source: step1_define_discovery.md]
- AI Proxy 가상 인터뷰 결과 Pain≥4 비율은 80~85% [가정]이나, 민감정보 조건부 수용은 65~70% [가정]로 경계선이다. [source: step2_market_discovery.md]
- Step2 실험은 AI Proxy only 모드이며, 모든 수치는 가설로서 Human Validation 전까지 확정 불가다. [source: step2_experiment_plan.md]
- Red-Team Round1/2/3 모두 `Conditional Pass / fatal=false / step3_gate=pending`으로 일치한다. [source: round1_adjudication.md, round2_adjudication.md, round3_adjudication.md]
- Round2에서 입력 2단계(직장2/예산)가 구조적 이탈 지점으로, 최소 입력 완료율은 15~25% [가정]까지 하락 가능하다. [source: round2_adjudication.md]
- Round3에서 `result_view → inquiry_click` 구간이 신뢰 관점 1차 병목이며, 신뢰 UI 없이는 전환 복구가 어렵다. [source: round3_adjudication.md]
- 서비스 법적 포지션은 `정보 제공/분석 도구`로 고정했으나, 공인중개사법 외부 자문·위치정보 신고·표시광고 스키마가 미완료다. [source: step0_legal_compliance.md]
- Step3 목표는 "기능 확장"이 아니라 "입력/신뢰 전환 복원" 우선이다. [source: step2_summary.md]
- 컨시어지/리포트 방식이 랜딩 단독보다 보수 시나리오에서 유리하다. [source: step2_market_discovery.md]

### 9) P0 블로커 해결을 위한 업계 사례·벤치마크 리서치

> 본 섹션은 섹션 6의 P0 Must-Do 5건 각각에 대해 업계 사례·벤치마크·정량 데이터를 정리한 것이다.
> 핵심 시사점: "가치를 먼저 보여주고, 민감 정보는 범위로 수집하며, 점수 산출 근거를 투명하게 공개하되, 법적으로는 '정보 제공 서비스'로 명확히 포지셔닝하라."

#### 9-1. 3단 점진 입력 플로우 — 완료율을 2배 이상 끌어올린다

> 대응 P0: `No-Input Teaser → 최소 입력 → 정밀 입력` 3단 플로우 설계 확정

점진적 입력(Progressive Disclosure)은 UX 설계의 핵심 원칙이며, 멀티스텝 폼의 전환율이 싱글페이지 대비 **86% 이상 높아진다**는 HubSpot 연구 결과가 이를 뒷받침한다.

**사례 ① Lemonade Insurance — 90초 견적 플로우.**
가상 에이전트 "Maya"가 한 화면에 한 질문씩 대화형으로 진행한다. 주소 → 자가/임차 → 건물 유형 → 방 수 → 입주일 순으로 묻고, 이메일은 견적이 생성된 **이후** 마지막에 수집한다. Extra Coverage 전환이 **50% 증가**, 출시 6개월 만에 14,000명 가입 달성. 핵심 패턴은 "가치(견적)를 먼저 보여준 뒤 추가 정보를 요청"하는 Value-First 구조.

**사례 ② Zillow — Zero-Input 맵 브라우징.**
랜딩 페이지에서 회원가입 없이 즉시 지도와 매물을 탐색할 수 있다. 가격 핀이 지도 위에 직접 표시되며, 필터는 가격·침실·욕실 3개만 상단에 노출하고 나머지는 점진 공개. **월간 2.45억 순방문자** 규모에서 이 패턴이 동작함을 입증. 관심 매물 저장 시점에 비로소 가입을 유도하는 "Explore → Save → Sign up" 흐름.

**사례 ③ 직방 — 역별 검색 + AI 중개사.**
서울 사용자 특성에 맞춰 지하철역 기반 검색을 제공하며, "역에서 도보 몇 분?"이라는 단일 슬라이더로 최소 입력을 구현. 2023년 도입한 AI 중개사 기능은 필터 설정 없이 채팅만으로 추천 제공. 다만 **이중 통근·보육·치안 통합 점수화는 미제공**하여 우리 서비스의 명확한 차별점.

**벤치마크 수치:**

| 지표 | 수치 | 출처 |
|------|------|------|
| 멀티스텝 폼 vs 싱글스텝 전환 향상 | **+86%** | HubSpot |
| Typeform 스타일(한 화면 한 질문) 완료율 | **47.3%** (업계 평균 2배) | Startup Spells |

| 전화번호 필수 입력 시 이탈 | **37%** | Zuko Analytics |

- **시사점 1**: "통근 스윗스팟 맵"을 티저로 선제공하라. 입력 없이도 서울 인기 신혼부부 거주 지역과 평균 전세가를 지도에 표시하여 즉시 가치를 보여준 뒤, 두 직장 주소 입력만으로 개인화된 통근 최적 구역을 보여주는 2단계로 전환. 재무 정보는 이미 결과에 몰입한 3단계에서 수집하면 보안 우려(29% 이탈) 리스크를 최소화할 수 있다.
- **시사점 2**: "추천 정확도 미터"로 추가 입력을 동기부여하라. LinkedIn 프로필 완성도 바와 유사하게, "추천 정확도: 65% → 더 알려주시면 95%로!"라는 게이미피케이션 요소를 추가하면 사용자가 자발적으로 민감 정보를 제공하는 인센티브가 된다.

#### 9-2. 범위/프리셋 입력 전환 — 프리셋 버튼이 자유 입력 대비 전환율 15~25% 우위

> 대응 P0: 직장2/예산/월상한의 정확값 강제 제거 → 범위/프리셋 + Optional 전환 설계

예산·소득·자산 같은 민감 재무정보 수집에서 **정확한 숫자를 요구하면 인지 부하와 프라이버시 불안이 동시에 증가**한다. Baymard Institute 연구에 따르면 이중 포인트 레인지 슬라이더는 **피험자 50% 이상이 오해**하여 잘못 조작했으며, 대다수 부동산 플랫폼은 슬라이더가 아닌 드롭다운 프리셋 범위를 채택하고 있다.

**사례 ① Zillow — 월 납입액 기준 슬라이더 + 프리셋 드롭다운 병행.**
주요 가격 필터는 $200K/$300K/$400K 단위의 프리셋 드롭다운으로 min/max를 선택하는 구조. 2024년 추가된 월 납입액(PITI 포함) 필터는 $100 단위 슬라이더로, 사용자가 실제 주거 비용 감각으로 탐색 가능. **핵심 교훈: 1차 필터는 프리셋, 탐색용 보조 필터는 슬라이더**라는 이원 구조.

**사례 ② 토스(Toss) — MyData 자동 연동으로 수동 입력 자체를 제거.**
1,910만 활성 사용자를 보유한 토스는 대출 비교 시 소득·자산 정보를 사용자 인증 한 번으로 MyData API에서 자동 수집. 수동 입력이 아예 필요 없어 전환 마찰이 근본적으로 제거. **우리 서비스에 MyData 연동이 불가능한 초기 단계에서는 프리셋 범위가 차선**이며, 향후 MyData 연동을 로드맵에 포함해야 한다.

**사례 ③ 다방 — "쉬운 방찾기" 프리셋 템플릿.**
인기 검색 조건을 사전 조합한 프리셋 시나리오를 제공하여, 시나리오 카드를 탭하면 즉시 결과를 볼 수 있다. **입력 스텝 수를 5~6개에서 1개로 단축**하는 효과.

**벤치마크 수치:**

| 지표 | 수치 | 출처 |
|------|------|------|
| 라디오 버튼 vs 드롭다운 전환율 차이 | **+15%** (라디오 우위) | ConvertCart A/B 테스트 |
| 프로그레스 바 추가 시 완료율 향상 | **+20~30%** | Venture Harbour |
| 인라인 밸리데이션 적용 시 성공률 | **+22%**, 완료 시간 42% 감소 | Luke Wroblewski |
| 체크아웃 최적화 시 평균 전환 향상 | **+35%** | Baymard Institute |
| 레인지 슬라이더 오작동/오해 비율 | **50% 이상** | Baymard Institute |

**우리 서비스 적용 시사점:**

- **시사점 1**: 재무 정보는 4~6개 프리셋 카드 버튼으로 수집하라. 예산(2억 이하/2~4억/4~6억/6~8억/8억 이상), 가구소득(3천만 이하/3천~5천/5천~7천/7천~1억/1억 이상)을 각각 탭 한 번으로 입력. 정확한 금액이 필요한 사용자를 위해 "직접 입력" 링크를 하단에 작게 배치. 프라이버시 부담 감소와 모바일 타이핑 제거라는 이중 효과.
- **시사점 2**: 개인정보보호법 최소수집 원칙과 자연스럽게 정합된다. 소득을 정확한 금액이 아닌 구간으로 수집하면 개인정보보호법 제16조 최소수집 원칙을 원천적으로 충족하면서, 추천 알고리즘에는 충분한 입력을 확보하는 일석이조 효과.

#### 9-3. 신뢰 UI 최소 세트 — 점수 분해와 단점 공개가 신뢰를 28% 이상 끌어올린다

> 대응 P0: 신뢰 UI 최소 세트(점수 분해/출처/갱신일/단점/Why-Not) 결과 템플릿 확정

추천 서비스의 핵심 신뢰 장벽은 **"이 점수가 어떻게 나왔는가?"와 "혹시 내게 불리한 정보를 숨기는 건 아닌가?"**이다. 2025년 SSRN 연구(n=450)에서 설명 가능한 추천(XAI) UI를 적용한 그룹은 비적용 그룹 대비 **신뢰도 28% 향상(4.1 vs 3.2/5점)**, 투명성 인식 **48% 향상(4.3 vs 2.9/5점)**을 기록했다.

**사례 ① Credit Karma — 요인별 영향도 컬러 코딩.**
신용점수를 6개 요인(결제 이력·이용률·계좌 수·경과 기간·조회 횟수·부정적 기록)으로 분해하여 각각 녹색/황색/적색으로 표시. 정확한 모델명과 데이터 출처를 명시하며, 변화 방향(↑↓)까지 표시. **핵심 패턴: 복합 점수 → 요인별 분해 → 출처 명시 → 변화 추적.**

**사례 ② Netflix — "Because You Watched" 맥락 설명.**
추천 행마다 "○○을 시청했으므로", "당신을 위한 Top 픽" 등 맥락 레이블을 부착. 콘텐츠 소비의 **75~80%가 추천 시스템을 통해** 발생. 알고리즘 전체를 공개하지 않되, 인과 관계를 직관적으로 이해할 수 있는 "충분한 수준의 투명성"을 제공하는 것이 핵심.

**사례 ③ 호갱노노 — 20개 이상 공공 데이터 출처 명시.**
국토교통부 실거래가, 행정안전부, 교육부 등 데이터 출처를 명시하고, 거래일·층·면적·가격을 실시간으로 업데이트. **이탈률 21.8%로 경쟁사(직방 38.3%, 다방 54.7%) 대비 최저**를 기록. 커뮤니티 리뷰(소음·주차·안전·학군)가 정량 데이터 위에 정성 정보를 덧씌우는 구조.

**단점 공개가 신뢰를 높이는 근거:**
Eisend(2006)의 메타분석은 **양면 메시지(Two-sided message)가 정보원 신뢰도를 유의미하게 높인다**는 것을 실증했다. 온라인 리뷰 연구에서도 긍정·부정을 모두 포함한 양면 리뷰가 일방적 긍정 리뷰보다 신뢰도가 높게 평가된다. 다만 부정 정보의 비중은 곡선 관계로, 적정 수준을 초과하면 브랜드 태도가 하락한다.

**벤치마크 수치:**

| 지표 | 수치 | 출처 |
|------|------|------|
| XAI 설명 UI 적용 시 신뢰도 향상 | **+28%** (4.1 vs 3.2/5) | Rahman 2025, n=450 |
| 투명성 인식 향상 | **+48%** (4.3 vs 2.9/5) | Rahman 2025, n=450 |
| 개인화 추천을 통한 CTR 향상 | **+38%** | Google News (Das et al.) |
| 개인화 메시지 28일 리텐션 | **61~74%** | Google 퍼스널라이제이션 연구 |
| 양면 메시지의 신뢰도 효과 | 메타분석적으로 유의미 | Eisend 2006 |

**우리 서비스 적용 시사점:**

- **시사점 1**: 레이더 차트 + 요인별 점수 분해를 기본 UI로 제공하라. 종합 점수(예: 87/100) 아래에 💰재무적합도(82)·🚇통근(91)·👶보육(75)·🛡️치안(88) 서브 점수를 바 차트로 표시하고, 각 서브 점수를 탭하면 지역 평균 대비 비교와 데이터 출처·기준일이 펼쳐지는 점진 공개 구조. 출처 배지는 "🏛️ 국토교통부 실거래가 2026.02.10" 형태로 표시.
- **시사점 2**: "고려사항(Cons)" 섹션을 명시적으로 포함하라. 각 추천 아파트에 "✅ 장점: 예산 범위 내, 강남 통근 25분, 우수 초등학교 400m" 옆에 "⚠️ 고려사항: 건축 15년, 주차비율 0.8대, 고층 도로 소음"을 함께 표시. 반박(refutation) 패턴도 활용하여 "건축 15년차이나 2022년 리모델링 완료" 형태로 맥락을 추가.

#### 9-4. 광고/제휴/중개 오인 방지 — 법적 생존 요건이다

> 대응 P0: 광고/제휴/중개 오인 방지 문구 + 외부 링크 정책 확정

2024년 12월 시행된 한국 공정거래위원회의 **추천·보증 심사지침 개정안**은 경제적 관계 공시 의무를 대폭 강화했다. 외국어 표현('AD', 'PR', 'Sponsored')은 금지되고, **반드시 한국어('광고', '유료 광고', '대가성 광고')로 표기**해야 한다. 위반 시 표시광고법에 따라 **2년 이하 징역 또는 1.5억원 이하 벌금**.

**사례 ① Google 검색 — 유료 결과 그룹 헤더 분리(2025년).**
개별 결과마다 작은 "Ad" 태그를 붙이던 방식에서, 유료 결과 전체를 **"Sponsored results" 단일 헤더**로 묶고 스크롤 시에도 고정(sticky)되는 방식으로 전환. "Hide sponsored results" 버튼으로 유료 섹션 자체를 접을 수 있게 하여 유기적 결과와의 구분을 극대화. **핵심 패턴: 개별 태그 → 섹션 단위 분리.**

**사례 ② NerdWallet — 제휴 공시 배너 + 편집 독립성 선언.**
기사 제목 바로 아래, 본문 위에 제휴 공시 배너를 배치. 별도 광고 정책 페이지로 링크하여 전체 제휴 프로그램을 공개하며, 콘텐츠 제작팀과 사업개발팀을 엄격히 분리. CPA(건당 과금) 모델에서 이 투명성이 오히려 전환의 기반.

**사례 ③ 직방/다방 — 플랫폼 광고 모델로 중개 리스크 회피.**
직방은 등록 공인중개사로부터 **광고비**를 수취하는 구조로, 플랫폼 자체가 중개행위를 하지 않는다. 매물 등록·관리는 파트너 중개사가 직접 수행하며, 직방은 부가통신사업자 + 부동산 거래정보사업자로 포지셔닝. **중개수수료가 아닌 광고비·구독료 형태의 수익 구조**가 법적 안전의 핵심.

**외부 링크(아웃링크) UX 패턴:**
미국 USWDS(연방 웹 디자인 시스템)는 모달/인터스티셜 차단 페이지를 명시적으로 반대. 대신 외부 링크 아이콘(↗) + 설명적 링크 텍스트("직방에서 매물 보기 ↗")를 권장하며, 별도 정책 페이지에 면책 사항을 게시하는 방식을 추천.

**우리 서비스 적용 시사점:**

- **시사점 1**: 3중 공시 체계를 구축하라. ①추천 결과 상단에 "본 추천 결과는 광고가 아닌 데이터 분석 기반입니다" 기본 문구 상시 표시, ②향후 제휴/광고 콘텐츠 도입 시 해당 항목에 "광고" 또는 "제휴" 한국어 레이블 + 시각적 분리(배경색·테두리), ③푸터와 이용약관에 "본 서비스는 공인중개사법상 중개업을 영위하지 않습니다" 면책 문구 배치.
- **시사점 2**: 아웃링크는 새 탭 + 인라인 마이크로 공시로 처리하라. "직방에서 매물 보기 ↗ (외부 사이트로 이동합니다)" 형태의 설명적 링크를 사용하고, 모달 차단 없이 새 탭으로 열어 사용자 흐름을 유지.

#### 9-5. 법무 선결과제 — "정보 제공"과 "중개"의 경계를 명확히 긋는 것이다

> 대응 P0: 법무 선결과제 3건(공인중개사법 자문 / 위치정보 신고 / 표시광고 스키마)의 일정·오너·기한 확정

한국 프롭테크 법무의 핵심은 **공인중개사법 제2조의 "중개(알선)" 정의를 넘지 않는 것**이다. 대법원 판례(2005다32197)는 중개행위 여부를 행위자의 주관적 의사가 아닌 **"객관적으로 사회통념상 거래 알선·중개를 위한 행위인지"**로 판단한다. 무등록 중개업은 **3년 이하 징역 또는 3천만원 이하 벌금**(제48조).

**사례 ① 호갱노노 — 순수 정보 분석 서비스 모델.**
국토부 실거래가 등 공공데이터만을 기반으로 시세 분석·일조량·학군 정보를 제공하며, 매물 등록·거래 연결을 일체 하지 않는다. 약 230억원에 직방에 인수될 당시 임직원 18명 중 11명이 개발자일 만큼 기술/데이터 기업으로 포지셔닝. **공공데이터 활용 + 거래 비관여**가 법적 안전 모델의 원형.

**사례 ② 리치고(Richgo) — 이용약관 면책 + 구독 수익 모델.**
이용약관에 "부동산과 관련된 일체 **정보를 제공하는 서비스**"로 명시하고, "참고 용도로 사용"이라는 면책 조항을 둔다. 수익은 프리미엄 구독과 광고로 구성하여 중개수수료 성격의 수익을 원천 배제.

**사례 ③ 트러스트 부동산 사건 — 위반 사례(경고용).**
변호사가 "부동산"이라는 명칭으로 인터넷 사이트를 통해 거래를 알선하여 **벌금 500만원** 유죄 판결(서울고등법원). 공인중개사법 제48조(무등록 중개업), 제49조(유사명칭), 제18조의2(무자격 표시·광고) 복합 위반. **서비스명에 "중개", "부동산중개" 등 명칭을 절대 사용해서는 안 된다.**

**위치기반서비스사업 신고 체크리스트:**

| 항목 | 내용 |

#### 9-6. P0 실행 우선순위 종합

5개 P0 항목은 독립적이지 않다. **프리셋 범위 입력(9-2)은 개인정보보호법 최소수집 원칙(9-5)을 자연스럽게 충족**하고, **점수 분해 투명성 UI(9-3)는 AI 자동화 결정 설명요구권(9-5)과 광고 비광고 구분(9-4)을 동시에 해결**한다. 점진 입력(9-1)의 3단계 구조에서 1단계 "가치 선제공"은 신뢰 UI(9-3)의 데이터 출처·기준일 표시와 결합할 때 가장 강력하다.
