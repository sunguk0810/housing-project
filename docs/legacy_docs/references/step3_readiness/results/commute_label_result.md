# 통근시간 라벨 정책 분석

> **역할:** Mobility UX Research Analyst  
> **판단 기준:** 사용자가 통근시간 정보를 보고 내리는 판단의 정확도(최우선), 정보 이해 속도(차선)  
> **작성일:** 2026-02-13

---

## 1) 현재 리스크

> 단일 수치 표기(예: "강남역 30분")는 사용자에게 앵커링을 유발하여 해당 값을 확정된 사실로 과신하게 만들고, 실제 통근 경험과 괴리가 발생할 때 서비스 전체의 신뢰가 역전될 수 있다. [source: round3_adjudication.md]  
> 이 통근 체감 괴리는 광고/제휴 오인, 근거 불투명과 함께 신뢰 붕괴의 3대 축 중 하나로 확인되었으며, `result_view → inquiry_click` 전환을 직접 저해하는 1차 병목이다. [source: round3_adjudication.md]  
> 현재 통근 표시의 가정(시간대, 교통수단, 오차 범위)이 노출되지 않아 사용자가 정보의 조건부 성격을 인지하지 못하고, 이는 문의 전환율(result_to_inquiry_ctr)의 하방 리스크를 확대한다. [source: step2_summary.md, round3_adjudication.md]

## 2) 라벨 정책 비교

| 안 | 표기 방식 예시 | 장점 | 오해 리스크 | 운영 난이도 |
|---|---|---|---|---|
| **A안 — 범위 표기** | "25~45분" | 불확실성을 직관적으로 전달하여 과신 방지. 가장 단순한 구현. | 범위의 의미(언제 25분이고 언제 45분인지)를 사용자가 스스로 해석해야 함. | 하 |
| **B안 — 기준 시간대 명시** | "출근 8시 기준 40분" | 가정(시간대)을 명시하여 조건부 정보임을 인지시킴. 정보 이해 속도 높음. | 명시된 시간대 외 통근 상황을 과소평가할 수 있음. 단일값 과신 문제가 부분적으로 잔존. | 중 |
| **C안 — 신뢰구간 표기** | "보통 35분, 혼잡 시 50분+" | 대표값과 최악 시나리오를 동시 제공하여 판단 정확도가 가장 높음. 과신과 불안을 동시에 관리. | 정보량 증가로 카드 UI 공간 부담. 문구가 길어 한눈에 읽히지 않을 수 있음. [가정] | 중 |

## 3) 오해 시나리오

### A안

사용자가 "25~45분" 표기를 보고 낙관 편향으로 하한값(25분)을 자기 통근 시간으로 앵커링한 뒤, 실제 출근 시간대에 45분 이상 소요되어 "범위 안에도 못 들어간다"고 느끼며 서비스 신뢰를 상실하는 시나리오. 범위의 조건(시간대, 교통수단)이 없으므로 사용자가 자의적으로 해석할 여지가 크다. [가정]

### B안

"출근 8시 기준 40분"을 본 사용자가 자신의 실제 출근 시간(예: 9시)에도 동일하게 40분이 걸릴 것으로 일반화한 뒤, 9시 출근 시 60분이 소요되어 "거짓 정보"로 인식하는 시나리오. 기준 시간대를 명시했으나 다른 시간대에 대한 정보가 없어 단일값 과신이 시간대만 바뀌어 재현된다. [가정]

### C안

"보통 35분, 혼잡 시 50분+"을 본 사용자가 "보통"이라는 단어를 "대부분의 날"로 해석하고 35분을 기대치로 설정했으나, 실제로는 "보통"이 비혼잡 시간대(오전 10시~오후 3시) 기준이어서 출퇴근 시간대에는 거의 매일 50분 이상이 소요되는 시나리오. "보통"의 정의가 모호하여 대표값 자체가 오해를 유발할 수 있다. [가정]

## 4) 권장 포맷

**포맷: C안 (신뢰구간 표기) — 단, B안의 시간대 명시를 결합**

### 선택 근거

1. Round3 Adjudication에서 통근 표시 전환 방향은 "단일값에서 **범위 + 가정(시간대/오차 포함)**으로 전환"이 P0으로 확정되었다. 이는 단순 범위(A안)만으로는 부족하고, 가정 조건의 명시가 필수임을 의미한다. [source: round3_adjudication.md, Must-Do #3]

2. 신뢰 UI 최소 세트에서 "출처/갱신일/단점" 표시를 전부 P0으로 채택한 방침과 일관되게, 통근 정보에서도 조건(시간대)과 불리한 정보(혼잡 시 소요시간)를 함께 노출해야 서비스 신뢰를 방어할 수 있다. [source: step2_summary.md, Must-Do #3]

3. Step3 Preflight Check에서도 "통근 시간 범위 + 가정 표기 전환"이 P1 항목으로 명시되어 있어, 범위와 가정을 동시에 담는 C+B 결합 포맷이 문서상 요구사항에 가장 부합한다. [source: step3_preflight_check.md, P1 항목]

4. C안 단독의 오해 리스크("보통"의 모호성)는 B안의 시간대 명시를 결합함으로써 완화할 수 있다. [가정]
