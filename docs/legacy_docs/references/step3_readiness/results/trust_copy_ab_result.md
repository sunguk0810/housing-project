## 1) 신뢰 카피 위치 정의

| # | 위치 | 필요 이유 |
|---|------|-----------|
| 1 | **결과 카드 — 점수 표시 영역** | 점수가 블랙박스로 인식되면 `result_view → inquiry_click` 전환이 붕괴된다. [source: round3_adjudication.md] |
| 2 | **결과 상세 — 통근 시간 표시 영역** | 통근 체감 괴리가 핵심 가치 신뢰를 역전시키는 3대 리스크 중 하나다. [source: round3_adjudication.md] |

## 2) A/B 카피 설계

### 위치 1: 결과 카드 — 점수 표시 영역

| 구분 | 카피 |
|------|------|
| **A안 (투명성)** | "이 점수는 통근(40%) · 예산 적합(30%) · 생활 인프라(20%) · 단지 품질(10%) 기준으로 산출되었습니다. 데이터 기준일: 2026-01-15" |
| **B안 (안심)** | "두 분의 조건에 맞춰 여러 항목을 종합한 점수입니다. 자세한 근거는 카드를 눌러 확인하세요." |
| **기대효과** | A안: 근거 투명성으로 '블랙박스' 인식 해소 → 결과 신뢰 ↑ / B안: 인지 부하 최소화로 결과 카드 체류 ↑ |
| **역효과 우려** | A안: 비율 수치가 과잉 정보로 느껴져 카드 스킵 유발 가능 [가정] / B안: 근거 부재로 광고성 콘텐츠 오인 지속 가능 [source: round3_adjudication.md] |

### 위치 2: 결과 상세 — 통근 시간 표시 영역

| 구분 | 카피 |
|------|------|
| **A안 (투명성)** | "출근 시간대(7:30~9:00) 대중교통 기준 35~52분 · 카카오맵/네이버 지도 API 산출 · 실제 체감과 ±10분 내외 차이가 있을 수 있습니다." |
| **B안 (안심)** | "출퇴근 약 35~52분 예상이에요. 실제 경로와 다를 수 있으니 참고용으로 확인해 주세요." |
| **기대효과** | A안: 가정·시간대·오차 명시로 통근 괴리 불만 사전 차단 [source: round3_adjudication.md] / B안: 부드러운 톤으로 수치 거부감 완화 |
| **역효과 우려** | A안: 텍스트 길이가 모바일에서 가독성을 해칠 수 있음 [가정] / B안: 출처 미표기로 수치 자체에 대한 의심 잔존 가능 |

### 위치 3: 문의 CTA 직전 영역

| 구분 | 카피 |
|------|------|
| **A안 (투명성)** | "외부 매물 플랫폼으로 이동합니다. 본 서비스는 중개·알선을 하지 않으며, 연락처는 수집하지 않습니다." |
| **B안 (안심)** | "전화 없이 매물을 직접 확인할 수 있어요. 외부 사이트로 이동합니다." |
| **기대효과** | A안: 중개 오인 + 연락처 공포 동시 해소 [source: step0_legal_compliance.md, round2_adjudication.md] / B안: '전화 없음' 강조로 문의 공포 직접 완화 |
| **역효과 우려** | A안: "중개·알선을 하지 않습니다"가 과잉 방어로 느껴져 오히려 의심 유발 가능 [가정] / B안: 법적 고지 요소(외부 이동 명시)가 약해 컴플라이언스 리스크 잔존 [source: step0_legal_compliance.md] |

### 위치 4: 입력 폼 상단 — 개인정보 수집 안내 영역

| 구분 | 카피 |
|------|------|
| **A안 (투명성)** | "입력하신 직장 권역과 예산 범위는 추천 계산에만 사용되며, 결과 확인 후 즉시 삭제됩니다. 연락처는 수집하지 않습니다." |
| **B안 (안심)** | "입력 정보는 저장하지 않아요. 안심하고 조건을 입력해 주세요." |
| **기대효과** | A안: 처리 목적·보관기간·파기를 구체적으로 명시해 데이터 불안 해소 [source: step0_legal_compliance.md] / B안: 최소 텍스트로 입력 진입 마찰 최소화 [source: round2_adjudication.md] |
| **역효과 우려** | A안: "즉시 삭제"가 서비스 미성숙 인상을 줄 수 있음 [가정] / B안: 법적 근거(동의 분리, 보관기간) 미충족 시 컴플라이언스 위반 가능 [source: step0_legal_compliance.md] |

### 위치 5: 랜딩 하단 — 서비스 포지션/면책 고정 영역

| 구분 | 카피 |
|------|------|
| **A안 (투명성)** | "본 서비스는 공공 데이터와 허가된 API를 기반으로 정보를 제공합니다. 부동산 중개·알선·자문 행위를 하지 않으며, 결과는 거래 성사를 보장하지 않습니다." |
| **B안 (안심)** | "집 찾기의 첫 단계를 도와드려요. 추천 결과는 참고용 정보이며, 최종 결정은 전문가와 함께하시길 권합니다." |
| **기대효과** | A안: 법적 포지셔닝 3대 금지(중개/알선/자문) 명확화 [source: step0_legal_compliance.md] / B안: 면책 톤을 부드럽게 전환해 랜딩 이탈 방지 |
| **역효과 우려** | A안: 법률 용어("중개·알선·자문")가 일반 사용자에게 위화감 유발 가능 [가정] / B안: 공인중개사법 관점에서 포지셔닝이 불충분할 수 있음 [source: step0_legal_compliance.md] |

## 3) 실험 KPI

| KPI | 측정 방법 | 기대 방향 |
|-----|-----------|-----------|
| **result_to_inquiry_ctr** | `inquiry_click / result_view × 100` · Step3 게이트 기준: ≥8% go, 3~8% pending, <3% no-go [source: step3_preflight_check.md] | ↑ (신뢰 카피 적용 시 결과→문의 전환율 상승 기대) |
| **입력 완료율 (min_input_completion)** | `min_complete / input_start × 100` · 스트레스 기준: 교집합 15~25% [source: round2_adjudication.md] | ↑ (입력 폼 안심 문구로 이탈 감소 기대) |
| **점수 근거 확인율** | `score_detail_expand / result_view × 100` · 결과 카드에서 점수 분해를 펼쳐본 비율 [가정] | ↑ (투명성 카피가 근거 탐색 행동을 유도하는지 확인) |
| **CTA 직전 이탈률** | `(cta_modal_view − inquiry_click) / cta_modal_view × 100` · 외부 이동 고지 모달에서 이탈한 비율 [가정] | ↓ (CTA 안심 카피가 이탈을 줄이는지 확인) |
| **VOC 신뢰 키워드 비율** | 문의 후 설문 또는 VOC에서 "광고", "믿을 수 없음", "영업" 등 불신 키워드 출현율 [가정] | ↓ (오인 방지 카피가 부정 인식을 줄이는지 확인) |

## 4) 역효과 시나리오 및 완화책

| 위치 | 역효과 시나리오 | 완화책 |
|------|----------------|--------|
| **1. 결과 카드 점수** | A안의 비율(40%/30%/20%/10%)이 과잉 정보로 느껴져 카드를 스킵하거나, 특정 비율에 대한 논쟁("왜 통근이 40%냐")이 발생한다. [가정] | 카드 표면에는 "통근 · 예산 · 생활 · 단지 종합 점수"만 표시하고, 비율은 펼침(expand) 영역으로 이동. 정보 밀도를 레이어로 분산한다. |
| **2. 통근 시간** | 범위 표기(35~52분)가 "정확하지 않다"는 인상을 주거나, 오차 고지(±10분)가 서비스 품질 불신으로 이어진다. [가정] | 범위의 하한을 대표값으로 강조하고, 상한과 오차는 보조 텍스트로 처리. "출근 시간대 기준 약 35분~" 형태로 핵심 수치를 먼저 노출한다. |
| **3. 문의 CTA** | "중개·알선을 하지 않습니다"라는 부정형 문구가 오히려 "그런 서비스가 있나?"라는 의심을 유발한다(Streisand 효과). [가정] | 부정형("하지 않습니다")을 긍정형("외부 매물 사이트에서 직접 확인")으로 전환. 법적 필수 고지는 모달 하단 작은 글씨로 분리 배치한다. |
| **4. 입력 폼** | "즉시 삭제됩니다"가 반복 방문 시 재입력 부담으로 이어지거나, 서비스의 데이터 관리 역량에 대한 의문을 유발한다. [가정] | "이번 추천에만 사용되고 저장하지 않습니다"로 표현을 전환하고, 재방문 시 빠른 재입력을 위한 프리셋/범위 선택 UI를 보완한다. [source: round2_adjudication.md] |
| **5. 랜딩 면책** | 법률 용어가 밀집된 면책 문구가 랜딩 첫인상을 무겁게 만들어 이탈률을 높인다. 특히 모바일에서 면책 텍스트가 화면 상당 부분을 차지할 수 있다. [가정] | 랜딩 본문에는 B안 수준의 부드러운 한 줄 요약만 노출하고, 전체 법적 고지는 "자세히 보기" 링크로 분리. 법적 필수 요소는 별도 페이지(정책 문서)에서 충족한다. [source: step0_legal_compliance.md] |

## 5) P0 카피 선정 (Step3 즉시 반영)

선정 기준: **법적 필수 > 신뢰 임팩트 大 > 구현 난이도 低**

| 순위 | 위치 | 카피 (권장안) | 선정 근거 |
|------|------|--------------|-----------|
| 1 | **문의 CTA 직전** | "외부 매물 사이트로 이동합니다. 본 서비스는 부동산 중개·알선을 하지 않습니다." | **법적 필수.** 공인중개사법 오인 방지 + 외부 이동 고지가 Step0 P0 요구사항이다. [source: step0_legal_compliance.md §1, §8] |
| 2 | **랜딩 하단 면책** | "본 서비스는 참고용 정보 제공이며, 거래·승인을 보장하지 않습니다." | **법적 필수.** Claim Governance 권장 문구이자 금지 문구 위반 방지의 최소 요건이다. [source: step0_legal_compliance.md §7] |
| 3 | **결과 카드 점수** | "통근 · 예산 · 생활 · 단지 종합 점수 | 출처: 공공데이터·지도API | 기준일: YYYY-MM-DD" | **신뢰 임팩트 大.** Round3에서 '블랙박스' 인식이 result_to_inquiry_ctr 붕괴의 1차 원인으로 확정되었다. [source: round3_adjudication.md §2] |
| 4 | **입력 폼 상단** | "입력 정보는 이번 추천에만 사용되며, 별도 저장하지 않습니다." | **신뢰 임팩트 大 + 구현 난이도 低.** 입력 2단계 이탈이 Round2 핵심 병목이며, 텍스트 한 줄 추가로 즉시 적용 가능하다. [source: round2_adjudication.md §2, step0_legal_compliance.md §3] |
| 5 | **통근 시간 표시** | "출근 시간대(7:30~9:00) 대중교통 기준 약 35~52분 · 실제와 차이가 있을 수 있습니다." | **신뢰 임팩트 大.** 통근 체감 괴리가 Round3 3대 리스크 중 하나이며, 범위+가정 전환은 P0 Must-Do다. [source: round3_adjudication.md §6] |

## 부록: 실험 운영 가이드

### 실험 설계 권장사항 [가정]

- **표본 기준**: result_view ≥ 200 도달 후 판정 (Step3 게이트 최소 표본과 동일) [source: step3_preflight_check.md §2]
- **실험 단위**: 사용자 단위 할당 (세션 단위 시 동일 사용자가 A/B 모두 경험할 수 있어 오염 우려)
- **실험 기간**: 최소 1주, 요일별 편차 안정화를 위해 2주 권장 [가정]
- **법적 필수 카피(순위 1, 2)는 A/B 대상에서 제외**: 법적 고지는 실험 없이 전수 적용하고, 표현 톤만 A/B로 비교

### 카피 거버넌스 연동

- 신규 카피는 Step0 §7 금지 문구 목록과 교차 검증 후 배포 [source: step0_legal_compliance.md §7]
- "보장", "확정", "최적" 등 단정형 표현이 포함되지 않았는지 배포 전 체크리스트로 확인
- 법무 윈도우(2026-02-13 ~ 2026-02-27) 내 외부 자문 결과에 따라 CTA 문구 재조정 가능 [source: step0_legal_compliance.md §9.1]
