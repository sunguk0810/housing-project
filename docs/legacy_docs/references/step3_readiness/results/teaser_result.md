## 2) Teaser 카피 3안

### A안 — 정보 소구

- **헤드라인:** 출퇴근 30분, 월세 70만
- **서브카피:** 직장 두 곳과 예산만 넣으면, 조건에 맞는 동네를 찾아드려요
- **CTA:** 어떤 동네가 나올까?

설계 의도: 서비스의 핵심 출력물(통근 시간 + 비용 기반 동네 추천)을 구체 수치로 미리 보여줘 "이 서비스가 뭘 해주는지"를 즉시 인지시킨다. Round3에서 통근 체감 괴리가 신뢰 붕괴 원인으로 지목되었으므로 [source: round3_adjudication], 첫 화면부터 통근·비용이라는 핵심 축을 노출해 기대값을 정확히 세팅한다.

### B안 — 감성 소구

- **헤드라인:** 첫 집, 어디서 시작할까
- **서브카피:** 예산도 출퇴근도 다 걱정인 두 사람을 위해 만들었어요
- **CTA:** 우리 조건으로 찾아보기

설계 의도: 신혼부부의 불안(예산 부담, 통근 절충)에 직접 공감하되, "두 사람"이라는 표현으로 커플 전용 서비스임을 암시한다. Round2에서 입력 공포(민감정보/예산 노출)가 핵심 이탈 원인으로 확인되었으므로 [source: round2_adjudication], CTA를 "우리 조건"이라는 부드러운 표현으로 입력 부담을 완화한다.

### C안 — 사회적 증거

- **헤드라인:** 1,200쌍이 찾은 우리 동네
- **서브카피:** 비슷한 조건의 신혼부부들이 어떤 동네를 선택했는지 확인하세요
- **CTA:** 다른 부부 결과 보기

설계 의도: 타인의 이용 수치를 보여줘 서비스 신뢰도를 선제적으로 확보한다. Round3에서 광고/제휴/중개 오인이 신뢰 붕괴의 핵심 원인으로 지목되었으므로 [source: round3_adjudication], "다른 부부들이 실제 사용했다"는 사회적 증거로 상업적 의심을 우회한다. 단, 1,200쌍은 Human Validation 전 가설 수치이므로 [가정] 실제 운영 시 검증된 수치로 교체해야 한다.

## 3) 안별 평가

| 안 | 기대효과 | 부작용 | 적합 상황 |
|---|---|---|---|
| **A — 정보** | 서비스 출력물이 명확해 "입력하면 뭘 얻는지"가 즉시 전달됨. 입력 전 기대가치/인지노력 비율이 가장 높을 수 있음 [가정] | 통근 30분·월세 70만이라는 예시 수치가 본인 조건과 다를 경우 "내 얘기가 아니다"로 이탈할 수 있음. Round3에서 통근 단일값 표기가 신뢰 리스크로 지적된 맥락과 동일한 위험 [source: round3_adjudication] | 서비스 개념이 생소한 초기 유입(검색/광고 랜딩) 상황 |
| **B — 감성** | 입력 전 심리적 안전감을 확보해 "민감정보 입력 공포"를 낮춤. Round2에서 확인된 입력 마찰 완화에 직접 대응 [source: round2_adjudication] | 서비스가 구체적으로 뭘 해주는지 전달이 약해, 기대가치 인지가 늦어질 수 있음 [가정] | 커뮤니티·SNS 등 맥락이 이미 공유된 유입 경로 |
| **C — 사회적 증거** | 광고/제휴 오인 리스크를 "실사용자 데이터"로 우회. Round3 신뢰 붕괴 대응에 가장 직접적 [source: round3_adjudication] | 수치의 진위를 의심받으면 오히려 신뢰가 역전됨. Step2 수치가 가설 상태라 검증 전 사용 시 리스크 큼 [source: step2_summary §보류] | 충분한 실사용 데이터가 축적된 이후 단계 |

## 5) 권고안

- **선택:** A안 (정보 소구) — 단, 수치 표현을 범위형으로 수정

### 근거 1

Step2 Must-Do #1에서 확정된 3단 플로우의 첫 단계(No-Input Teaser)는 "입력 없이 가치를 체감"시키는 것이 목적이다. A안은 서비스 출력물(통근 시간 + 비용 기반 동네 추천)을 가장 직접적으로 보여줘, 입력 전 기대가치를 가장 빠르게 형성한다. [source: step2_summary §5 Must-Do #1]

### 근거 2

Round3에서 신뢰 붕괴의 핵심 원인이 "근거 투명성 부재"로 합의되었다. A안의 정보 소구 구조는 첫 화면부터 "이 서비스가 무엇을 계산해주는지"를 명시함으로써, 블랙박스 인식을 선제적으로 완화하는 방향과 일치한다. [source: round3_adjudication §2 Consensus #1, #2]

### 근거 3

Round2에서 입력 마찰의 가장 큰 원인은 "입력의 대가로 무엇을 얻는지 불확실"한 상태였다. B안(감성)은 기능 전달이 약하고, C안(사회적 증거)은 실측 데이터 부재(Step2 수치가 가설 상태 [source: step2_summary §보류])로 현 시점에서 사용이 부적절하다. A안이 "입력 대비 기대 보상"을 가장 명확히 전달한다. [가정 — Round2 문서에서 입력 대가 불확실성이 직접 언급되지는 않았으나, 입력 완료율 15~25%라는 낮은 스트레스 기준이 이를 시사함] [source: round2_adjudication §4]

### 적용 시 수정 사항

- 헤드라인의 단일 수치("30분, 70만")를 범위형("20~40분대, 50~80만원대")으로 전환 — Round3 Must-Do #3 준수 [source: round3_adjudication]
- CTA 하단에 "광고·중개가 아닌 독립 추천입니다" 고지 삽입 — Preflight Q4(3중 고지) 준수 [source: step3_preflight_check]
- 입력 분량 암시를 제거하고 "조건 2개만 넣으면" 등 최소 노력 메시지로 교체 — Round2 Must-Do #2(정확값 강제 제거) 취지 반영 [source: round2_adjudication]
