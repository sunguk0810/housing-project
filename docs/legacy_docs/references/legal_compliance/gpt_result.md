> 아래 매핑은 사용자가 제공한 1~12번 답변을 “직전 Q&A 질문 흐름”에 맞춰 정리한 것입니다.  
> 만약 질문 의미가 달랐다면 해당 항목만 수정하면 됩니다. **일부는 [가정]**으로 해석했습니다.

1. (중개/협상/계약 관여 여부) **아니오**  
2. (매물 문의/상담 처리) **외부(중개업소)로 넘김**  
3. (수익모델) **성과 수수료(성공보수/CPA 성격) 수취**  
4. (책임 범위) **면책/책임제한 조항을 두고 싶음**  
5. (개인정보 보관) **세션 계산 후 폐기** *(단, “폐기하지 않는 경우”도 분석 요청)*  
6. (외부 전달 데이터 형태) **가공 후 전달**  
7. (국외 이전) **없음**, AWS **ap-northeast-2(서울)** 리전 사용 예정  
8. (동의/정책 적용 방식) **자동으로 적용** → **[가정]**: 선택동의(마케팅/제3자 제공 등)를 *기본 ON* 하려는 의사로 해석(⚠️ 법적으로 매우 위험)  
9. (민감정보/고유식별정보 수집) **없음**  
10. (마케팅 메시지 발송) **있음**  
11. (GPS/위치기반 기능) **되면 하고 싶음**  
12. (자동화된 추천/점수화/AI 적용) **있음**

## 3) Q5(“세션 계산 후 폐기”) 구현안 + “폐기하지 않는 경우” 리스크/대안

### 3-1. 기본안: 세션 계산 후 폐기(추천)
**정의**
- 사용자 입력값을 서버에서 계산 후 **즉시 파기**  
- 저장하는 것은 “익명/집계 통계(예: 예산 구간, 지역 클릭)” 정도로 제한

**근거**
- 목적 달성 후 파기 의무(개인정보 보호법 제21조)  
  - 출처/확인: 국가법령정보센터, 개인정보 보호법 제21조 [시행 2025.10.2], 확인일 2026-02-13  
    https://www.law.go.kr/LSW/lsLinkCommonInfo.do?ancYnChk=&chrClsCd=010202&lsJoLnkSeq=1020398651

**구현 Task**
- “세션 데이터”는 메모리/캐시(짧은 TTL)만 사용  
- 결과 공유가 필요하면:
  - (옵션 A) 결과를 **개인정보 제거 후** “결과 토큰/링크”로 저장(예: 추천 단지 ID 리스트 + 점수만)  
  - (옵션 B) 사용자가 “저장”을 누르면 별도 동의/로그인 후 저장(아래 3-2로 이동)

---

### 3-2. 예외안: 세션/프로필을 저장하는 경우(리스크 증가)
**추가로 필요한 것**
- 보유기간, 파기 절차, 권리행사(열람/삭제/처리정지) 대응 체계가 실질적으로 운영돼야 함(제30, 제38, 제21).  
  - 처리방침: 제30조 / 권리행사: 제38조 / 파기: 제21조 (각 출처는 상단 참고)

**저장 시 권장 설계(리스크 완화)**
- 저장 데이터 최소화: 소득/대출은 “정확값” 대신 **구간값(예: 0~5천, 5천~8천)**  
- 직장 주소는 “상세주소” 대신 **역/행정동 수준**  
- 디바이스 식별자/광고ID는 최소화(수집 목적·근거 명확화)  
- 사용자에게 “삭제” 버튼 제공 + 즉시 처리(배치 파기 아님)

## 4) 성과 수수료(3번) 모델 — 공인중개사법 리스크 관점 구현 가이드

### 4-1. 핵심 리스크
- 성공보수가 “거래 성사에 대한 대가”로 보이면, 사실관계에 따라 **중개 관여**로 오해될 수 있습니다. **[가정]**  
- 또한 중개업소 입장에서는 플랫폼 비용을 소비자에게 전가하면 “중개보수 한도” 민원으로 번질 수 있습니다.

### 4-2. 권장 구조(리스크 완화 순)
1) **광고비/리드비(CPL)**: “문의 전환(클릭/폼 제출)” 기준 정산(거래 성사와 분리)  
2) **월 정액 + 성과 인센티브(상한)**: 성과를 쓰더라도 상한을 두고 관여를 금지  
3) (가능하면 지양) **거래 성사 기반 CPA**: 가장 리스크가 큼

### 4-3. 계약/운영 통제
- “플랫폼은 중개/협상/계약에 관여하지 않는다” 조항  
- 성과 측정 지표를 “클릭/리드”로 한정  
- 중개업소가 사용자에게 별도 비용을 청구하지 않도록(권고)  
- 고객 민원 시 책임주체(중개업소) 명확화
