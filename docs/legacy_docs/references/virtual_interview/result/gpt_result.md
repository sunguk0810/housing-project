    - 리포트 문의 CTR: 리포트(결과) 화면 기준
- *주의*: 분모 정의가 성과를 크게 바꾸므로 Step2 실험에서 반드시 고정/공유.

#### 보수적 단일 KPI 가정치(충돌 해결 후)

#### 전세 vs 매매(보수적 운영 가정)  *(전부 [가정])*
- **전세 탭**: 유입/관심은 높을 가능성 큼. 다만 **단순 리스트만으로 문의 전환이 높다고 가정하지 않음**(리포트/안전진단 제공 시 상승 가능). **[실검증 필요]**
- **매매 탭**: 입력/검토 변수가 많아 초기 퍼널(입력 완료/문의)에서 이탈 위험이 높다고 가정. **[실검증 필요]**

### C-3) 최종 Go/No-Go 판정 조건(“Step2에서 실제로 판단할 기준”)

> 아래 조건은 **가상 리서치 기반으로 정한 운영 기준**이며, **반드시 실제 트래픽/사용자 테스트로 판정**합니다. **[실검증 필요]**

#### 1) Go (진행/확대)
- (G1) **실제 인터뷰/테스트**에서 Pain≥4 비중 **≥70%**(예: 10쌍 중 7쌍 이상) 확인 **[실검증 필요]**
- (G2) **최소 입력(직장1/2 권역 + 월상한 + 예산 범위)** 완료율 **≥50%** **[실검증 필요]**
- (G3) KPI(둘 중 하나 충족) **[실검증 필요]**
    - **리포트 문의 CTR ≥20%** (기본 목표)
    - 또는 **랜딩 문의 CTR ≥12%** (난이도 높음, 달성 시 강한 신호)

#### 2) Pivot (방향 유지하되 설계 변경)
- (P1) 리포트 문의 CTR이 **15~19%** 구간에 머무르거나,
- (P2) 최소 입력 완료율이 **30~49%**로 낮아 퍼널이 무너지나,
- (P3) 정성 반응에서 “가치/신뢰는 있으나 입력이 부담”이 반복될 때  
  → **입력 단계를 더 뒤로(점진 입력 강화), 무입력 데모/티저 확대, ‘전화 없는 문의’ 옵션 강화**로 피벗 후 재실험.

#### 3) No-Go (중단 또는 대피벗)
- (N1) 실제 인터뷰에서 Pain≥4 비중이 **<50%**로 확인되거나, **[실검증 필요]**
- (N2) 리포트 문의 CTR이 **<15%**로 반복되고(충분한 트래픽/표본에서), 개선 레버가 확인되지 않을 때 **[실검증 필요]**

### C-4) 인터뷰 대체(가상) 리서치의 한계 & 보완 계획

#### 한계(구조적으로 피할 수 없음)
- **의향(말)과 행동(클릭/입력)은 다를 수 있음** → CTR/입력완료율은 실제 데이터가 필요. **[실검증 필요]**
- **민감정보 입력 저항은 UI/카피/신뢰 설계에 따라 급변** → 화면에서의 실제 이탈을 관찰해야 함. **[실검증 필요]**
- **통근 시간은 ‘계산값’과 ‘체감’이 다름**(환승, 혼잡, 도보) → 실제 사용자 불만이 발생할 수 있음. **[실검증 필요]**
- **치안/학군/보육 지표는 정의·출처·낙인 논란** → 설계/표현 방식 검증 필요. **[실검증 필요]**

#### 보완 계획(최소 실행 세트)
- 실제 맞벌이 신혼부부 **심층 인터뷰(최소 10쌍)** + **프로토타입 테스트(최소 5쌍)**로
    - 입력 거부 포인트
    - 결과 신뢰 포인트
    - 문의 트리거 문구/형태  
      를 확인한다. **[실검증 필요]**
- 랜딩 A/B로 **CTA(“리포트 신청” vs “바로 문의”) / 입력 방식(범위 vs 정확값)**을 검증한다. **[실검증 필요]**

|2) 추천 결과 신뢰 부족(광고/제휴 의심)|문의 CTR↓, 브랜드 신뢰 훼손|“왜 이 단지?” “광고 같음” 피드백|점수 분해/근거/출처/갱신일/한계 + **단점(Bad news)** 포함, 제휴/비제휴 구분 고지|
|3) 통근 계산 정확도/체감 괴리|불만/이탈, 추천 가치 붕괴|“실제로는 더 걸림/환승이 힘듦” 반복|문앞→문앞 가정 명시, 시간대 가정, “체감 오차 가능” 고지 + 사용자 피드백 수집|
|4) 아웃링크 품질 문제(만료/허위/중복)|사용자 피로, 신뢰 하락|링크 클릭 후 되돌아옴/불만|중복 제거, 신선도 표시, 링크 소스 다변화, “링크 품질 신고” 버튼|
|5) 법/규제 리스크(중개·자문, 개인정보, 금융정보)|서비스 중단/분쟁 가능|문구/기능이 ‘중개’로 해석될 소지|면책/표현 가이드, 개인정보 최소수집, 민감정보 옵션화, 법무/컴플라이언스 체크리스트|
