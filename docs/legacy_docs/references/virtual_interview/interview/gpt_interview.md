# 신혼부부 맞춤 주거 추천 — 가상 인터뷰 기반 리서치(시뮬레이션)

⚠️ **필독 / 명시(요청사항 준수)**  
아래 결과물은 **“실제 인터뷰를 대체하기 위한 가상 리서치(시뮬레이션)”** 입니다.
- **실제 사용자 검증이 아니며**, 실제 행동 데이터/시장 데이터 기반이 아닙니다.
- 따라서 이 문서의 인사이트/수치/판정은 **의사결정 참고용 가정**일 뿐이고, 실제 출시/투자/리소스 투입 전에는 **실제 인터뷰·사용성 테스트·랜딩 A/B**가 반드시 필요합니다.

**확정 사실(사용자 제공)**
- 서비스: 신혼부부 맞춤 주거 추천
- 대상: 서울/경기/인천 맞벌이 신혼부부
- 범위: 아파트, 전세/매매 탭 분리
- 핵심 입력: 보유현금, 가능 예산, 월 상한, 직장1/직장2
- 핵심 조건: 2직장 중 긴 통근 기준 60분 이내(대중교통 우선)
- 점수 요소: 예산/통근/보육/치안/학군
- 출력: Top10 단지 추천 + 추천 사유 + 외부 매물 아웃링크
- Step2 핵심 KPI: 문의 클릭률

**가정**
- 아래 **20명 인터뷰 응답/수치/행동 가능성/CTR 추정은 전부 [가정]** 입니다. (가상의 페르소나/가상의 발화)

## A. Executive Summary (10줄 이내)

1) [가정] **가상 인터뷰 20명 중 17명**이 주거 탐색 Pain **4~5점(>=4)**으로 높음.
2) [가정] 민감정보(현금/예산/직장) 입력은 **조건부 수용(>=6/10)**이 **14/20명(70%)**으로 과반.
3) [가정] 반복 Pain Top은 “**두 직장 통근 최적화 + 예산/전세리스크 + 신뢰 가능한 근거 부족**”에 집중.
4) [가정] 랜딩(바로 추천/문의 CTA) 문의 클릭률은 **보수 7~10% / 기준 11~14% / 낙관 15~18%**로 추정.
5) [가정] 컨시어지 리포트(개인화 리포트 제공 후 문의 CTA) 문의 클릭률은 **보수 18~22% / 기준 23~28% / 낙관 29~35%**로 추정.
6) [가정] **전세는 유입은 크고**(탐색층 많음) **매매는 문의 전환이 높음**(의사결정 임박층).
7) 결론: **Go** (단, 초기엔 **컨시어지 리포트 중심**으로 신뢰/전환을 확보하는 방향이 유리).
8) 성공의 핵심 레버: **입력 장벽 최소화(점진 입력) + 결과 근거 투명화(점수 분해/출처/갱신일)**.
9) 리스크 Top: 데이터 신뢰/개인정보/통근 정확도/아웃링크 품질/추천 편향.
10) 다음 2주: “**입력→추천→근거→문의**” 퍼널을 가장 얇게 만들고 계측부터.

## C. 패턴 분석 요약 *(전부 [가정])*

### 1) 반복 Pain Point Top 5
1. **두 직장 통근 최적화의 어려움** (특히 “중간지점” 찾기)
2. **예산과 월 상한을 동시에 만족시키는 후보군이 좁음**
3. **보육/치안/학군을 ‘단지 단위’로 비교하기 어려움**
4. **매물 정보 과잉 + 중복/만료/낚시성 매물 피로**
5. **중개사 접촉 불신/피로(전화, 푸시, 압박)**

### 2) 기존 대체행동(앱/엑셀/중개사) 패턴
- **앱(지도/필터)**: 가격·평형 필터 → 단지 북마크 → 다른 앱에서 재확인(중복)
- **지도앱 통근 시뮬레이션**: 두 직장 경로를 각각 저장 → 중간역 후보 수작업 추림
- **엑셀/노션 가중치표**: 개인 기준 점수화(하지만 지표 수집이 어려움)
- **중개사 병행**: 막판 “현장 느낌” 검증용으로 1~2곳만 쓰려는 경향

### 3) 입력 장벽 원인 Top 3
1. **민감정보 입력 = 스팸 연락/유출 공포**
2. **입력 대비 얻는 가치 불명확**(왜 필요한지/얼마나 좋아지는지)
3. **입력 난이도·시간 비용 + 결과 정확도 불신**

### 4) 신뢰를 높이는 결과 표현 방식 Top 3
1. **점수 분해 표시 + 가중치 조절**
    - 예: 총점 87 = 예산 30/30 + 통근 25/30 + 보육 18/20 + 치안 8/10 + 학군 6/10
2. **통근 근거의 구체화(문앞→문앞, 시간대 가정, 환승/도보 포함)**
3. **데이터 출처/갱신일/한계 명시 + ‘왜 제외됐는지(Why not)’**

|문의 클릭률(리포트)|전세 20~26%|매매 24~32%|리포트가 결정 근거로 작동|

## E. Go/No-Go 판정 (현재 가정 기준)

### Go 기준 재검증(가상 데이터 기반)
- (a) **20명 중 14명 이상 pain>=4**
    - [가정] **17/20명** Pain 4~5 → ✅ 충족
- (b) **50% 이상 민감정보 입력 조건부 수용**
    - [가정] “수용도 ≥ 6/10”으로 정의 시 **14/20명(70%)** → ✅ 충족
- (c) **문의 클릭률 >=12% 또는 컨시어지 문의 클릭률 >=20%**
    - [가정] 랜딩 기준 11~14%(경계) / 컨시어지 기준 23~28% → ✅ 충족(컨시어지 확실)

### 판정: **Go**
- 근거: Pain 강도/입력 수용/전환 지표(컨시어지)가 **Go 기준을 충족**하는 가정 결과.
- 단, 초기엔 **“셀프서브 랜딩”보다 “컨시어지 리포트 + 근거 강화”**가 더 안정적인 전환을 만들 가능성이 큼(가정).
