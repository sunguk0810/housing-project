역할:
너는 한국 PropTech PM 리서처다.
주의: 이 작업은 "실제 인터뷰 대체용 가상 리서치"이며, 실제 사용자 검증이 아님을 명확히 표시하라.

프로젝트 배경:
- 서비스: 신혼부부 맞춤 주거 추천
- 대상: 서울/경기/인천 맞벌이 신혼부부
- 범위: 아파트, 전세/매매 탭 분리
- 핵심 입력: 보유현금, 가능 예산, 월 상한, 직장1/직장2
- 핵심 조건: 2직장 중 긴 통근 기준 60분 이내(대중교통 우선)
- 점수 요소: 예산/통근/보육/치안/학군
- 출력: Top10 단지 추천 + 추천 사유 + 외부 매물 아웃링크
- Step2 핵심 KPI: 문의 클릭률

요청 작업:
1) 가상 인터뷰 20명 생성
- 각 인터뷰 대상의 간단 페르소나
- 12문항 핵심 답변 요약
- Pain score(1~5), 민감정보 입력 수용도(0~10), 문의 행동 가능성(0~100%)

2) 인터뷰 통합 분석
- 반복 pain point Top 5
- 기존 대체행동(앱/엑셀/중개사) 패턴
- 입력 장벽 원인 Top 3
- 신뢰를 높이는 결과 표현 방식 Top 3

3) 가상 실험 결과 추정
- 랜딩 실험 문의 클릭률 추정(보수/기준/낙관 3시나리오)
- 컨시어지 리포트 문의 클릭률 추정(보수/기준/낙관)
- 전세 탭 vs 매매 탭 반응 차이

4) Go/No-Go 판정(현재 가정 기준)
- Go 기준:
  a. 20명 중 14명 이상 pain>=4
  b. 50% 이상 민감정보 입력 조건부 수용
  c. 문의 클릭률 >=12% 또는 컨시어지 문의 클릭률 >=20%
- 위 기준으로 Go/No-Go/Pivot 중 하나를 선택하고 근거 제시

5) 리스크와 대응
- Top 5 리스크
- 각 리스크별 P0 대응 1개

출력 형식:
A. Executive Summary (10줄 이내)
B. 가상 인터뷰 표(20명)
C. 패턴 분석 요약
D. KPI 추정 표(보수/기준/낙관)
E. Go/No-Go 판정
F. 다음 2주 액션 7개
G. 한계(왜 실제 인터뷰가 여전히 필요한지)

- 불확실한 항목은 [가정] 표시
