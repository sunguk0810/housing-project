# 신혼부부 주거 추천 서비스를 위한 UX 전략 리서치 보고서

신혼부부 대상 수도권 주거 추천 서비스의 핵심 UX 원칙은 **"커플 공동 의사결정 지원, 금융 정보 입력의 최소 마찰, 그리고 불안 감소를 위한 투명성"** 세 축으로 수렴한다. Zillow 사용자의 **86%가 파트너와 공동으로 주택을 탐색**하고, 73%의 커플이 주거 기준 합의 과정을 스트레스로 인식한다는 조사 결과는 이 서비스의 핵심이 '개인 검색 도구'가 아니라 '커플 의사결정 플랫폼'이어야 함을 시사한다. 한국 프롭테크 시장에서는 허위매물이 사용자 불만 1위이며, 호갱노노의 실거래가 기반 접근이 **이탈률 21.8%라는 업계 최저 수치**를 달성한 사실은 가격 투명성이 신뢰 구축의 가장 강력한 수단임을 입증한다.

## 1. 커플 공동 의사결정 UX: "같이 고르는 경험"을 설계하라

### 핵심 원칙: 비동기 "스낵킹 모드" 우선 설계

Zola의 UX 리서치에서 발견한 결정적 인사이트가 있다. 커플의 의사결정은 함께 앉아서 한 번에 하는 것이 아니라, **출퇴근 중·점심시간·잠들기 전에 각자 짧게 "스낵킹"하는 방식**으로 진행된다. 이 서비스의 주 사용 맥락도 동일하다. 따라서 실시간 공동 브라우징보다 비동기 협업을 기본 모드로 설계해야 한다.

**구현 가이드라인:**

**① 공유 찜 목록 + 매칭 가시화 (Redfin 패턴)** — 각자 독립적으로 매물을 찜한 후, "둘 다 찜한 매물" / "한 사람만 찜한 매물"을 시각적으로 분리하여 보여준다. Redfin의 Shared Search는 이 패턴으로 **앱 사용자의 70%가 1개 이상 찜**, 40%가 30개 이상을 저장하는 높은 참여를 이끌어냈다. 이 시스템의 핵심은 의견 불일치를 대립이 아닌 **대화의 시작점**으로 자연스럽게 전환하는 데 있다.

**② 블라인드 투표 → 결과 공개 패턴 (Paired 앱 참고)** — 최종 후보 2~3곳에 대해 각자 독립적으로 우선순위를 매긴 후, 둘 다 입력을 완료해야 상대의 선택이 공개되는 구조. Paired 앱에서 이 패턴은 **800만 다운로드, 80% 사용자가 "소통 개선"을 보고**하는 성과를 보였다. 앵커링 바이어스(한 사람의 의견이 다른 사람에게 영향을 미치는 현상)를 방지하고, 게임적 재미를 더한다.

**③ 매물별 인라인 코멘트/이모지 반응** — Honeydue가 거래 내역에 이모지 반응과 질문을 달 수 있게 한 패턴을 매물 카드에 적용한다. "여기 통근 30분이래! 👍" 같은 빠른 피드백을 별도 앱 전환 없이 매물 컨텍스트 안에서 주고받을 수 있어야 한다.

**④ 알림 브릿지 시스템** — 파트너가 매물을 찜하거나 코멘트를 남기면 즉시 푸시 알림. Zillow에서는 iMessage를 통해 한 달에 **168만 사용자**가 매물을 공유했으며, 이러한 저노력·고효과 터치포인트가 커플 간 지속적 참여를 유도했다.

**⑤ 프라이버시 컨트롤 (Honeydue 참고)** — 예산·급여 등 민감 정보에 대해 "파트너에게 총액만 공개" / "세부 내역 공개" / "비공개" 중 선택할 수 있는 단계별 공개 설정. 커플 관계의 다양한 단계와 금전 감수성을 존중하는 설계다.

## 2. 예산 입력 온보딩: 3개 질문으로 첫 가치를 전달하라

### 핵심 원칙: "값을 보여준 뒤에 정보를 요청하라"

핀테크 앱의 온보딩 이탈률은 평균 **68%**에 달한다. 그러나 Shine Bank는 최적화된 단계별 온보딩으로 **80% 완료율**을 달성했다. 차이를 만드는 핵심은 **최소 입력 → 즉시 가치 전달 → 점진적 정보 추가**의 흐름이다.

**구현 가이드라인:**

**① 원 퀘스천 퍼 스크린 (토스 패턴)** — 토스가 **1,600만 MAU**를 달성한 핵심 UX 원칙이다. 한 화면에 하나의 질문만 배치하고, 왜 이 정보가 필요한지 한 줄로 설명한다. 부부 합산 월 소득 → 현재 보유 자산 → 기존 대출 잔액의 3단계로 시작한다.

**② 빠른 입력 버튼 (토스 DSR 계산기 참고)** — "+1,000만" "+1억" 같은 터치 버튼으로 키보드 입력을 최소화한다. 모바일 환경에서 금액 입력의 마찰을 획기적으로 줄이는 패턴이다.

**③ 3개 범위로 결과 제시 (Rocket Mortgage 패턴)** — "안정적" / "다소 부담" / "적극적" 세 가지 예산 범위를 DTI 기준으로 보여준다. 하나의 확정 숫자 대신 범위를 제시하면 사용자가 상황에 맞게 탐색할 수 있고, "정답이 없다"는 불안을 줄인다.

**④ 금융 용어의 인라인 해설 (핀다 패턴)** — DSR, LTV, 전세자금대출 같은 용어를 별도 용어사전이 아닌 **해당 용어가 등장하는 바로 그 위치에서** 쉬운 말로 풀어준다. 핀다의 접근: "DSR이 40%라면, 연봉 5,000만 원 기준 연간 대출 원리금 상환이 최대 2,000만 원까지 가능하다는 뜻이에요." 구체적 금액 예시가 이해도를 극적으로 높인다.

**⑤ 마이데이터 연동으로 자동 채움** — 뱅크샐러드가 **180조 원 규모의 연동 자산**을 관리하는 기반인 한국 마이데이터 인프라를 활용하면, 수동 입력 대신 자동으로 자산·부채·소득을 불러올 수 있다. 단, 연동 전에도 수동 입력으로 기본 결과를 먼저 보여주어야 한다.

**⑥ 전세/월세/매매 분기 설계** — 한국 주거 시장의 고유한 구조를 반영하여, 매매·전세·월세·반전세 중 원하는 거래 유형을 먼저 선택하게 한 후, 각 유형에 맞는 맞춤형 예산 계산 흐름을 제공한다. 핀다는 거래 유형별로 별도 계산기를 운영하며, 특히 전세의 경우 DSR 적용 여부가 지역·주택 보유 수에 따라 달라지는 규제 복잡성을 실시간 반영한다.

## 3. 모바일 지도+리스트 인터페이스: 바텀시트 3단 구조를 기본으로

### 핵심 원칙: 지도 맥락을 잃지 않으면서 매물을 탐색하게 하라

**구현 가이드라인:**

**① 3단 넌모달 바텀시트 (Google Maps 표준)** — Peek(~120px, 매물 카드 1개 미리보기) → Half(50%, 스크롤 가능한 매물 리스트 3~4개) → Full(90%, 전체 리스트+정렬/필터). 바텀시트는 항상 **넌모달**로 유지하여 시트가 올라와 있어도 지도 인터랙션이 가능해야 한다. NN/g 가이드라인에 따라 반드시 **그랩 핸들과 X 닫기 버튼을 동시 제공**하고, 뒤로가기 제스처로도 닫을 수 있어야 한다.

**② 가격 라벨 핀 (Zillow 표준)** — 지도 위 마커에 가격을 직접 표시하는 것은 이제 업계 필수 요소다. 핀 크기 **60~80dp × 24~32dp**, 브랜드 컬러 배경에 흰색 텍스트로 높은 대비를 유지한다. 한국 맥락에서는 호갱노노처럼 단지 단위 최근 실거래가를 표시하고, 가격 추이(상승/하락)를 색상 코드(빨강/파랑)로 즉시 파악할 수 있게 한다.

**③ 양방향 싱크 (Airbnb 패턴)** — 카드 캐러셀을 스와이프하면 해당 매물의 핀이 지도에서 하이라이트되고, 핀을 탭하면 해당 카드로 자동 스크롤된다. 이 **양방향 동기화**가 공간 인지와 매물 정보를 자연스럽게 연결한다.

**④ 바운딩 박스 자동 검색** — Redfin이 채택한 패턴으로, 지도를 팬/줌할 때 별도의 "이 지역 검색" 버튼 없이 자동으로 매물이 갱신된다. 지도 이동 자체가 검색 쿼리가 되는 직관적 경험이다.

**⑤ 수평 필터 칩** — 검색바 아래에 가격·전세/월세·방 수·면적 등의 필터를 수평 스크롤 칩으로 배치한다. 칩 탭 시 바텀시트에서 상세 옵션이 열리며, **실시간 결과 수를 필터 옵션 옆에 표시**하여 제로 결과 상태를 사전 방지한다. Baymard Institute에 따르면 e-커머스 사이트의 **약 50%가 제로 결과 복구 경로를 제공하지 않아** 이탈을 유발한다.

**⑥ 출퇴근 동선 오버레이 (이 서비스의 핵심 차별점)** — 맞벌이 커플의 두 직장 위치를 지도에 마커로 표시하고, 각 매물에서 두 직장까지의 통근 시간을 색상 그라데이션으로 오버레이한다. Trulia의 34개 지도 레이어 접근법처럼, 통근·어린이집·안전 점수를 토글 가능한 데이터 레이어로 제공한다.

## 4. 점수 시각화: "왜 이 점수인지"를 보여주는 것이 신뢰의 핵심

### 핵심 원칙: 점수 단독이 아니라 "맥락 + 구성요소 + 비교 기준"을 함께 제시하라

Walk Score가 **30,000개 이상 웹사이트에 통합**되고 **42편 이상의 학술 논문**에 인용될 만큼 신뢰받는 이유는 세 가지다: ① 0-100 숫자에 "Walker's Paradise" 같은 카테고리 라벨 부여, ② 거리 감쇄 함수 등 방법론의 투명한 공개, ③ 걸어서 5분 이내 편의시설에 최대 점수를 부여하는 직관적 로직. 이 원칙들을 신혼부부 주거 점수에 적용해야 한다.

**구현 가이드라인:**

**① 통합 점수 대신 독립 점수 4개** — "통근 편의" / "육아 인프라" / "안전·보안" / "가성비" 4개 차원을 각각 독립된 0-100 점수로 제시한다. Walk Score가 Walk·Transit·Bike 세 점수를 절대 하나로 합치지 않는 것처럼, 복합 점수 하나보다 **독립 점수가 더 명확하고 실행 가능하다**. 사용자가 자신에게 중요한 차원에 가중치를 스스로 부여할 수 있기 때문이다.

**② 카테고리 라벨 + 백분위 비교** — 각 점수에 "매우 편리(90+)" / "편리(70-89)" / "보통(50-69)" / "불편(~49)" 같은 한국어 라벨을 붙인다. 추가로 "수도권 상위 15%" 같은 백분위 프레이밍을 병행하면 숫자에 의미가 부여된다.

**③ 요인 기여도 모델 (토스 신용점수 참고)** — 토스가 신용점수의 상위 3개 긍정 요인과 3개 부정 요인을 시각적으로 보여주는 것처럼, 각 점수의 주요 구성요소를 표시한다. 예: 통근 편의 점수 82점 → "지하철 도보 5분 (+15)" / "배우자 직장 환승 1회 (+12)" / "출근 시간대 혼잡 (-8)". 이렇게 하면 점수가 정보 전달에서 **행동 유도**로 전환된다.

**④ 레이더 차트는 2~3개 매물 비교 전용으로 제한** — 레이더 차트는 5~8개 차원에서 2~3개 항목을 비교할 때만 효과적이다. 3개 이상 겹치면 가독성이 급락한다. 단일 매물에는 수평 막대 차트 + 카테고리 라벨이 더 명확하다.

**⑤ 색상 접근성** — 빨강-초록 조합 대신 **파랑-주황** 또는 **보라-금색** 대비를 사용한다. 인구의 약 4.5%가 적녹 색맹이며, 색상 외에 아이콘(체크마크, 경고 삼각형)과 텍스트를 반드시 병행해야 한다.

**⑥ 점수 방법론의 계층적 공개** — 점수 옆 작은 ⓘ 아이콘 → 탭하면 1줄 요약 → "자세히 보기" → 전체 방법론 페이지. Niche는 각 랭킹의 정확한 요인·가중치·데이터 출처를 공개하여 신뢰를 구축한다. 방법론을 숨기면 점수가 자의적으로 느껴진다.

## 5. 고관여 의사결정의 불안 감소: 가격 투명성이 가장 강력한 무기

### 핵심 원칙: 신뢰 신호는 "적지만 깊게" — 과도하면 오히려 역효과

한 SaaS 기업이 가격 페이지에 보안 배지, 고객 로고, 30일 보증 마크를 대량 배치한 결과 전환율이 오히려 **12% 하락**했다. 신뢰 신호가 과도하면 "이 서비스에 신뢰 문제가 있나?"라는 역신호를 보낸다. 핵심은 유기적으로 통합된 소수의 깊은 신뢰 장치다.

**구현 가이드라인:**

**① 실거래가 기반 가격 맥락 제공** — 호갱노노가 **이탈률 21.8% (다방 54.7%, 직방 38.3% 대비 최저)**와 평균 사용 기간 **519일**을 달성한 핵심은 국토교통부 실거래 데이터의 투명한 제공이다. 추천 매물 옆에 동일 단지·동일 평형의 최근 실거래가 추이 차트를 배치하면, 호가가 적정한지 사용자가 스스로 판단할 수 있다. 이것이 **가장 강력한 불안 감소 장치**다.

**② 점진적 몰입 사다리 (Progressive Commitment)** — 자유 탐색(0 몰입) → 찜하기(마이크로 몰입) → 알림 설정(자동 참여) → 비교하기(능동 평가) → 배우자 공유(사회적 몰입) → 외부 중개 사이트 이동(행동 몰입). Zillow는 이 흐름을 통해 사용자가 자연스럽게 몰입도를 높이도록 설계하며, 각 단계에서 등록을 요구하지 않는 소프트 CTA("이 검색 저장하기", "가격 변동 알림 받기")를 사용한다.

**③ 사회적 증거의 정량적 제시** — Zillow 데이터에 따르면 **일 250회 이상 조회**된 매물은 1주 내 거래 성사, **일 10회 이상 저장**된 매물은 호가 이상 거래 확률이 높다. "이 매물을 127명이 보고 있어요", "이번 주 8쌍의 신혼부부가 관심 등록" 같은 정량적 사회 증거를 매물 카드에 배치한다. 단, 인위적 희소성 압박("곧 없어집니다!")은 피하고, 사실 기반 데이터만 제시한다.

**④ 안전·보안 데이터의 지도 오버레이** — 다방이 매물 주변 CCTV 위치, 경찰서, 안전 시설을 지도에 표시하는 패턴이 젊은 여성 사용자에게 특히 호응을 얻었다. 이 서비스에서는 야간 보행 안전 등급, 범죄 발생 히트맵, 가로등 밀도 등을 지도 레이어로 제공한다.

**⑤ 비교 도구의 불안 감소 효과** — 나란히 비교(Side-by-side) 기능은 "정보를 충분히 검토했다"는 감각을 제공한다. 2~3개 매물의 통근 시간, 어린이집 수, 안전 점수, 월 예상 비용을 한 화면에서 비교할 수 있어야 하며, 이 비교 화면을 **카카오톡으로 그대로 공유**할 수 있어야 한다.

## 6. 프롭테크 사용자 여정의 마찰 지점과 해결책

### 이 서비스가 반드시 피해야 할 6가지 마찰

**① 가치 전달 전 회원가입 강제 금지** — 모바일 앱 사용자의 **43%가 본인 인증 과정에서 이탈**한다. 예산 입력 → 추천 결과 확인까지는 비로그인으로 가능하게 하고, 찜하기·알림·공유 등 고가치 행동에서만 자연스럽게 회원가입을 유도한다. 카카오 로그인 한 번으로 완료되어야 한다.

**② 제로 결과 상태의 적극적 복구** — 필터를 너무 좁게 설정하여 "결과 없음"이 표시되면 즉각 이탈로 이어진다. 해결책: 필터 옵션 옆 실시간 결과 수 표시, "예산을 500만 원 늘리면 12개 매물 추가" 같은 구체적 필터 완화 제안, 인기 매물 자동 표시.

**③ 금융 용어 장벽 해소** — 전세, 월세, 반전세, 보증금, 관리비, DSR, LTV, 용적률, 건폐율 등은 부동산 초보 신혼부부에게 진입 장벽이다. 모든 전문 용어에 인라인 툴팁을 제공하고, 첫 등장 시 자동으로 짧은 설명을 펼쳐 보여준다.

**④ 복수 앱 교차 사용 문제 해결** — 현재 한국 사용자는 직방(매물) + 호갱노노(시세) + 네이버 부동산(검증) + 뱅크샐러드(예산) 등 복수 앱을 오가야 한다. 이 서비스가 예산 분석·매물 추천·시세 비교·안전 점수를 원스톱으로 제공하면, 이 파편화된 경험 자체가 핵심 차별점이 된다.

**⑤ 필터 복잡도 관리** — 직방이 필터 옵션 부족으로, 다방이 투룸/쓰리룸 병합으로 비판받는 것은 필터 설계가 양날의 검임을 보여준다. **핵심 필터 3개(예산·위치·평형)를 1차로 노출**하고, 나머지(층수·방향·건축연도·전세대출 가능 여부)는 "상세 필터"로 숨긴다.

**⑥ 모바일 지도 성능 최적화** — 페이지 로드 1초 → 3초 증가 시 **전환율 50% 하락, 이탈률 6% 증가**. 뷰포트 내 핀을 50~100개로 제한하고, 프로그레시브 클러스터링으로 줌 레벨에 따라 동적 단순화한다. 고해상도 매물 사진은 레이지 로딩 필수다.

## 7. 카카오톡 공유 전략: 단순 링크가 아닌 "의사결정 허브"로

### 핵심 원칙: 한국 스마트폰 사용자의 97%가 카카오톡을 사용하므로, 공유 경험이 곧 서비스 경험이다

**구현 가이드라인:**

**① 단일 매물 공유 — Feed Template Type B 활용** — 카카오 SDK의 피드 템플릿 B형이 최적이다. 구성: 매물 대표 이미지(400×400px 이상) + 아이템 리스트(거래유형·가격·면적·층수·위치, 최대 5항목) + 서머리(월 예상 비용) + 버튼 2개("매물 보기" + "지도에서 보기"). 가격을 가장 눈에 띄게 배치하고, CTA 버튼 텍스트는 **8자 이내**로 한다.

**② 비교 목록 공유 — List Template 활용** — 부부가 후보 2~3곳을 정리한 비교 목록을 하나의 카카오톡 메시지로 공유한다. 리스트 템플릿의 헤더에 "🏠 우리 후보 매물 3곳"을, 각 아이템에 매물 썸네일·단지명·가격·핵심 점수를 배치한다. 수신자가 탭하면 앱 내 비교 화면으로 바로 이동한다.

**③ 딥링크 전략** — Kakao SDK의 `androidExecutionParams` / `iosExecutionParams`를 활용하여 매물 상세 화면으로 직접 이동한다. **앱 미설치 시 디퍼드 딥링크**(Airbridge 또는 Branch 활용)로 앱스토어 → 설치 → 해당 매물 화면까지 연결한다. 딥링크 적용 시 클릭-설치 전환율이 **30% 이상** (미적용 시 5%)으로 상승한다는 업계 데이터가 있다.

**④ 공유 수신 경험의 차별화** — 현재 한국 프롭테크 앱 중 **진정한 협업형 공유 보드를 제공하는 서비스는 없다**. 이것이 이 서비스의 가장 큰 기회다. 카카오톡으로 받은 링크를 탭하면 단순히 매물 상세가 아니라, "배우자가 보낸 후보 매물 → 내 의견 추가(찜/패스/코멘트) → 우리의 비교 보드 자동 업데이트"로 이어지는 양방향 협업 흐름을 구축한다.

**⑤ 카카오톡 그룹채팅방을 비공식 허브로 인식** — 오늘의집 스크랩북, 당근마켓 링크 공유 등 한국 앱들의 공유는 대부분 카카오톡 그룹채팅방으로 수렴한다. 이 현실을 인정하고, 공유 카드에 충분한 정보를 담아 "채팅방에서 바로 논의할 수 있는" 수준의 정보 밀도를 제공해야 한다.

## 종합 UX 원칙 10가지

이상의 리서치를 종합하여 이 서비스의 UX 설계를 관통하는 10가지 핵심 원칙을 도출한다.

1. **비동기 협업 우선**: 커플의 실제 탐색은 각자 따로, 짧게, 반복적으로 이루어진다. 실시간 공동 브라우징은 부가 기능이지 핵심이 아니다.

2. **3입력 내 첫 가치 전달**: 소득·자산·대출 3개 입력만으로 "어디에 살 수 있는지" 범위를 즉시 보여준다. 추가 정보는 이후 점진적으로 수집한다.

3. **지도가 곧 검색이다**: 바운딩 박스 자동 갱신 + 가격 라벨 핀 + 3단 바텀시트로 지도 맥락을 유지하며 매물을 탐색한다.

4. **점수는 숫자가 아니라 이야기다**: 0-100 점수 + 카테고리 라벨 + 구성요소 분해 + 백분위 비교의 4층 구조로 신뢰와 실행 가능성을 동시에 확보한다.

5. **실거래가가 최고의 신뢰 장치**: 호가 옆에 동일 조건 실거래 이력을 항상 병기하여 가격 불안을 근본적으로 해소한다.

6. **점진적 몰입**: 등록 없이 탐색 → 카카오 로그인으로 찜 → 파트너 초대 → 비교 → 외부 사이트 연결. 각 단계에서 자연스러운 전환을 유도한다.

7. **공유는 양방향이다**: 카카오톡 공유 후 수신자가 반응(찜/패스/코멘트)하면 공유 보드가 자동 업데이트되는 협업 루프를 구축한다.

8. **금융 용어의 인라인 해설**: 전문 용어는 등장 즉시, 구체적 금액 예시와 함께 풀어준다. 별도 용어사전 페이지는 효과가 낮다.

9. **신뢰 신호는 적지만 깊게**: CCTV 위치, 실거래가, 인증 중개사 배지 등 검증된 소수의 신뢰 장치를 콘텐츠에 유기적으로 통합한다. 배지를 대량 나열하면 역효과다.

10. **멀티세션 지원**: 부동산 의사결정은 수주~수개월에 걸친 멀티세션 과정이다. 탐색 상태 자동 저장, 필터 유지, 찜 목록 영속성, 가격 변동 알림이 이 긴 여정을 지원한다.

## 결론: 파편화된 한국 프롭테크 경험의 통합이 핵심 기회

현재 한국 신혼부부는 예산 분석(뱅크샐러드/토스), 매물 탐색(직방/다방), 시세 확인(호갱노노), 안전 정보(다방/경찰청 앱), 의사결정 논의(카카오톡 그룹채팅)를 최소 4~5개 앱을 오가며 수행해야 한다. **이 파편화 자체가 가장 큰 마찰이자 가장 큰 기회**다. 예산→추천→비교→공유→행동의 전체 여정을 하나의 서비스 안에서 완결하면서, 커플의 비동기 의사결정 패턴을 네이티브로 지원하는 서비스는 아직 한국 시장에 존재하지 않는다. 특히 카카오톡 기반의 진정한 양방향 협업 보드와, 맞벌이 커플의 이중 통근 동선 최적화는 기존 어떤 서비스도 제공하지 않는 차별화 포인트로, 이 서비스의 핵심 가치 제안이 될 수 있다.
