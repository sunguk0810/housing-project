> **용도**: `디자인_리서치_프롬프트.md`에 포함된 8개 프롬프트를 어떤 순서로 실행하고, 결과를 어떻게 연결하는지 안내합니다.

## 단계별 실행 가이드

### 1단계: 경쟁 분석 (단독 실행)

| 항목 | 내용 |
|------|------|
| **실행** | 프롬프트 1 |
| **방식** | 단독 실행 |
| **컨텍스트 인풋** | 없음 |
| **산출물** | 경쟁 서비스 UI 패턴 벤치마크 분석표, 카테고리별 베스트 패턴 |
| **소요 시간** | 약 15~20분 |

---

### 2단계: UX 원칙 수립 (순차)

| 항목 | 내용 |
|------|------|
| **실행** | 프롬프트 2 |
| **방식** | 순차 — 1단계 완료 후 |
| **컨텍스트 인풋** | **1단계 결과** (경쟁 분석 벤치마크표 + 베스트 패턴) |
| **산출물** | UX 원칙 5~7개, UI 구현 가이드라인, 차별화 포인트, 이탈 위험 지점 |
| **소요 시간** | 약 10~15분 |

**투입 방법**: 프롬프트 2 상단에 다음을 추가
```
## 선행 리서치 결과 (경쟁 서비스 분석)
[여기에 1단계 산출물 붙여넣기]
```

---

### 3단계: 디자인 시스템 + 법적 가드레일 + 페이지 구조 (병렬 가능)

3개 프롬프트를 **동시에** 각각 별도 세션에서 실행할 수 있습니다.
서로 의존성이 없고, 각각 2단계 결과만 참조합니다.

| 프롬프트 | 영역 | 컨텍스트 인풋 | 산출물 |
|---------|------|-------------|--------|
| **P3** | 디자인 시스템 기초 | 2단계 결과 (UX 원칙) | 컬러·타이포·컴포넌트 가이드 |
| **P6** | 법적 가드레일 | 2단계 결과 + 컴플라이언스 가이드 문서 | 용어 매핑, 동의 UX, 안전 점수 UI |
| **P7** | 페이지 구조 & 라우팅 | 2단계 결과 | 사이트맵, 라우팅, 플로우 다이어그램 |

**투입 방법**: 각 프롬프트 상단에 다음을 추가
```
## 선행 리서치 결과 (UX 원칙)
[여기에 2단계 산출물 붙여넣기]
```

P6의 경우 추가로:
```
## 법적 컴플라이언스 가이드
[컴플라이언스_가이드.md 내용 붙여넣기 또는 핵심 요약]
```

**소요 시간**: 각 10~15분 (병렬이면 총 15분)

---

### 4단계: 온보딩 + 결과 화면 상세 설계 (병렬 가능)

2개 프롬프트를 **동시에** 실행할 수 있습니다.
둘 다 동일한 컨텍스트(P2 + P3 + P6)가 필요합니다.

| 프롬프트 | 영역 | 컨텍스트 인풋 | 산출물 |
|---------|------|-------------|--------|
| **P4** | 온보딩 플로우 상세 | P2 + P3 + P6 결과 | 스텝별 와이어프레임 JSX |
| **P5** | 결과 화면 & 점수 시각화 | P2 + P3 + P6 결과 | 결과 화면 컴포넌트 코드 |

**투입 방법**: 각 프롬프트 상단에 다음을 추가
```
## 디자인 컨텍스트 (선행 리서치 요약)

### UX 원칙 (P2 결과 요약)
[핵심 원칙 5~7개 요약]

### 디자인 시스템 (P3 결과 요약)
[컬러 팔레트, 서체, 핵심 컴포넌트 규격 요약]

### 법적 가드레일 (P6 결과 요약)
[용어 제한, 동의 UX, 안전 점수 표현 규칙 요약]
```

> **⚠️ 토큰 절약 팁**: P2, P3, P6 결과를 전문 그대로 넣으면 컨텍스트가 너무 길어질 수 있습니다. 각 결과의 **핵심 결론만 요약**(각 10~20줄)해서 "디자인 컨텍스트"로 정리한 뒤 투입하세요.

**소요 시간**: 각 15~20분 (병렬이면 총 20분)

---

### 5단계: 디자인 브리프 통합 (순차)

P1~P7 전체 산출물을 하나의 문서로 종합합니다. **이 단계는 리서치가 아니라 정리 작업**입니다.

| 항목 | 내용 |
|------|------|
| **실행** | 프롬프트 8 |
| **방식** | 순차 — 4단계 완료 후 |
| **컨텍스트 인풋** | **P1~P7 산출물 전체** |
| **산출물** | `design_brief.md` — 디자인 관점의 프로젝트 브리프 단일 문서 |
| **소요 시간** | 약 10~15분 |

**투입 방법**: 프롬프트 8에 P1~P7 산출물을 투입
```
## 입력 문서
[P1_경쟁분석.md 붙여넣기]
[P2_UX원칙.md 붙여넣기]
[P3_디자인시스템.md 붙여넣기]
[P4_온보딩.md 붙여넣기]
[P5_결과화면.md 붙여넣기]
[P6_법적가드레일.md 붙여넣기]
[P7_페이지구조.md 붙여넣기]
```

> **⚠️ 토큰 절약 팁**: 7개 문서 전문이 컨텍스트를 초과하면, 각 문서의 Executive Summary + 핵심 테이블/결론만 추출하여 투입하세요. P8 프롬프트는 새로운 결정을 내리지 않으므로 결론 부분만 있으면 충분합니다.

**이 문서의 용도**:
- 개발 착수 시 "디자인은 이거 하나 읽으세요"로 전달
- 요구사항 베이스라인(`requirements_baseline.md`)과 함께 MVP 개발의 양대 참조 문서
- 외주 디자이너/개발자 온보딩 시 컨텍스트 전달용

## 주의사항

- **P6(법적 가드레일)을 빼먹지 마세요**: P4, P5 실행 시 법적 제약(용어 금지, 비저장 신뢰 UI, 안전 점수 프레이밍)이 반영되지 않으면 나중에 전면 수정이 필요합니다.
- **컴플라이언스 문서를 함께 투입하세요**: P6 실행 시 `컴플라이언스_가이드.md`를 컨텍스트로 제공하면 훨씬 정확한 가드레일이 나옵니다.
- **P7(페이지 구조)은 독립적입니다**: 4단계에 투입하지 않아도 되지만, P4/P5 작업 시 "이 화면이 어떤 URL에 위치하는지" 참고하면 좋습니다.
- **P8은 리서치가 아닙니다**: P1~P7이 모두 끝난 뒤에만 실행하세요. 새로운 디자인 결정을 내리는 게 아니라 기존 결과를 하나로 묶는 정리 작업입니다.
- **리서치 결과를 파일로 관리하세요**: 각 프롬프트의 산출물을 `P1_경쟁분석.md`, `P2_UX원칙.md` 등으로 저장해두면 이후 Claude Code에서 컨텍스트로 활용하기 편합니다.
