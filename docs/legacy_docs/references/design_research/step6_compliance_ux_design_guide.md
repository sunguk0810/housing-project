## 0) 설계 원칙 요약

| 원칙 | 설명 |
| --- | --- |
| Value-First | 동의/인증 전에 서비스 가치를 먼저 전달한다 |
| Contextual Trust | 신뢰 신호를 별도 배치하지 않고 콘텐츠 맥락에 유기적으로 통합한다 |
| Positive Framing | 부정적 표현을 사용하지 않고 긍정적 지표로 전환한다 |
| Progressive Disclosure | 면책/고지 정보를 사용자 여정 단계에 맞춰 강도를 조절한다 |
| Minimal Friction | 법적 요건을 충족하면서 이탈률을 최소화하는 인라인 설계를 우선한다 |

## 1) 용어 가이드라인 — 전체 서비스 적용

### 1-1. 금지/대체 용어 매핑 테이블

서비스 전체(기획서·디자인·개발·QA·마케팅·푸시 알림·이메일)에 적용한다.

#### 핵심 용어 (공인중개사법)

| 금지 표현 | 대체 표현 | 금지 사유 |
| --- | --- | --- |
| 추천 | 분석 결과 안내 | '추천'은 알선/중개 행위로 오인 가능 |
| 알선 | 정보 연결 | 직접적 중개 용어 |
| 중개 | 정보 분석 | 중개업 등록 없이 사용 불가 |
| 매물 추천 | 조건 맞춤 단지 탐색 | '추천 + 매물' 조합이 중개 알선 인상을 강화 |
| 최적 매물 | 분석 상위 단지 | 단정적 표현 + 매물 조합이 자문/알선으로 오인 |
| 맞춤 매물 | 조건 부합 단지 | '맞춤'이 개인별 알선 인상을 줄 수 있음 |

#### CTA 용어

| 금지 표현 | 대체 표현 | 금지 사유 |
| --- | --- | --- |
| 매물 문의 | 외부 매물 보러가기 | 서비스가 문의 접수 주체로 오인됨 |
| 상담 신청 | 외부 사이트에서 확인 | 직접 상담을 제공하는 인상 |
| 중개사 연결 | 주변 중개사무소 정보 | 3자(매물·중개사·사용자) 연결 = 알선 행위 해당 가능 |
| 바로 문의 | 외부 페이지로 이동 | 서비스 내에서 문의가 완결되는 인상 |

#### 금융 용어 (금융 자문 오인 방지)

| 금지 표현 | 대체 표현 | 금지 사유 |
| --- | --- | --- |
| 대출 가능 | 예상 대출 범위 (참고용) | 금융 자문 오인, 승인 보장 인상 |
| 투자 수익률 | 과거 실거래가 변동 추이 | 투자 자문 오인 가능 |
| 대출 승인 가능 | 참고용 시뮬레이션 결과 | 금융 승인 보장으로 오인 |
| 예산 적정 | 참고 범위 이내 | 재무 자문으로 오인 가능 |

#### 치안 표현 (명예훼손·부동산가치 훼손 방지)

| 금지 표현 | 대체 표현 | 금지 사유 |
| --- | --- | --- |
| 범죄율 높음 | 안전 편의시설 현황 | 지역 비하·명예훼손 리스크 |
| 위험 지역 | 안전 인프라 보통 | 부정적 단정 표현 금지 |
| 치안 열악 | CCTV·가로등 보강 여지 | 부정적 프레이밍 → 긍정적 지표로 전환 |
| 가장 안전한 | 안전 인프라 상위 | 절대적 단정 표현 지양 |
| 범죄 위험 지역 | 안전시설 확충 중인 지역 | 직접적 위험 표현 금지 |

### 1-2. 용어 운영 규칙

- PR/디자인 리뷰 시 금지 용어 체크리스트 자동 검사 (CI Lint 권장)
- 마케팅 소재·푸시 알림·이메일 제목에도 동일 기준 적용
- 외부 제휴사 콘텐츠 노출 시에도 금지 표현 필터링 필수
- 분기 1회 전체 UI 텍스트 감사(audit) 실행
- 신규 문구 추가 시 용어 가이드 사전 검토 후 반영

## 2) 동의 플로우 UX — 이탈률 최소화 설계

### 2-0. 전체 플로우 개요

```
랜딩 진입 ──→ 입력 시작 직전 ──→ 입력 플로우 (3단계) ──→ 결과 확인 ──→ 찜/공유 (로그인)
 (가치 전달)    (동의 수집)         (원퀘스천퍼스크린)      (비로그인)     (점진적 몰입)
```

핵심 원칙: **결과 확인까지 로그인을 요구하지 않는다** (43%의 모바일 사용자가 본인 인증에서 이탈).

### 2-1. Step 0 — 랜딩 진입: 가치 전달 우선

- 동의/인증을 요청하지 않는다.
- "3가지만 입력하면 바로 결과 확인"으로 즉시 가치를 전달한다.
- 하단에 "회원가입 없이 무료로 이용 · 입력 정보 서버 미저장" 메시지로 신뢰를 선제적으로 확보한다.
- 서비스 포지션: "맞벌이 통근·육아·예산을 한 번에 분석하는 정보 플랫폼"

**랜딩 페이지 정보 구조:**
- 서비스 메인 카피 (정보 분석 플랫폼 포지션 반영)
- CTA: "분석 시작하기" (추천/중개 용어 미사용)
- 하단 고정: 서비스 포지션 면책 문구 + 법적 링크 (이용약관, 개인정보처리방침, 위치정보이용약관)

### 2-2. Step 1 — 입력 시작 직전: 동의 수집

별도 페이지 이동 없이, 입력 플로우 상단에 인라인으로 동의를 배치한다.

**동의 수집 구조:**

| 구분 | 항목 | 수집 목적 | 보유 기간 |
| --- | --- | --- | --- |
| **[필수]** 개인정보 수집·이용 동의 | 직장 권역, 예산 범위 | 통근 시간 계산, 후보군 필터링 | 세션 종료 시 즉시 삭제 |
| **[선택]** 정밀 분석 추가 정보 | 보유 현금, 상세 소득 | 예산 현실성 정밀 계산 | 세션 종료 시 즉시 삭제 |
| **[필수]** 위치정보 이용 동의 | 직장 주소 → 좌표 변환 | 통근 시간 계산 | 좌표 미저장 |

**UI 설계 요구사항:**

1. 필수/선택이 시각적으로 명확히 분리되어야 한다 (색상·라벨·위치 구분).
2. 각 항목에 수집 목적·보유기간이 1줄로 요약 표시되어야 한다.
3. "전문 보기" 링크로 상세 약관 접근이 가능해야 한다.
4. 필수 동의 미체크 시 CTA 버튼이 비활성화 상태를 유지해야 한다.
5. 선택 동의를 하지 않아도 기본 분석 이용이 가능해야 한다.
6. 상단에 "입력 정보 비저장" 신뢰 배지를 배치하여 금융 정보 입력 불안을 해소한다.

**신뢰 배지 메시지:**
> 🔒 입력하신 금융 정보는 분석에만 사용되며 **서버에 저장되지 않습니다**

### 2-3. Step 2 — 입력 플로우: 원 퀘스천 퍼 스크린

토스 패턴을 적용한 최소 마찰 입력.

**입력 3단계:**
1. 부부 합산 월 소득
2. 현재 보유 자산
3. 기존 대출 잔액

**UI 설계 요구사항:**

1. 한 화면에 하나의 질문만 배치한다.
2. 상단에 프로그레스 바 (3단계)를 표시한다.
3. 각 질문에 "왜 이 정보가 필요한지" 한 줄 설명을 포함한다.
4. "+1,000만", "+5,000만", "+1억" 빠른 입력 버튼으로 키보드 의존도를 줄인다.
5. 매 입력 화면 하단에 "🔒 이 금액은 분석 후 즉시 삭제됩니다" 축약형 리마인더를 배치한다.
6. 금융 용어(DSR, LTV 등)가 등장하면 인라인 툴팁으로 즉시 해설한다.

### 2-4. Step 3 — 결과 확인 후: 점진적 몰입

**결과 확인은 비로그인으로 가능**해야 한다. 고가치 행동 시점에서만 인증을 유도한다.

**점진적 몰입 사다리:**

| 단계 | 행동 | 인증 필요 |
| --- | --- | --- |
| 0 | 자유 탐색·결과 확인 | ✕ 불필요 |
| 1 | 찜하기 | ✓ 카카오 로그인 |
| 2 | 알림 설정 | ✓ 카카오 로그인 |
| 3 | 파트너 초대·공유 | ✓ 카카오 로그인 |
| 4 | 비교 보드 | ✓ 카카오 로그인 |
| 5 | 외부 매물 사이트 이동 | ✕ 불필요 |

**로그인 유도 UI:**
- "카카오로 3초 만에 시작하기" CTA 제공
- "로그인 없이 계속 둘러보기" 대안을 항상 함께 제공
- 카카오 로그인 한 번으로 완료 (추가 본인 인증 없음)

## 3) 신뢰 UI 패턴 — "적지만 깊게" 원칙

> 참고: SaaS 기업이 가격 페이지에 보안 배지·고객 로고·보증 마크를 대량 배치한 결과  
> 전환율이 오히려 12% 하락한 사례가 있다. 과도한 신뢰 신호는 역신호를 보낸다.

### 3-1. 인라인 보안 메시지 (정보 비저장)

**배치 위치:** 금융 정보 입력 폼 상단

**메시지 구조:**
```
[🔒 아이콘] + [타이틀: 입력 정보 비저장] + [부연: 분석 후 즉시 삭제 · 서버에 금융 정보를 보관하지 않아요]
```

**반복 노출 전략:**
- 전체 배지: 입력 폼 상단 (1회)
- 축약형 (🔒 아이콘 + 한줄): 각 입력 화면 하단에 리마인드

**설계 주의:**
- 배지 스타일을 과도하게 크거나 화려하게 하지 않는다 (역신호 방지).
- 콘텐츠 맥락에 자연스럽게 녹여서 "당연한 정보"로 인식되게 한다.

### 3-2. 데이터 출처 인라인 태그

**배치 위치:** 결과 카드마다

**표시 항목:**
- 데이터 출처 (국토교통부 실거래가, ODsay 경로 API, 공공데이터포털 보육시설 등)
- 데이터 기준일 (예: 2026.02.01)
- 갱신 주기 안내 (예: "실거래 반영까지 최대 30일 소요")

**표시 형식:**
```
[📊 국토교통부 실거래가 · 2026.02] [🚇 ODsay 경로 API] [🏫 공공데이터포털 보육시설]
데이터 기준일: 2026.02.01 · 실거래 반영까지 최대 30일 소요
```

**설계 요구사항:**
- 태그 형태로 콤팩트하게 표시하되, 탭하면 상세 정보로 확장 가능
- 모든 공공 데이터에 출처 표시 의무 (공공데이터 이용약관 준수)
- 카카오맵 사용 시 로고 표시 필수

### 3-3. 점수 방법론 계층적 공개

Walk Score의 신뢰 구축 방식을 참고한 3단계 공개 구조:

| 단계 | 트리거 | 표시 내용 |
| --- | --- | --- |
| 1단계 | 점수 옆 ⓘ 아이콘 탭 | 1줄 요약 (예: "지하철 접근성·환승 횟수·혼잡도 기반 산출") |
| 2단계 | "자세히 보기" 탭 | 요인 기여도 분해 (예: 지하철 도보 5분 +15 / 환승 1회 +12 / 혼잡 -8) |
| 3단계 | "전체 방법론 보기" 링크 | 전체 방법론 페이지 (가중치, 데이터 소스, 계산 공식) |

**요인 기여도 표시 형식 (2단계):**
```
통근 편의 점수: 82점
├ 🚇 지하철 도보 5분        +15
├ 🔄 배우자 직장 환승 1회    +12
├ 🚌 버스 노선 다양         +8
└ ⏰ 출근 시간대 혼잡        -8
```

### 3-4. 비단정 문구 패턴 (금융 시뮬레이션)

예산/대출 시뮬레이션 결과에 반드시 적용하는 면책 패턴:

**결과 표시 구조:**
- 3개 범위로 제시: "안정적" / "다소 부담" / "적극적" (DTI 기준)
- 하나의 확정 숫자가 아닌 범위 제시

**필수 면책 메시지:**
> ℹ️ 참고용 시뮬레이션이며 실제 대출 승인 금액과 다를 수 있습니다.
> 정확한 한도는 금융기관에서 확인해주세요.

**설계 주의:**
- 면책 메시지를 회색 톤의 보조 영역에 배치하여 시각적 부담을 최소화
- "승인 가능", "보장" 등 단정 표현 절대 금지
- 결과값 자체에 "참고용" 또는 "예상" 접두어를 항상 포함

## 4) 안전 점수 UI — 긍정적 프레이밍 설계

### 4-1. 프레이밍 원칙

| 원칙 | 설명 |
| --- | --- |
| 시설 현황 중심 | "범죄율"이 아니라 "CCTV 수, 가로등 밀집도, 경찰서 거리" 등 시설 수량을 표시 |
| 상대 비교 | "위험/안전"이 아니라 "서울시 평균 대비" 또는 "수도권 상위 N%" 로 표현 |
| 색상 코드 | 시설 충분도를 표현 (충분 = 녹색, 보통 = 주황, 부족 = 회색) — "위험 = 빨강" 사용 금지 |
| 라벨 | "안전 편의시설 현황"으로 섹션 타이틀 통일 |

### 4-2. 안전 편의시설 현황 — 표시 지표

| 지표 | 데이터 출처 | 표시 형식 | 비교 기준 |
| --- | --- | --- | --- |
| CCTV 수 | 경찰청 생활안전지도 / 지자체 공공데이터 | `23대` (평균 15대) | 서울시 평균 대비 |
| 가로등 밀집도 | 서울시 안전환경 데이터 | 상위 N% | 서울시 평균 대비 |
| 가장 가까운 경찰서 | 경찰청 데이터 | `450m` (평균 800m) | 서울시 평균 대비 |
| 비상벨 수 | 지자체 공공데이터 | `8개` (평균 5개) | 서울시 평균 대비 |
| 편의점·약국 야간 운영 | 공공데이터포털 | 도보 N분 내 N개 | 서울시 평균 대비 |

### 4-3. 시각화 설계

**진행 바 형태:**
- 각 지표를 수평 진행 바로 표시
- 0~100% 범위에서 서울시 평균 대비 상대 위치를 표현
- 색상: 80%+ 녹색 / 60~79% 주황 / 60% 미만 회색 (빨강 사용 금지)

**종합 안내 영역:**
```
안전 인프라 종합 — 수도권 상위 18%
출처: 경찰청 생활안전지도 · 서울시 안전환경 데이터 · 기준일 2026.01
```

### 4-4. Good / Bad 비교

#### ✓ Good Example (사용)

```
섹션 타이틀: 안전 편의시설 현황
비교 기준: 서울시 평균 대비

📹 CCTV          23대 (평균 15대)    ████████████████░░░░ 85%
💡 가로등 밀집도   상위 28%           ██████████████░░░░░░ 72%
🏥 경찰서 거리    450m (평균 800m)   ██████████████████░░ 90%
🚨 비상벨         8개 (평균 5개)     █████████████░░░░░░░ 68%

종합: 안전 인프라 — 수도권 상위 18%
출처: 경찰청 생활안전지도 · 서울시 안전환경 데이터 · 기준일 2026.01
```

- 모든 지표가 긍정적 프레이밍 (시설 수, 거리, 밀집도)
- "서울시 평균 대비"로 상대적 위치 전달
- 색상이 시설 충분도를 표현 (부족 = 회색, '나쁨'이 아님)
- 출처와 기준일 명시로 법적 요건 충족

#### ✕ Bad Example (금지)

```
섹션 타이틀: 치안 위험도        ← 부정적 타이틀
등급: 위험!                    ← 단정 표현

🔴 범죄 발생률    높음          ← 부정적 단정 표현
⚠️ 야간 위험도    위험 지역      ← 지역 비하
📉 치안 등급      D (하위 20%)  ← 순위형 비하
🚨 범죄 빈도     월 평균 12건   ← 원시 범죄 데이터 직접 노출
```

**문제점:**
- "범죄", "위험", "열악" 등 부정적 표현 직접 사용
- 특정 지역을 비하하는 인상 → 명예훼손·부동산가치 훼손 소지
- 원시 범죄 데이터 노출 → 표현 방식의 법적 리스크
- 생활안전지도 이용약관 위반 가능성

## 5) 외부 링크 CTA 패턴

### 5-1. 설계 원칙

| 원칙 | 설명 |
| --- | --- |
| 외부 이동 명시 | CTA 텍스트와 아이콘에 외부 사이트임을 반드시 표시 |
| 출처 명시 | CTA 옆에 이동 대상 사이트명을 표시 (예: "↗ 네이버 부동산") |
| 모달 확인 | 클릭 시 외부 이동 확인 모달을 표시 (면책 고지 포함) |
| 서비스 비관여 | 외부 사이트의 정보에 대해 본 서비스가 책임지지 않음을 명시 |

### 5-2. Pattern A — 결과 카드 내 단일 CTA

```
┌─────────────────────────────────────────┐
│ 래미안 블레스티지 84㎡                      │
│ 전세 5억 2,000만                          │
│ [실거래 5억 1,500만 (2026.01)]            │
│                                          │
│ ┌──────────────────────────────────────┐ │
│ │  외부 매물 보러가기    ↗ 네이버 부동산    │ │
│ └──────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

**CTA 버튼 구성요소:**
- 텍스트: "외부 매물 보러가기"
- 외부 링크 아이콘: ↗
- 대상 사이트명: 네이버 부동산 / 직방 등

### 5-3. Pattern B — 복수 외부 플랫폼 선택

```
이 단지의 매물을 확인할 수 있는 외부 사이트:

● 네이버 부동산   매물 12건   ↗
● 직방           매물 8건    ↗
```

각 항목 클릭 시 동일한 외부 이동 확인 모달을 표시한다.

### 5-4. 외부 이동 확인 모달

**모달 정보 구조:**
```
↗ 외부 사이트로 이동합니다

**네이버 부동산**의 매물 상세 페이지로 이동합니다.
해당 페이지의 정보는 네이버 부동산이 제공하며, 본 서비스와 무관합니다.

┌──────────────────────────────────────────┐
│ 본 서비스는 정보 분석 플랫폼이며 부동산     │
│ 중개 서비스가 아닙니다. 매물 정보의 정확성은  │
│ 해당 외부 사이트에서 확인해주세요.           │
└──────────────────────────────────────────┘

[네이버 부동산으로 이동 ↗]
[취소]
```

**모달 필수 요소:**
1. 이동 대상 사이트명 명시
2. 정보 제공 주체가 외부 사이트임을 명시
3. 서비스 포지션 면책 문구
4. 확인/취소 양쪽 선택지

### 5-5. 금지 CTA 패턴

| 금지 CTA | 금지 사유 |
| --- | --- |
| "매물 문의하기" | 서비스가 문의 주체로 오인 |
| "중개사에게 연락" | 3자 연결 = 알선 행위 |
| "지금 바로 계약 상담!" | 계약/중개 행위 유도 |
| "바로 연결" | 직접 알선 인상 |
| "전화하기" | 서비스 내 직접 연결 기능 제공 인상 |

## 6) 면책 고지 배치 전략

### 6-1. 핵심 면책 문구

> 본 서비스는 공공데이터 기반 정보 분석 플랫폼이며 부동산 중개·알선·자문 서비스가 아닙니다.

### 6-2. 사용자 여정 5개 접점별 배치

| # | 배치 위치 | 톤 강도 | 노출 빈도 | 표시 방식 |
| --- | --- | --- | --- | --- |
| ① | 랜딩 페이지 하단 | 소프트 | 상시 (모든 페이지 하단 고정) | 푸터 텍스트 + 법적 링크 |
| ② | 입력 플로우 시작 시 | 중립 | 1회 | 인라인 정보 배너 |
| ③ | 결과 페이지 상단 | 컨텍스트 내장형 | 결과 확인 시 | 메타 태그 (출처·기준일과 함께) |
| ④ | 외부 링크 클릭 모달 | 명시적 | 매 외부 이동 클릭 시 | 모달 내 면책 영역 |
| ⑤ | 이용약관 | 법률 문서 | 동의 시 | 조항 형태 |

### 6-3. 접점별 상세 설계

#### ① 랜딩 페이지 하단 (상시 노출) — 톤: 소프트

**표시 형식:**
```
본 서비스는 공공데이터 기반 정보 분석 플랫폼입니다 · 부동산 중개·알선·자문 서비스가 아닙니다
© 2026 서비스명 · 이용약관 · 개인정보처리방침 · 위치정보이용약관
```

- 모든 페이지 하단에 항상 고정
- 주요 법적 링크 (이용약관, 개인정보처리방침, 위치정보이용약관)와 함께 배치
- 시각적으로 부담 없는 보조 텍스트 스타일

#### ② 입력 플로우 시작 시 (1회) — 톤: 중립

**표시 형식:**
```
ℹ️ 입력하신 정보는 **분석 목적으로만** 사용되며 거래 연결에 사용되지 않습니다
```

- 정보 입력을 시작하는 시점에서 '분석 도구'라는 서비스 성격을 자연스럽게 전달
- 입력 폼 상단에 인라인 배너로 배치

#### ③ 결과 페이지 상단 (점수 옆) — 톤: 컨텍스트 내장형

**표시 형식:**
```
[📊 공공데이터 기반 분석 결과] [기준일 2026.02.01] [참고용 정보]
```

- 별도 면책 문구 대신, 결과 메타 정보의 일부로 자연스럽게 배치
- "공공데이터 기반 분석 결과"라는 태그 자체가 면책 역할 수행
- 결과의 성격(참고용)을 맥락에서 자연스럽게 인식

#### ④ 외부 링크 클릭 모달 (매 클릭) — 톤: 명시적

**표시 형식:**
```
본 서비스는 정보 분석 플랫폼이며 부동산 중개 서비스가 아닙니다.
매물 정보의 정확성은 해당 외부 사이트에서 확인해주세요.
```

- 외부 이동은 법적으로 가장 민감한 순간
- 모달 내에 면책을 명시적으로 포함하되, 보조 영역(회색 톤)에 배치하여 시각적 부담 최소화

#### ⑤ 이용약관 (법적 구속력) — 톤: 법률 문서

**표시 형식:**
```
제2조 (서비스의 성격)

① 본 서비스는 공공데이터 및 허가된 API를 기반으로 부동산 관련 정보를 
   분석·제공하는 정보 분석 플랫폼입니다.
② 본 서비스는 「공인중개사법」 제2조에 따른 중개행위를 수행하지 않으며, 
   부동산 거래의 알선·중개·자문을 제공하지 않습니다.
③ 본 서비스가 제공하는 분석 결과, 점수, 시뮬레이션은 참고용 정보이며 
   거래 의사결정의 최종 근거로 사용될 수 없습니다.
```

- 법적 구속력 있는 약관에 서비스 성격을 가장 명확하게 정의
- 공인중개사법 조항 직접 인용으로 법적 방어력 확보

## 7) 결과 화면 필수 표시 요소 (Step0 요구사항 반영)

Step0 Legal Compliance 문서의 Functional Requirement #3에 따라, 결과 화면에 아래 요소를 필수 표시한다.

| 요소 | 표시 위치 | 표시 형식 |
| --- | --- | --- |
| 점수 분해 (요인 기여도) | 점수 카드 내 확장 영역 | ⓘ 탭 → 요인별 +/- 점수 |
| 출처 | 결과 카드 하단 태그 | 인라인 태그 (출처명 + 아이콘) |
| 데이터 기준일 | 결과 카드 하단 | 텍스트 (예: 기준일 2026.02.01) |
| 단점 (Why-Not) | 결과 카드 내 | 감점 요인 표시 (예: 출근 시간대 혼잡 -8) |

**Why-Not 표시 설계:**
- 단점을 감추지 않고 투명하게 공개하여 신뢰를 확보한다.
- 부정적 프레이밍이 아닌 "개선 가능 요인" 또는 "참고 사항"으로 표현한다.
- 예시: "⏰ 출근 시간대 혼잡 (-8)" / "🏗️ 건축 연도 2005년 (-5)"

## 8) 동의 철회·삭제 요청 UX

Step0의 User Notice & Consent UX 정책에 따라, 동의 철회/삭제 요청 경로를 설계한다.

**설계 요구사항:**

- 동의 철회 시 즉시 관련 데이터를 삭제하고 확인 메시지를 표시한다.
- 삭제 요청 SLA를 운영 기준으로 문서화하고 사용자에게 안내한다.

**접근 경로:**
```
하단 고정 링크 "개인정보 관리" → 동의 내역 확인 → 동의 철회 / 삭제 요청
```
