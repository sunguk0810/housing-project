# 신혼부부 주거 추천 서비스 — UI/UX 디자인 리서치 프롬프트

> **용도**: Claude Code에서 디자인 리서치 및 구현 시 사용할 프롬프트 모음
> **사용법**: 필요한 프롬프트를 복사하여 Claude Code에 입력. 순서대로 진행하면 체계적인 디자인 시스템 구축 가능.

## 프롬프트 1: 경쟁 서비스 UI 패턴 분석

```
당신은 프롭테크/부동산 서비스 전문 UI/UX 리서처입니다.

아래 서비스들의 UI/UX 패턴을 웹에서 조사하고 분석해주세요:

[국내 부동산/프롭테크]
- 직방 (zigbang.com)
- 호갱노노 (hogangnono.com)
- 아실 (asil.kr)
- 네이버 부동산 (land.naver.com)
- 카카오맵 부동산 (map.kakao.com)

[국내 금융/대출 비교]
- 뱅크샐러드 (banksalad.com)
- 핀다 (finda.co.kr)
- 토스 (toss.im)

[해외 부동산 추천]
- Zillow (zillow.com) — 특히 "Home Recommendation" 기능
- Redfin (redfin.com) — "Commute Filter" 기능
- Trulia (trulia.com) — "Neighborhood" 점수 시스템
- Niche (niche.com) — 지역 평가 점수 카드 UI

각 서비스에 대해 다음을 분석해주세요:

1. **온보딩 플로우**: 사용자 정보(예산, 조건 등)를 어떤 순서로, 어떤 UI 패턴(스텝퍼, 슬라이더, 카드 선택 등)으로 수집하는가?
2. **결과 화면 구성**: 리스트 vs 지도 vs 카드 중 어떤 레이아웃을 쓰는가? 정렬/필터 UX는?
3. **점수/평가 시각화**: 종합 점수, 카테고리별 점수를 어떻게 표현하는가? (레이더 차트, 바 차트, 별점, 게이지 등)
4. **지도 인터랙션**: 마커 클러스터링, 호버/클릭 시 정보 표시 방식, 지도-리스트 연동
5. **모바일 반응형**: 지도+리스트 동시 노출 시 모바일에서의 처리 방식 (바텀시트, 토글 등)
6. **CTA 설계**: 매물 상세 페이지로의 전환 유도 방식, 외부 링크 처리 UX

분석 결과를 표로 정리하고, 우리 서비스에 채택할 만한 "베스트 패턴"을 카테고리별로 추천해주세요.
```

## 프롬프트 2: 타겟 사용자 기반 UX 원칙 수립

```
당신은 UX 전략 컨설턴트입니다.

아래 서비스의 타겟 사용자와 핵심 사용 시나리오를 기반으로 UX 원칙을 수립해주세요.

## 서비스 개요
- **서비스명**: (미정) 수도권 신혼부부 주거 추천 서비스
- **핵심 기능**: 예산(보유현금+연봉+기존대출) + 통근시간(맞벌이 직장 2곳) + 보육(어린이집/학교) + 치안(안전 인프라)을 통합 분석하여 최적 주거지를 추천
- **타겟**: 수도권 신혼부부 (25~39세), 특히 맞벌이·예비 출산 부부
- **플랫폼**: 모바일 우선 (Mobile-First), 웹 반응형

## 타겟 사용자 특성
- 맞벌이 부부: 시간이 부족하고, 의사결정을 빠르게 하고 싶어함
- 부동산 초보: 전문 용어에 익숙하지 않음. 실거래가, DSR, LTV 등을 쉽게 설명해야 함
- 불안감: "이 가격이 맞는 건지", "이 동네가 안전한지" 불확실성이 큼
- 부부 공동 의사결정: 혼자 보고 결정하지 않음. 배우자와 공유하고 비교해야 함
- 모바일 중심: 출퇴근 시간, 점심시간에 짧게 탐색

## 핵심 사용 시나리오
1. **첫 방문**: "우리 예산으로 어디 살 수 있어?" → 조건 입력 → 추천 결과 확인
2. **탐색**: 추천된 단지들을 지도에서 비교, 통근시간·보육·치안 점수 확인
3. **비교**: 후보 2~3곳을 나란히 비교, 각 카테고리별 장단점 파악
4. **공유**: 배우자에게 카카오톡으로 후보 리스트 공유
5. **행동**: 관심 매물의 외부 페이지(네이버 부동산 등)로 이동

## 요청사항
1. 위 특성을 반영한 **UX 원칙 5~7개**를 수립해주세요 (예: "3탭 이내에 결과 도달", "부부 공유 최적화" 등)
2. 각 원칙에 대해 **구체적인 UI 구현 가이드라인**을 제시해주세요
3. 경쟁 서비스 대비 **차별화 UX 포인트** 3가지를 제안해주세요
4. 사용자가 이탈할 수 있는 **위험 지점(friction point)** 과 해결 방안을 정리해주세요
```

## 프롬프트 3: 디자인 시스템 & 비주얼 가이드라인

```
당신은 시니어 프로덕트 디자이너입니다.

아래 서비스를 위한 디자인 시스템 기초를 설계해주세요.

## 서비스 컨텍스트
- 수도권 신혼부부 대상 주거 추천 서비스
- 모바일 우선 웹앱
- 핵심 감성 키워드: **신뢰감**, **따뜻함**, **명쾌함**
- 피해야 할 인상: 차갑고 기술적인 느낌, 복잡한 금융 서비스 느낌, 저렴하거나 유치한 느낌

## 요청사항

### 1. 컬러 시스템
- Primary, Secondary, Accent 컬러 제안 (HEX 코드 포함)
- 시맨틱 컬러: Success, Warning, Error, Info
- 점수 시각화 전용 컬러 스케일 (5단계: 매우 좋음 → 보통 → 미흡)
- 다크 모드 고려 여부 판단
- 컬러 선정 근거 (타겟 사용자 감성, 접근성 WCAG AA 기준)

### 2. 타이포그래피
- 한글 서체 추천 (웹폰트, 라이선스 무료)
- 영문/숫자 서체 추천
- 타입 스케일 (H1~H6, Body, Caption, Label)
- 금액 표시 전용 스타일 (예: ₩3.2억)
- 모바일 최소 폰트 사이즈 가이드

### 3. 컴포넌트 가이드라인
다음 핵심 컴포넌트의 디자인 방향을 정의해주세요:
- **점수 카드**: 종합 점수 + 4개 카테고리(예산·통근·보육·치안) 시각화
- **단지 리스트 아이템**: 썸네일, 단지명, 가격대, 핵심 점수, CTA
- **비교 테이블**: 2~3개 단지를 카테고리별로 나란히 비교
- **온보딩 입력 폼**: 금액 슬라이더, 주소 검색, 스텝 인디케이터
- **지도 마커**: 점수 등급별 마커 색상/크기, 클러스터링
- **외부 링크 CTA**: "네이버 부동산에서 매물 보기" 버튼 (외부 이동 명시)

### 4. 레이아웃 패턴
- 모바일: 결과 화면에서 지도/리스트 전환 방식 (탭 vs 바텀시트 vs 스와이프)
- 데스크톱: 좌측 리스트 + 우측 지도 2단 레이아웃 비율
- 반응형 브레이크포인트 제안

### 5. 마이크로인터랙션
- 점수 로딩 애니메이션 (계산 중 → 결과)
- 지도 마커 호버/탭 시 프리뷰 카드
- 비교 목록 추가/제거 시 피드백
- 외부 링크 이동 시 "외부 사이트로 이동합니다" 안내

각 항목을 Tailwind CSS 기반으로 구현 가능하도록 구체적인 클래스명이나 수치를 포함해주세요.
```

## 프롬프트 4: 온보딩 플로우 상세 설계

```
당신은 프로덕트 디자이너이자 프론트엔드 개발자입니다.

아래 서비스의 온보딩(조건 입력) 플로우를 상세 설계해주세요.

## 수집할 정보 (순서 미정)
1. 보유현금 (만 원 단위)
2. 합산연봉 (만 원 단위)
3. 기존대출 잔액 (만 원 단위)
4. 직장1 주소 (주소 검색)
5. 직장2 주소 (주소 검색, 외벌이면 "없음" 선택 가능)
6. 최대 월주거비 (만 원 단위)
7. 자녀 계획 (있음/없음/이미 있음 → 보육 가중치 결정)
8. 주거 형태 선호 (매매/전세/월세)

## 제약 조건
- **입력값 비저장**: 금융 정보(1~3번)는 서버에 저장하지 않음. 사용자에게 이 사실을 명확히 전달하여 입력 신뢰도를 높여야 함.
- **모바일 우선**: 엄지 한 손 조작 가능해야 함
- **3분 이내 완료**: 이탈률을 낮추기 위해 전체 플로우가 3분을 넘지 않아야 함
- **부동산 초보 친화**: 전문 용어 없이 쉬운 말로 안내

## 요청사항

### 1. 스텝 구성
- 8개 입력을 몇 개의 스텝으로 묶을지 결정 (너무 많으면 이탈, 너무 적으면 한 화면이 복잡)
- 각 스텝의 제목, 부제목, 입력 UI 패턴 정의
- 스텝 간 진행률 표시 방식

### 2. 입력 UI 패턴
각 입력값에 최적인 UI 컴포넌트를 제안해주세요:
- 금액 입력: 슬라이더 vs 직접입력 vs 구간 선택 버튼 중 어떤 것이 최적인가?
- 주소 검색: 카카오 주소 검색 API 연동 시 UX 플로우
- 선택지: 카드 선택 vs 라디오 vs 토글

### 3. 신뢰 확보 UI
- "입력하신 금융 정보는 저장되지 않습니다" 메시지를 어디에, 어떻게 노출할지
- 개인정보 처리 동의를 플로우 중 어느 시점에 받을지
- 보안 아이콘/배지 활용 방안

### 4. 이탈 방지 UX
- 중간 이탈 시 "여기까지만 입력해도 결과를 볼 수 있어요" (선택 항목 스킵 가능)
- 뒤로 가기 시 입력값 유지
- 입력 도중 "왜 이 정보가 필요한지" 설명 툴팁

### 5. 결과 전환
- 마지막 스텝 완료 후 결과 화면으로의 전환 UX (로딩 애니메이션, 기대감 조성)
- "분석 중" 상태의 시각적 표현

React + Tailwind CSS 기반으로, 각 스텝의 와이어프레임 수준 JSX 코드를 함께 제시해주세요.
```

## 프롬프트 5: 결과 화면 & 점수 시각화 설계

```
당신은 데이터 시각화 전문 프로덕트 디자이너입니다.

아래 서비스의 결과 화면(추천 단지 리스트 + 상세 점수)을 설계해주세요.

## 데이터 구조
각 추천 단지는 다음 정보를 가집니다:
- 단지명, 주소, 세대수, 준공년도
- 매매/전세/월세 실거래가 범위
- **종합 점수** (0~100)
- **예산 적합도** (0~100): 보유현금 + 대출 가능액 대비 가격 적합도
- **통근 점수** (0~100): 직장1·직장2까지의 가중 평균 통근시간 기반
- **보육 점수** (0~100): 반경 1km 어린이집·유치원·초등학교 접근성
- **안전 점수** (0~100): CCTV, 가로등, 경찰서 등 안전 인프라 기반 (주의: "치안 위험" 등 부정적 표현 금지, "안전 편의시설 현황"으로 프레이밍)
- 직장1까지 대중교통 소요시간 (분)
- 직장2까지 대중교통 소요시간 (분)

## 화면 구성 요청

### 1. 리스트 뷰
- 단지 카드 디자인 (모바일: 풀 와이드, 데스크톱: 그리드)
- 카드에 노출할 핵심 정보 우선순위
- 정렬 옵션 (종합 점수순, 가격순, 통근시간순)
- 무한 스크롤 vs 페이지네이션 vs "더 보기" 버튼

### 2. 지도 뷰
- 마커 디자인: 점수 등급별 색상 (상/중/하 3단계 또는 5단계)
- 마커 클릭 시 미니 카드 프리뷰 내용
- 지도-리스트 연동 (리스트 항목 호버 시 해당 마커 하이라이트)
- 통근 경로 시각화 여부 (직장 위치 마커 + 대중교통 라인)
- 모바일에서 지도-리스트 전환 UX

### 3. 점수 시각화
아래 시각화 방식을 비교 평가하고 최적 조합을 추천해주세요:
- 레이더 차트 (4축: 예산·통근·보육·안전)
- 수평 바 차트
- 원형 게이지 (종합 점수)
- 도넛 차트
- 숫자 + 컬러 배지 (미니멀)
- 아이콘 기반 (★★★☆☆)

고려 사항:
- 모바일 작은 화면에서 가독성
- 4개 카테고리를 한눈에 비교 가능성
- 구현 복잡도 (React + Tailwind + 차트 라이브러리)
- 색맹 접근성

### 4. 단지 상세 페이지
- 정보 섹션 구성 순서 (Hero → 점수 → 가격 → 통근 → 보육 → 안전 → CTA)
- 각 섹션의 시각화 방식
- "네이버 부동산에서 매물 보기" CTA 디자인 (외부 이동 명시 필수)
- "비교함에 추가" 인터랙션

### 5. 비교 기능
- 2~3개 단지를 나란히 비교하는 UI 패턴
- 카테고리별 "이 단지가 더 좋음" 하이라이트
- 모바일에서의 비교 UX (스와이프 vs 탭 vs 스크롤)
- 카카오톡 공유 시 미리보기 카드 디자인

React + Tailwind CSS + Recharts 기반으로 구현 가능한 수준의 컴포넌트 구조를 함께 제시해주세요.
```

## 프롬프트 6: 법적 제약을 반영한 UI/UX 가드레일

```
당신은 규제 준수(Compliance) 전문 UX 디자이너입니다.

아래 서비스는 여러 법적 제약 사항이 있습니다. 이를 사용자 경험을 해치지 않으면서 자연스럽게 반영하는 UI/UX를 설계해주세요.

## 법적 제약 사항

### 1. 공인중개사법 — 중개행위 금지
- UI에서 "추천", "알선", "중개"라는 용어 사용 금지
- 대체 용어: "분석", "안내", "정보", "탐색"
- 특정 매물에 대해 특정 중개사를 직접 연결하는 기능 금지
- CTA는 반드시 "외부 매물 페이지 링크" 방식 (네이버 부동산, 직방 등)
- 서비스가 "정보 분석 플랫폼"임을 명확히 전달해야 함

### 2. 개인정보보호법 — 동의 및 비저장
- 서비스 이용 전 개인정보 수집·이용 동의 필수 (필수/선택 분리)
- 금융 정보(연봉·대출)는 서버에 저장하지 않음 → 이 사실을 사용자에게 전달하여 신뢰 확보
- 위치정보 이용에 대한 별도 동의

### 3. 치안 데이터 표현 제한
- "위험", "범죄", "치안 열악" 등 부정적 표현 절대 금지
- "안전 편의시설 현황"으로 프레이밍 (CCTV 수, 가로등 밀집도, 경찰서 거리 등 긍정적 지표)
- 특정 지역을 비하하는 인상을 주지 않아야 함

### 4. 외부 링크 UX
- 매물 상세 페이지가 외부 사이트임을 명확히 표시
- 외부 이동 시 "외부 사이트로 이동합니다" 안내 필요
- 출처 표시 필수 (공공 데이터 기반)

## 요청사항

1. **용어 가이드라인**: 서비스 전체에서 사용할 용어/사용하지 말아야 할 용어 매핑 테이블
2. **동의 플로우 UX**: 개인정보 동의를 온보딩에 자연스럽게 녹이는 방법. 법적 요건을 충족하면서 이탈률을 최소화하는 디자인
3. **신뢰 UI 패턴**: "정보 비저장" 메시지, 보안 배지, 데이터 출처 표시를 매끄럽게 통합하는 방법
4. **안전 점수 UI**: 부정적 표현 없이 안전 정보를 효과적으로 전달하는 시각화 (좋은 예시 / 나쁜 예시 비교)
5. **외부 링크 CTA 패턴**: 외부 이동을 명시하면서도 사용자가 자연스럽게 행동하도록 유도하는 버튼/링크 디자인
6. **면책 고지 배치**: "본 서비스는 부동산 중개 서비스가 아닌 정보 분석 서비스입니다"를 어디에, 어떻게 배치할지

각 항목에 대해 구체적인 UI 목업 수준의 설명(또는 JSX 코드)을 제공해주세요.
```

## 프롬프트 7: 전체 페이지 구조 & 라우팅 설계

```
당신은 풀스택 프로덕트 디자이너입니다.

아래 서비스의 전체 페이지 구조(사이트맵)와 라우팅을 설계해주세요.

## 서비스 개요
- 수도권 신혼부부 대상 주거 추천 서비스
- 모바일 우선 웹앱 (React + Next.js 예상)
- 주요 기능: 조건 입력 → 추천 결과(리스트/지도) → 단지 상세 → 비교 → 외부 매물 링크
- 비회원도 이용 가능 (MVP 단계), 프리미엄 구독 시 회원가입

## 요청사항

### 1. 사이트맵
전체 페이지 목록을 트리 구조로 정리해주세요:
- 퍼블릭 페이지 (비회원 접근 가능)
- 인증 필요 페이지 (프리미엄)
- 유틸리티 페이지 (약관, 개인정보처리방침, 위치정보이용약관)

### 2. 라우팅 테이블
각 페이지의 URL 패턴, 페이지 제목, 핵심 컴포넌트를 표로 정리해주세요.

### 3. 네비게이션 설계
- 모바일: 하단 탭 바 구성 (최대 5개)
- 데스크톱: 상단 헤더 네비게이션
- 각 탭/메뉴의 아이콘 + 라벨

### 4. 사용자 플로우 다이어그램
주요 시나리오 3개의 플로우를 mermaid 다이어그램으로 그려주세요:
- 신규 방문 → 조건 입력 → 결과 확인 → 단지 상세 → 외부 매물 링크
- 결과 화면 → 비교 기능 → 카카오톡 공유
- 재방문 → 조건 재입력 (비저장이므로) → 결과 확인

### 5. SEO 및 메타 태그
- 각 페이지의 title, description 템플릿
- OG 태그 (카카오톡 공유 시 미리보기)
- 동적 페이지(단지 상세)의 SEO 전략
```

## 프롬프트 8: 디자인 리서치 통합 브리프

```
당신은 시니어 프로덕트 디자이너이자 테크니컬 라이터입니다.

## 목표
P1~P7 디자인 리서치 산출물을 개발팀이 바로 참조할 수 있는 단일 디자인 브리프로 통합합니다.

## 출력 형식

- 스텝 수와 각 스텝 요약 (1줄씩)

- 면책 고지 위치
```
