## 1) MVP Goal

- MVP가 증명해야 하는 것(가설):
  - `설명 가능한 컨시어지형 Top10 추천 리포트`가 맞벌이 신혼부부의 문의 행동을 유의미하게 유도한다.
- MVP 성공 기준(1차):
  - 컨시어지 문의 CTR: 기준 `18~21%`, 목표 `>=20%`
  - 최소 입력 완료율: 기준 `50~60%`, 목표 `>=50%`
- 게이트 조건:
  - 전환 KPI 충족 + Step0 법무 선결조건 진행계획 확정

## 2) Target User Journey

1. 유입: 랜딩에서 서비스 포지션(정보 제공)과 면책 문구 확인
2. 첫 사용: 최소 입력(예산 범위/직장 권역/통근 기준)으로 시작, 정밀 입력은 선택
3. 핵심 가치 경험: Top10 단지 결과에서 점수 근거/출처/기준일/단점 확인
4. 행동 전환: 외부 매물 링크로 이동(문의/계약은 외부 서비스에서 진행)

## 3) Feature Scope (P0/P1/P2)

| 기능 | 목적 | 우선순위 | 제외/포함 기준 |
| --- | --- | --- | --- |
| 컨시어지형 Top10 리포트 | 추천 신뢰 + 문의 전환 | P0 | 점수 분해/출처/기준일/Why-Not 표시 필수 |
| 점진 입력 퍼널(최소/정밀) | 민감정보 입력 이탈 완화 | P0 | 정밀 입력은 선택값으로만 노출 |
| 신뢰 UI(한계/면책/외부이동 고지) | 중개/광고 오인 방지 | P0 | 법무 문구 정책 위반 시 출시 불가 |
| 아웃링크 품질 가드레일 | 클릭 후 신뢰 유지 | P0 | 중복/만료/신선도 경고 포함 |
| 전세 안전 필터(기본) | 전세 탭 체감가치 | P1 | 고급 리스크 모델은 제외 |
| 2직장 통근 계산 고도화 | 차별화 강화 | P1 | P0는 기준 시간대 기반 근사 계산 |
| GPS 기반 정밀 위치 기능 | 통근 정확도 향상 | P2 | 신고/약관/동의/보호조치 완료 후만 오픈 |

## 4) Requirements

기능 요구사항:

1. 전세/매매 탭 분리 + 아파트 대상 Top10 결과 제공
2. 통근(2직장 중 긴 쪽 기준), 예산, 학군, 보육, 치안 점수화 제공
3. 결과 화면에 근거와 한계를 동시에 표시
4. 문의 CTA는 외부 이동 고지를 포함한 링크 아웃 방식으로만 동작

비기능 요구사항:

1. 개인정보 최소수집/파기 정책 적용
2. DB/로그/APM 비저장 및 마스킹 통제
3. 동의 이력/정책 버전 감사 가능성 확보
4. 크롤링 금지, 공공/API/제휴 데이터 소스만 허용

분석 이벤트(트래킹) 요구사항:

1. `landing_unique_view`, `min_input_start`, `min_input_complete`
2. `result_view`, `concierge_contact_click`, `outlink_click`
3. `consent_shown`, `consent_accepted`, `policy_version`

## 6) Non-goals

- MVP에서 하지 않을 기능:
  - 계약/협상/중개 행위
  - 거래성사형 수수료 모델(CPA)
  - 대출 승인 확정/법률 자문 오인 기능
  - 무단 크롤링 기반 매물 수집
- 이후 버전으로 미루는 이유:
  - 법적 리스크 및 데이터 정합성 리스크가 높아 Step0 통제 완료 후 확장 필요
