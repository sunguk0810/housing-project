# Step 1. Define Discovery

Status: done  
Owner: sungwookhwang  
Date: 2026-02-13

## 1) Problem Statement

템플릿:

`[타깃 사용자]는 [상황]에서 [핵심 문제]를 겪고 있으며, 그 결과 [비용/불편]이 발생한다.`

작성:

- 타깃 사용자: 서울/경기/인천 생활권의 맞벌이 신혼부부(결혼 예정~결혼 2년 이내)
- 상황: 전세/매매 아파트를 검토하면서 두 직장 통근, 보육 접근성, 치안, 예산을 동시에 맞춰야 하는 상황
- 핵심 문제: 기존 앱은 탐색 정보는 많지만 재무+이중통근+보육+치안을 통합 랭킹으로 제시하지 못함
- 비용/불편: 앱/엑셀/중개사를 오가는 수동 비교 비용이 높고 의사결정 불안이 커짐

문제정의 1문장:

`서울/수도권 맞벌이 신혼부부는 보유현금과 소득 범위 내에서 전세/매매 아파트 후보를 빠르게 비교하기 어려워, 통근·보육·치안을 동시에 만족하는 최적 선택을 찾는 데 높은 시간/인지 비용을 지불한다.`

## 2) Goal / Success Metrics

- North Star Metric: 추천 결과 노출 대비 `매물 문의 클릭률`
- 1차 성공지표:
  - 랜딩 실험 문의 클릭률 `>= 12%` 또는
  - 컨시어지 리포트 문의 클릭률 `>= 20%`
- 가드레일 지표(부작용 방지):
  - 민감정보(연봉/대출/직장주소) 입력 조건부 수용률 `>= 50%`
  - 결과 노출률(입력 완료 대비 추천 결과 도달) `>= 70%`

## 3) Scope / Non-goals

- 이번 사이클에서 할 것:
  - 1차 지역 범위: 서울 + 핵심 경기권(예: 성남/과천/하남)
  - 아파트 대상 전세/매매 탭 분리
  - 입력 기반(보유현금/예산/월상한/직장1·2) Top10 단지 추천
  - 점수화: 예산/통근/보육/치안/학군
  - 통근 60분, 보육 10분 근사, 월 고정비 추정, 외부 매물 아웃링크
- 이번 사이클에서 하지 않을 것(Non-goals):
  - 매물(동/호) 직접 제공
  - 대출 승인/가능액 확정 제시
  - 월세/빌라/오피스텔 동시 확장
  - 치안 "안전/위험" 단정 라벨링

## 4) Core Hypotheses

- 가설 1: 신혼부부는 `재무 + 2직장 통근 + 보육 + 치안` 통합 랭킹을 기존 탐색형 앱보다 더 가치 있게 느낀다.
- 가설 2: 추천 근거(점수 breakdown + 사유 문장)를 제공하면 민감정보 입력 장벽을 완화할 수 있다.
- 가설 3: 추천 정확도 자체보다 "문의 클릭" 행동 지표가 초기 PMF 신호로 더 빠르게 작동한다.

## 5) Risks

- 가장 큰 리스크:
  - 민감정보 입력 거부로 초기 퍼널 이탈
  - 공공데이터 한계로 추천 품질/설명 가능성 저하
- 리스크 완화 방법:
  - 간편 입력 모드(월상한 중심) 제공
  - 데이터 기준일/가정 표시 + 추천 사유 문장 표준화
  - P0는 단지 추천 + 아웃링크 중심으로 스코프 관리

## 6) Q&A Loop (미확정 항목)

확정된 항목:

1. North Star 분모는 `결과 노출`로 고정
2. 가드레일 `결과 노출률 >= 70%` 유지
3. 1차 출시 지역은 `서울 + 핵심 경기권`으로 축소
4. Step1 완료 기준의 "팀 합의"는 `본인 단독 의사결정`으로 대체
