# 서비스 기획서 – 신혼부부 주거 추천 서비스

## 1. 기획 배경

한국의 신혼부부, 특히 맞벌이 부부는 첫 주거 선택에서 **예산, 통근, 보육, 학군, 치안** 등 다양한 요소를 동시에 고려해야 하나, 이를 통합적으로 분석해 주는 서비스가 거의 없습니다. 기존 부동산 플랫폼은 예산 역산이나 두 직장의 통근시간, 보육·치안 데이터를 반영하지 못해 **명확한 시장 공백**이 존재합니다【103981203960773†L11-L16】【103981203960773†L41-L53】. 본 서비스는 이러한 공백을 채우고, 데이터 기반 의사결정 경험을 제공하고자 기획되었습니다.

## 2. 시장 및 대상 분석

- **대상 사용자**: 혼인 5년 이내의 맞벌이 신혼부부 5만여 쌍(2024년 기준)과 예비부부. 이들은 자녀 계획이 있거나 보육·교육 환경을 중시하며, 통근 시간을 줄이고자 하는 수요가 강함.
- **경쟁 서비스**: 네이버·직방 등 기존 부동산 포털은 매물 검색과 중개 중심이다. 일부 지역 맞춤형 서비스가 있으나 예산 역산 기능과 보육·치안 지표 통합이 부족하다【103981203960773†L41-L53】.
- **기회**: 공공데이터의 개방 확대와 API 호출 한도 증가(예: CCTV 통합관제센터 데이터의 오픈 API 제공【784864165518395†L74-L91】)로 정보 격차를 줄일 수 있는 환경이 조성되고 있다. 정보 제공 서비스는 법적 규제가 덜하며 사용자 신뢰를 얻기 쉽다.

## 3. 문제 정의

1. **정보 단절** – 예산, 두 직장 통근, 보육시설, 학군, 안전 인프라를 종합적으로 비교해주는 플랫폼이 없다.
2. **사용자 부담** – 기존 서비스는 매물을 하나씩 조회해야 하며, 정확한 비용 계산이나 통근시간 예측이 어렵다.
3. **낙인 논란** – 치안 데이터를 부적절하게 표현하면 특정 지역에 대한 편견을 유발할 수 있다【543942488311355†L41-L49】.

## 4. 서비스 컨셉 및 핵심 가치

- **예산 기반 추천** – 사용자의 현금·연소득·대출 정보를 입력하면 가용 예산을 자동 산정하여 적정 단지를 찾는다.
- **두 직장 통근 최적화** – 두 주소를 동시에 고려해 대중교통 기준 통근시간을 계산하고, 1시간 이내 등 조건을 설정할 수 있다.
- **보육·학군·안전 통합 지표** – 어린이집 수, 학교 성취도, CCTV·대피시설 등 안전 인프라 지표를 가중치별로 조합해 종합 점수를 산출한다. 부정적 표현을 피하고 인프라 정보 중심으로 제시한다【543942488311355†L41-L49】.
- **설명 가능한 결과** – 상위 10개 단지에 대해 월 고정비, 통근시간, 지표별 점수를 시각화하여 추천 근거를 명확히 보여준다.
- **정보 제공 플랫폼** – 매물 중개나 금융 상품 판매를 하지 않고, 외부 사이트 링크만 제공함으로써 법적 리스크를 최소화한다【543942488311355†L41-L49】.

## 5. 기능 로드맵

### P0 (필수 기능, 8주)

- 사용자 입력 폼: 현금·연소득·대출·월 상한·직장 주소 2곳·가중치 선택.
- 예산 역산 로직 및 월 고정비 계산.
- 기본 통근시간 계산(사전 계산+API 호출) 및 캐싱.
- 보육/학군/치안 지표 산정 및 종합 점수 계산.
- 지도 및 카드 기반 추천 결과 표시, 외부 링크 연결.
- 이용약관·개인정보 처리방침·주거지원 정책 소개 페이지 작성.

### P1 (추가 기능, 4주)

- 가중치 슬라이더 및 사용자 맞춤형 추천 프로필 저장.
- 주거 지원 정책 알림: 결혼/출산에 따른 정책 변화와 대출 상품 안내.
- 결과 공유 기능: 카카오톡 공유, PDF 저장.
- 시각화 강화: 레이더 차트·막대 차트로 점수 구성 요소를 표시.

### P2 (확장 아이디어)

- 사용자 계정 생성 및 즐겨찾기 기능.
- 파트너 중개사와 제휴하여 샘플 매물 정보 제공(중개 행위는 하지 않음).
- AI 챗봇을 통한 Q&A 및 라이프스타일 기반 추천.

## 6. 데이터 및 기술 전략 (요약)

- **데이터 수집**: 국토교통부 실거래가, 사회보장정보원 어린이집·유치원, 교육부 학교알리미, 행정안전부 재난안전데이터(CCTV, 대피시설) 등을 정기적으로 수집한다. 데이터는 월 혹은 분기별로 갱신한다【784864165518395†L74-L91】.
- **데이터베이스**: PostgreSQL과 PostGIS로 공간 데이터 연산을 수행한다. 테이블 설계는 `apartments`, `apartment_prices`, `childcare_centers`, `schools`, `safety_stats` 등으로 분리한다.
- **백엔드**: Next.js API Routes 또는 Express.js. 예산 계산 및 점수 계산 엔진을 모듈화한다. Redis를 활용해 통근시간과 주요 쿼리를 캐시한다.
- **프런트엔드**: Next.js + Tailwind CSS + 카카오 지도 SDK. 모바일 우선 설계와 반응형 UI를 구현한다.
- **보안/컴플라이언스**: HTTPS 적용, 개인 데이터 저장금지, 공공데이터 출처 명시, 중개행위 회피【543942488311355†L15-L21】【543942488311355†L41-L49】.

## 7. 디자인 및 사용자 경험

- **톤 앤 매너**: 신뢰감 있는 색상(블루/그린)과 깔끔한 레이아웃. 데이터 기반 서비스임을 강조하기 위해 인포그래픽 스타일을 활용한다.
- **입력 흐름**: 질문을 단계별로 나누어 사용자의 부담을 줄이고, 가이드 텍스트와 예시를 제공한다.
- **결과 시각화**: 지도와 카드 외에도 차트로 지표별 점수 분포를 보여줘 사용자가 이해하기 쉽게 한다.
- **접근성 고려**: 색약 사용자도 구분 가능한 색상 팔레트, 키보드 탐색 지원.

## 9. 리스크 및 대응 전략

- **API 호출 제한** – ODsay 등 통근 API 호출량 제한에 대비해 사전 계산된 통근 그리드와 Redis 캐시를 활용한다.
- **데이터 품질** – 공공 데이터의 누락·오류 가능성을 염두에 두고, 수집 시 예외 처리와 데이터 검증 로직을 포함한다.
- **법적 리스크** – 부동산 중개행위로 오해받지 않도록 외부 링크만 제공하고, 이용약관과 정책서를 통해 서비스 성격을 명시한다【543942488311355†L41-L49】.
- **개발 리소스** – 1인 개발자의 시간 제약을 고려하여 MVP 범위(P0)에 집중하고, P1/P2는 여유가 있을 때 추가한다.

## 10. 향후 전망 및 단계

1. **데이터 확장** – 생활편의 데이터(공중화장실, 무인민원발급기 등)와 의료·문화시설 데이터를 추가하여 지표 다양성을 높인다.
2. **사용자 참여** – 향후에는 사용자 리뷰나 커뮤니티 기능을 도입해 주관적 경험과 데이터 분석을 결합할 수 있다.
3. **상업화 고려** – 서비스가 안정화되면 중개사 제휴나 금융상품 광고 등 수익 모델을 탐색하되, 법적 자문을 선행해야 한다.
