설계 원칙 (§1)
1. 비동기 협업 우선

커플 탐색은 각자 따로, 짧게, 반복적. 공유 찜 목록 + "둘 다 찜/한 명만 찜" 매칭 표시.

2. 3입력 내 첫 가치 전달

소득·자산·대출 3개 입력만으로 예산 범위 즉시 표시. 온보딩 완료 후 결과 화면으로 직접 라우팅.

3. 지도가 곧 검색

바운딩 박스 자동 갱신 + 가격 라벨 핀 + 3단 바텀시트. 별도 "검색" 버튼 없이 자동 재검색.

4. 점수는 이야기

0-100 점수 + 라벨 + 요인 분해 + 백분위 4층 구조. 3단계 계층적 공개.

5. 실거래가가 최고의 신뢰

호가 옆에 동일 조건 실거래 이력 항상 병기. 모든 가격에 출처·기준일 인라인 태그.

6. 점진적 몰입

등록 없이 탐색 → 카카오 로그인 → 파트너 초대. 결과 확인까지 로그인 미요구.

7. 신뢰 신호는 적지만 깊게

검증된 소수의 신뢰 장치(실거래가·출처 태그·🔒 비저장)를 콘텐츠에 통합.
